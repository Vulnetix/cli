---
title: "VNX-271 – Privilege Dropping / Lowering Errors"
description: "Detect setuid() and setgid() called without checking return values, and setuid(getuid()) that doesn't clear saved UID."
---

## Overview

`setuid()` can fail if the process lacks `CAP_SETUID`. A failed setuid() without error checking leaves the process running as root. Additionally, `setuid(getuid())` only sets the effective UID, not the saved set-user-ID, so the process can re-elevate. This maps to [CWE-271: Privilege Dropping / Lowering Errors](https://cwe.mitre.org/data/definitions/271.html).

**Severity:** Medium | **CWE:** [CWE-271](https://cwe.mitre.org/data/definitions/271.html)

## Remediation

```c
// SAFE: check setuid and verify it worked
if (setuid(www_uid) != 0) {
    perror("setuid");
    exit(EXIT_FAILURE);
}
// Verify: should no longer be able to re-elevate
if (setuid(0) != -1) {
    fprintf(stderr, "setuid privilege drop failed\n");
    exit(EXIT_FAILURE);
}

// SAFE: use setresuid to clear all three UIDs
if (setresuid(www_uid, www_uid, www_uid) != 0) {
    perror("setresuid");
    exit(EXIT_FAILURE);
}
```
