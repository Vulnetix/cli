---
title: "VNX-129 – Improper Validation of Array Index"
description: "Detect array, slice, or list accesses using unchecked user-supplied indices that can cause out-of-bounds reads or writes."
---

## Overview

This rule flags code that uses a value derived from user input (via `atoi`, `Integer.parseInt`, `strconv.Atoi`, or `int(request.)`) as an array or list index without first validating that the index is within the valid range `[0, size)`. An out-of-range index causes a crash, buffer over-read, or heap corruption depending on the language and runtime. This maps to [CWE-129: Improper Validation of Array Index](https://cwe.mitre.org/data/definitions/129.html).

**Severity:** High | **CWE:** [CWE-129 – Improper Validation of Array Index](https://cwe.mitre.org/data/definitions/129.html)

## What Gets Flagged

```c
// FLAGGED: C — unchecked index from argv
int idx = atoi(argv[1]);
printf("%d\n", arr[idx]);  // no bounds check
```

```java
// FLAGGED: Java — parseInt without bounds check
int idx = Integer.parseInt(request.getParameter("index"));
String item = list.get(idx);
```

```python
# FLAGGED: Python — int from request without bounds
data[int(request.args['idx'])]
```

## Remediation

```c
// SAFE: C
int idx = atoi(argv[1]);
if (idx < 0 || idx >= ARRAY_SIZE) { return -1; }
printf("%d\n", arr[idx]);
```

```java
// SAFE: Java
int idx = Integer.parseInt(request.getParameter("index"));
if (idx < 0 || idx >= list.size()) { throw new IllegalArgumentException(); }
String item = list.get(idx);
```

```python
# SAFE: Python
try:
    idx = int(request.args['idx'])
except ValueError:
    abort(400)
if idx < 0 or idx >= len(data):
    abort(400)
item = data[idx]
```
