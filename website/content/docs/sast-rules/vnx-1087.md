description: "Detect C++ classes with virtual methods but no virtual destructor — causes undefined behaviour when deleting via base pointer."
---

## Overview

Detects C++ classes that define virtual methods but do not declare a virtual destructor. If the class is used as a base class, deleting a derived object through a base pointer without a virtual destructor is undefined behaviour.

**Severity:** Medium | **CWE:** [CWE-1087 – Class with Virtual Method without a Virtual Destructor](https://cwe.mitre.org/data/definitions/1087.html)

## Why This Matters

The missing virtual destructor is one of the most common C++ bugs. When a derived object holds sensitive resources (memory, file handles, cryptographic keys) and is deleted through a base pointer, those resources are never released — creating both a resource leak and potentially a use-after-free if another reference exists.

## What Gets Flagged

```cpp
// FLAGGED: virtual method but no virtual destructor
class Authenticator {
public:
    virtual bool verify(const std::string& token);
    // destructor is not virtual!
};
```

## Remediation

```cpp
class Authenticator {
public:
    virtual ~Authenticator() = default;  // ADD THIS
    virtual bool verify(const std::string& token);
};
```
