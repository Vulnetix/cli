title: "VNX-224 – Obscured Security-relevant Information by Alternate Name"
description: "Detect sensitive field names using abbreviations or aliases that bypass security scanners."
---

## Overview

Using abbreviated names (`pw`, `tok`, `sec`) for security-sensitive fields in log statements or data structures causes these fields to be missed by automated security scanners and manual reviewers. If `pw` is logged accidentally, the leak may not be caught. This maps to [CWE-224: Obscured Security-relevant Information by Alternate Name](https://cwe.mitre.org/data/definitions/224.html).

**Severity:** Low | **CWE:** [CWE-224](https://cwe.mitre.org/data/definitions/224.html)

## Remediation

Use standard, descriptive names for sensitive fields (`password`, `token`, `api_key`) and ensure these are masked or redacted in all log output.
