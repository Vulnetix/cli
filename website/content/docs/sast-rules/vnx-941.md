description: "Detect sensitive data sent to wrong or suspicious endpoints including localhost URLs, test services, and dynamically constructed destinations."
---

## Overview

When an application sends sensitive data to an incorrect destination — due to misconfigured URLs, environment variable confusion, test endpoints left in production, or dynamic URL construction using user input — that data reaches an unintended and potentially attacker-controlled location.

**Severity:** Medium | **CWE:** [CWE-941 – Incorrectly Specified Destination in a Communication Channel](https://cwe.mitre.org/data/definitions/941.html)

## Why This Matters

Test webhook services (webhook.site, ngrok, requestbin) and localhost addresses are common sources of data leaks when forgotten test configurations are deployed to production. Dynamic URL construction from user input enables server-side request forgery (SSRF).

## What Gets Flagged

```python
# FLAGGED: Test endpoint in production code
requests.post("https://webhook.site/abc123", json=sensitive_data)

# FLAGGED: Dynamic URL near sensitive data
url = f"https://{endpoint}/report"
requests.post(url, json={"password": password})
```

## Remediation

Use configuration-controlled endpoints and validate against an allowlist:

```python
# SAFE: Load from configuration, validate
endpoint = os.environ['REPORTING_ENDPOINT']
assert endpoint.startswith('https://reports.company.com/')
requests.post(endpoint, json=payload)
```
