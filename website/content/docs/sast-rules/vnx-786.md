---
title: "VNX-786 – Access of Memory Location Before Start of Buffer"
description: "Detect C negative array indices and pointer subtraction without bounds checks that access memory before the start of a buffer."
---

## Overview

This rule flags C code that accesses memory before the start of an allocated buffer via explicit negative array indices (`buf[-1]`) or pointer subtraction used as array access without a preceding bounds check. This maps to [CWE-786: Access of Memory Location Before Start of Buffer](https://cwe.mitre.org/data/definitions/786.html).

**Severity:** High | **CWE:** [CWE-786](https://cwe.mitre.org/data/definitions/786.html)

## Why This Matters

Buffer underread/underwrite accesses memory before the allocation, which may contain other heap chunks, stack frames, or read-only memory. Writing before a buffer corrupts adjacent allocations; reading before a buffer may disclose sensitive data from previous allocations.

## What Gets Flagged

```c
// FLAGGED: explicit negative index
int buf[10];
buf[-1] = 42;  // writes before the buffer

// FLAGGED: pointer subtraction as index without bounds check
char *ptr = buffer + offset;
char val = ptr[-2];  // may read before buffer start
```

## Remediation

```c
// SAFE: validate index before access
if (i >= 0 && i < BUF_SIZE) {
    value = buf[i];
}

// SAFE: validate pointer bounds before subtraction
if (ptr >= buffer && ptr - n >= buffer) {
    char val = *(ptr - n);
}
```
