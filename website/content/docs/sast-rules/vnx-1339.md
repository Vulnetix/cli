---
title: "VNX-1339 – Insufficient Precision or Accuracy of a Real Number"
description: "Detect floating-point types used for monetary or financial values, where IEEE 754 precision limitations cause silent rounding errors that accumulate in calculations."
---

## Overview

IEEE 754 floating-point numbers cannot represent most decimal fractions exactly. `0.1 + 0.2` in floating point equals `0.30000000000000004`, not `0.3`. For financial calculations involving prices, amounts, balances, or fees, this imprecision accumulates and can result in incorrect totals, audit discrepancies, or exploitable rounding behaviours. **Severity:** Medium | **CWE:** [CWE-1339 – Insufficient Precision or Accuracy of a Real Number](https://cwe.mitre.org/data/definitions/1339.html)

## Why This Matters

Floating-point rounding errors in financial software have caused real-world losses. The Vancouver Stock Exchange index error (1982) was due to truncation rather than rounding, losing 0.0001 per truncation over 2800 daily transactions — the index drifted 48 points over 22 months. Payment processors using float arithmetic may round in ways that allow fractional cent extraction at scale.

## What Gets Flagged

```java
// FLAGGED: float/double for monetary variable
float price = 19.99f;
double total = 0.0;
double balance = account.getBalance();
```

```python
# FLAGGED: float() for currency value
price = float(product.price)
amount = float(request.POST['amount'])
total = float(cart.total)
```

```javascript
// FLAGGED: parseFloat for monetary
const price = parseFloat(req.body.price);
const amount = parseFloat(req.body.amount);
const total = parseFloat(cart.total);
```

## Remediation

```java
// SAFE: Use BigDecimal for all monetary calculations
import java.math.BigDecimal;
import java.math.RoundingMode;

BigDecimal price = new BigDecimal("19.99");
BigDecimal quantity = new BigDecimal("3");
BigDecimal total = price.multiply(quantity).setScale(2, RoundingMode.HALF_UP);
```

```python
# SAFE: Use decimal.Decimal
from decimal import Decimal, ROUND_HALF_UP

price = Decimal('19.99')
quantity = Decimal('3')
total = (price * quantity).quantize(Decimal('0.01'), rounding=ROUND_HALF_UP)
```

```javascript
// SAFE: Use integer cents or a financial library
// Option 1: work in integer cents
const priceCents = Math.round(parseFloat(req.body.price) * 100);
const totalCents = priceCents * quantity;
const total = (totalCents / 100).toFixed(2);

// Option 2: dinero.js
const { Dinero } = require('dinero.js');
const price = Dinero({ amount: 1999, currency: 'USD' });
```
