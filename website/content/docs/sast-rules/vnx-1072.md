## Overview

Detects patterns that create new database connections for each operation instead of using connection pools. Creating connections per-request is slow, resource-intensive, and can be exploited to exhaust database connection limits causing denial of service.

**Severity:** Low | **CWE:** [CWE-1072](https://cwe.mitre.org/data/definitions/1072.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1072](https://cwe.mitre.org/data/definitions/1072.html) for authoritative guidance.
