title: "VNX-1336 – Server-Side Template Injection (SSTI)"
description: "Detect user-controlled input passed to template engines for rendering (render_template_string, pug.render, ERB.new, SpelExpressionParser), enabling arbitrary code execution on the server."
---

## Overview

Server-Side Template Injection occurs when user input is embedded in a template string that is then rendered by a server-side template engine. Template engines like Jinja2, Pug, ERB, and FreeMarker allow embedded code execution. When attackers control the template source, they can execute arbitrary server commands, read environment variables and files, and achieve full remote code execution. **Severity:** Critical | **CWE:** [CWE-1336 – Improper Neutralization of Special Elements Used in a Template Engine](https://cwe.mitre.org/data/definitions/1336.html)

## Why This Matters

SSTI is rated critical because it typically leads directly to Remote Code Execution (RCE). In Jinja2, the payload `{{config.from_object('os')}}` or `{{''.__class__.__mro__[1].__subclasses__()}}` allows accessing Python internals and executing OS commands. Uber's HackerOne program paid $10,000 for a Jinja2 SSTI in 2016. SSTI appears regularly in OWASP Top 10 discussions as a server-side injection variant.

## What Gets Flagged

```python
# FLAGGED: Flask render_template_string with user input
from flask import render_template_string, request

@app.route('/greet')
def greet():
    name = request.args.get('name', '')
    # Attacker sends: name={{config}} or name={{''.__class__.__mro__}}
    return render_template_string(f'Hello {name}!')  # SSTI
```

```javascript
// FLAGGED: pug.render with user-controlled template
const pug = require('pug');
app.post('/render', (req, res) => {
    const output = pug.render(req.body.template);  // RCE via pug template syntax
    res.send(output);
});
```

```java
// FLAGGED: SpEL expression from user input
ExpressionParser parser = new SpelExpressionParser();
Expression exp = parser.parseExpression(userInput);  // RCE via SpEL
Object result = exp.getValue(context);
```

```ruby
# FLAGGED: ERB with user input as template
require 'erb'
template = ERB.new(params[:template])
result = template.result(binding)  # RCE via <%= system("cmd") %>
```

## Remediation

The fundamental fix is to **never use user input as template source code**. Pass user data as template variables into a static template file:

```python
# SAFE: User data as variable, not template source
from flask import render_template, request

@app.route('/greet')
def greet():
    name = request.args.get('name', '')
    # Template is a file; name is data, not code
    return render_template('greet.html', name=name)
```

```javascript
// SAFE: User data as locals, template is a file
app.post('/render', (req, res) => {
    // Never pass req.body.template as the template itself
    res.render('output', { data: req.body.data });
});
```

```java
// SAFE: Evaluate expressions only from trusted sources
// For dynamic expressions from users, use a sandboxed evaluator
// or whitelist specific safe operations
```

If you need to allow user-customizable templates, use a **logic-less** template engine like Mustache or a sandboxed environment like Liquid (Ruby) with user-controllable syntax disabled.
