title: "VNX-587 – Assignment of a Fixed Address to a Pointer"
description: "Detect C/C++ code that assigns hardcoded memory addresses to pointers."
---

## Overview

Assigning a hardcoded hexadecimal address to a pointer (`ptr = (void*)0x12345678`) is non-portable and dangerous. The address is meaningful only on a specific system configuration, and on any other system it points to arbitrary memory — possibly kernel space, another process, or unmapped memory.

**Severity:** Medium | **CWE:** [CWE-587 – Assignment of a Fixed Address to a Pointer](https://cwe.mitre.org/data/definitions/587.html)

## Why This Matters

Hardcoded addresses that are written to can corrupt arbitrary memory. Reading from them can disclose kernel data or crash the process. In embedded/kernel contexts these are only valid for memory-mapped I/O registers and must be clearly documented.

## What Gets Flagged

```c
// FLAGGED: Hardcoded pointer assignment
void *ptr = (void*)0x12345678;
*((int*)0x1234) = value;

// FLAGGED: Writing to fixed address
int *reg = (int*)0x40001000;
*reg = 0xFF;
```

## Remediation

Use proper memory allocation or platform-abstracted MMIO macros:

```c
// SAFE: Dynamic allocation
void *ptr = malloc(sizeof(int));

// SAFE: MMIO with documented platform abstraction
#define GPIO_REGISTER  ((volatile uint32_t*)0x40001000)
*GPIO_REGISTER = 0xFF;  // Document: Platform X GPIO control register
```
