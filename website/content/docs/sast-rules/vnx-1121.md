## Overview

Detects functions with high cyclomatic complexity by counting decision points (if/else/for/while/case/&&/||). Functions with complexity above 15 are very hard to test completely, increasing the likelihood of untested security code paths.

**Severity:** Low | **CWE:** [CWE-1121](https://cwe.mitre.org/data/definitions/1121.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1121](https://cwe.mitre.org/data/definitions/1121.html) for authoritative guidance.
