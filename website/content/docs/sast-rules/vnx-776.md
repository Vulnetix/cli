## Overview

XML parsers that process DTD entity definitions can be exploited via the Billion Laughs attack: a small XML document with recursive entity references expands exponentially in memory, causing denial of service. Secure XML parsing requires disabling DTD processing or enabling secure processing features.

**Severity:** High | **CWE:** [CWE-776 – XML bomb (recursive entity expansion in DTDs)](https://cwe.mitre.org/data/definitions/776.html)

## Why This Matters

Violations of CWE-776 can expose the application to exploitation. This rule detects source code patterns associated with this weakness across multiple languages and frameworks.

## What Gets Flagged

This rule fires when source code contains patterns indicative of CWE-776. Review the flagged code to confirm whether the identified pattern represents a genuine vulnerability in context.
