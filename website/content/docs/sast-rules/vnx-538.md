---
title: "VNX-538 – Insertion of Sensitive Information into Externally-Accessible File"
description: "Detect sensitive data written to web-accessible directories or publicly readable file locations."
---

## Overview

Writing API keys, passwords, or configuration secrets to files within the web root or other publicly accessible directories allows attackers to retrieve them with a direct HTTP request. CWE-538 covers files written to `MEDIA_ROOT`, `static/`, `public/`, and similar web-accessible paths.

**Severity:** High | **CWE:** [CWE-538 – Insertion of Sensitive Information into Externally-Accessible File or Directory](https://cwe.mitre.org/data/definitions/538.html)

## Why This Matters

Any file in a web-accessible directory can be retrieved by any internet user if directory listing is enabled, if the filename is guessable, or if it is linked from another resource. Credentials in such files represent an immediate compromise.

## What Gets Flagged

```python
# FLAGGED: Writing API key to media directory
open(settings.MEDIA_ROOT + '/config.txt', 'w').write(api_key)
```

```php
// FLAGGED: Writing secret to public directory
file_put_contents('/var/www/html/public/secret.txt', $api_key);
```

## Remediation

Store secrets outside the web root and use environment variables:

```python
# SAFE: Secrets stored outside web root or in environment variables
import os
api_key = os.environ['API_KEY']  # Never write to filesystem
```

Use a secrets manager (AWS Secrets Manager, HashiCorp Vault) for production deployments.
