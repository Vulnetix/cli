title: "VNX-645 – Overly Restrictive Account Lockout Mechanism"
description: "Detect account lockout configurations that are too aggressive and can be used to deny service to legitimate users."
---

## Overview

Account lockout policies that trigger after only 1-2 failed attempts, or that lock accounts permanently (requiring manual admin intervention), can be weaponized as denial-of-service attacks against legitimate users. An attacker can deliberately lock out all accounts by submitting wrong passwords, preventing legitimate access to the service.

**Severity:** Low | **CWE:** [CWE-645 – Overly Restrictive Account Lockout Mechanism](https://cwe.mitre.org/data/definitions/645.html)

## Why This Matters

If an account locks after 1-2 failed attempts and a competing organization or malicious actor knows valid usernames (often easy to enumerate), they can run a script that submits one wrong password per account, locking out the entire user base simultaneously. This converts a security feature into a denial-of-service vector.

## What Gets Flagged

```python
# FLAGGED: Locks after only 1 attempt
MAX_LOGIN_ATTEMPTS = 1

# FLAGGED: Locks after 2 attempts — too aggressive
MAX_FAILED_ATTEMPTS = 2

# FLAGGED: Permanent lockout with no time-based unlock
if failed_attempts >= threshold:
    account.locked = True
    account.save()
    # No unlock mechanism, no expiry
```

```java
// FLAGGED: Threshold of 1
private static final int MAX_ATTEMPTS = 1;
```

## Remediation

Use progressive delays and time-based lockouts rather than permanent locks:

```python
# SAFE: Reasonable threshold with time-based unlock
MAX_FAILED_ATTEMPTS = 10  # NIST SP 800-63B recommends at least 100 attempts

# Exponential backoff instead of hard lockout
def check_lockout(user):
    if user.failed_attempts >= 10:
        lockout_duration = min(2 ** (user.failed_attempts - 10), 3600)  # max 1 hour
        if (datetime.utcnow() - user.last_failed).seconds < lockout_duration:
            return True  # still locked out
        else:
            user.failed_attempts = 0  # reset after lockout period
    return False
```

Consider CAPTCHA challenges after a moderate number of failures rather than hard lockouts, and alert the account owner on suspicious activity.
