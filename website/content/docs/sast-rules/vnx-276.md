## Overview

VNX-276 maps to [CWE-276: Incorrect Default Permissions](https://cwe.mitre.org/data/definitions/276.html). Detects source patterns associated with CWE-276 (Incorrect Default Permissions). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-276](https://cwe.mitre.org/data/definitions/276.html) | **Languages:** c, cpp, python

## Why This Matters

This weakness class (Incorrect Default Permissions) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-276 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'open(' pattern
open(
```

```python
// FLAGGED: contains '0o777' pattern
0o777
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-276 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-276` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
