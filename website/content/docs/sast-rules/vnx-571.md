---
title: "VNX-571 – Expression is Always True"
description: "Detect tautological conditions that always evaluate to true, potentially bypassing security gates."
---

## Overview

A condition that is always true (`if (true)`, `if (1 == 1)`) means the guarded block always executes. In access control contexts this means the guard never restricts — equivalent to removing the check entirely.

**Severity:** Low | **CWE:** [CWE-571 – Expression is Always True](https://cwe.mitre.org/data/definitions/571.html)

## Why This Matters

Tautological conditions in security gates grant unconditional access. This pattern is also used in SQL injection tautology attacks (`' OR '1'='1`) and may appear in code that was meant to be temporary.

## What Gets Flagged

```java
// FLAGGED: Always-true gate
if (true) {
    grantAccess(user);
}
```

```python
# FLAGGED: Tautological check
if 1 == 1:
    allow_request()
```

## Remediation

Replace with the actual security condition:

```java
// SAFE: Real authorization check
if (userHasPermission(user, "admin")) {
    grantAccess(user);
}
```
