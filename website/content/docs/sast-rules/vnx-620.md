---
title: "VNX-620 – Unverified Password Change"
description: "Detect password change operations that don't verify the current password, enabling account takeover."
---

## Overview

Allowing users to change their password without providing their current password means anyone with access to an authenticated session (through XSS, session hijacking, or physical access) can permanently lock out the legitimate account owner.

**Severity:** High | **CWE:** [CWE-620 – Unverified Password Change](https://cwe.mitre.org/data/definitions/620.html)

## Why This Matters

A brief XSS attack or session hijacking window is converted into a permanent account takeover if the attacker can change the password without knowing the current one. This is a critical account takeover vector in authentication systems.

## What Gets Flagged

```python
# FLAGGED: Password change without current password check
user.set_password(new_password)
user.save()

# FLAGGED: Django update without current password
User.objects.filter(id=user_id).update(password=new_password)
```

```javascript
// FLAGGED: setPassword without current password in context
user.setPassword(newPassword);
```

## Remediation

Always require and verify the current password:

```python
# SAFE: Verify current password first
from django.contrib.auth import authenticate

def change_password(request):
    current_password = request.POST.get('current_password')
    new_password = request.POST.get('new_password')

    user = authenticate(username=request.user.username, password=current_password)
    if user is None:
        return HttpResponseForbidden("Current password incorrect")

    user.set_password(new_password)
    user.save()
```
