---
title: "VNX-910 – Use of Expired File Descriptor"
description: "Detect file descriptors or handles used after they have been closed."
---

## Overview

After `close(fd)` or `fclose(fp)`, the descriptor is returned to the OS and may be immediately reused by the next `open()` call in any thread. Subsequent reads or writes on the closed descriptor operate on an unrelated file, potentially leaking data from or corrupting a sensitive file.

**Severity:** Medium | **CWE:** [CWE-910 – Use of Expired File Descriptor](https://cwe.mitre.org/data/definitions/910.html)

## Why This Matters

This is a time-of-check/time-of-use variant for file descriptors. In multi-threaded applications it is a race condition. In single-threaded code it manifests as use-after-close when the programmer closes a resource early and then accidentally uses it later.

## What Gets Flagged

```c
// FLAGGED: close followed by read within a few lines
close(fd);
// ... some code ...
read(fd, buf, len);  // fd is expired
```

## Remediation

Set descriptors to an invalid value after closing and check before use:

```c
// SAFE
close(fd);
fd = -1;
// any subsequent use: check fd != -1 first
```
