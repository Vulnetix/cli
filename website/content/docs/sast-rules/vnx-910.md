## Overview

VNX-910 maps to [CWE-910: Use of Expired File Descriptor](https://cwe.mitre.org/data/definitions/910.html). Detects source patterns associated with CWE-910 (Use of Expired File Descriptor). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-910](https://cwe.mitre.org/data/definitions/910.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Use of Expired File Descriptor) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-910 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'close(' pattern
close(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-910 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-910` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
