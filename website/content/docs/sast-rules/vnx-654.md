title: "VNX-654 – Reliance on a Single Factor in a Security Decision"
description: "Detect high-privilege operations (account deletion, fund transfer) protected by only a single authentication factor."
---

## Overview

High-risk operations such as deleting accounts, transferring funds, changing email addresses, or granting administrative privileges should require more than a single authentication factor. Relying on a session cookie alone (which may be compromised via XSS or session hijacking) for these irreversible operations violates the principle of defense in depth.

**Severity:** Low | **CWE:** [CWE-654 – Reliance on a Single Factor in a Security Decision](https://cwe.mitre.org/data/definitions/654.html)

## Why This Matters

A user whose session is compromised via XSS can have their account deleted, funds transferred, or sensitive data exfiltrated in a single request. Requiring re-authentication (password confirmation) or a second factor for critical operations adds a meaningful barrier: the attacker needs both the session cookie AND the user's password.

## What Gets Flagged

```python
# FLAGGED: Account deletion with only session check
def delete_account(request):
    if password == stored_password:
        request.user.delete()
        # Missing: no 2FA, no re-authentication for destructive action

# FLAGGED: Fund transfer with single factor
def transfer_funds(request):
    if password == user.password:
        transfer(amount, destination)
```

```javascript
// FLAGGED: Critical action with simple password check
if (password === user.password) {
    deleteAccount(userId);
}
```

## Remediation

Require re-authentication and consider step-up authentication (MFA) for sensitive operations:

```python
# SAFE: Password re-confirmation + TOTP for destructive operations
from django.contrib.auth import authenticate
import pyotp

def delete_account(request):
    password = request.POST.get('password')
    totp_code = request.POST.get('totp_code')
    
    # Re-authenticate with password
    user = authenticate(username=request.user.username, password=password)
    if not user:
        return HttpResponseForbidden("Incorrect password")
    
    # Verify TOTP second factor
    totp = pyotp.TOTP(user.totp_secret)
    if not totp.verify(totp_code):
        return HttpResponseForbidden("Invalid 2FA code")
    
    user.delete()
```

At minimum, require password re-confirmation; preferably add a second factor for irreversible operations.
