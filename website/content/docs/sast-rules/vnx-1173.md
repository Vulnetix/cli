title: "VNX-1173 – Improper Use of Validation Framework"
description: "Detect incorrect or incomplete use of validation frameworks: missing @Valid on Spring MVC parameters, Django validate=False bypass, and form data used without calling is_valid()."
---

## Overview

Validation frameworks provide powerful input-validation mechanisms, but misuse leaves applications exposed. Common failures include applying Bean Validation annotations without `@Valid`/`@Validated` on controller parameters, calling `form.save()` before `form.is_valid()` in Django, or passing `validate=False` to bypass Django's model validation entirely. **Severity:** Medium | **CWE:** [CWE-1173 – Improper Use of Validation Framework](https://cwe.mitre.org/data/definitions/1173.html)

## Why This Matters

A validation framework that is configured but not properly invoked provides no protection. Attackers can supply malformed or malicious data that passes unvalidated into business logic, database writes, or command execution. These bugs are particularly subtle because the validation annotations or form class appear to be present — the failure is in the invocation path.

## What Gets Flagged

```java
// FLAGGED: @RequestBody without @Valid — bean validation annotations not enforced
@PostMapping("/user")
public ResponseEntity<User> createUser(@RequestBody UserDto userDto) {
    // userDto.name could be blank/null even with @NotBlank annotation
    userService.create(userDto);
}
```

```python
# FLAGGED: validate=False bypasses Django model's full_clean()
instance = MyModel(**data)
instance.save(validate=False)
```

```python
# FLAGGED: accessing cleaned_data without calling is_valid()
form = UserForm(request.POST)
email = form.cleaned_data['email']  # KeyError or unvalidated data
```

## Remediation

```java
// SAFE: @Valid enforces constraints before method body executes
@PostMapping("/user")
public ResponseEntity<User> createUser(@Valid @RequestBody UserDto userDto) {
    userService.create(userDto);
}
```

```python
# SAFE: always call is_valid() before accessing cleaned_data
form = UserForm(request.POST)
if form.is_valid():
    email = form.cleaned_data['email']
    form.save()
else:
    return render(request, 'form.html', {'form': form})
```
