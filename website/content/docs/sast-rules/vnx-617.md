---
title: "VNX-617 – Reachable Assertion"
description: "Detect Python assert statements and Java assert keywords used for security validation that are disabled by optimization flags."
---

## Overview

Python's `assert` statement is completely removed when Python runs with the `-O` (optimize) flag. Java's `assert` keyword is disabled by default at runtime unless `-ea` is explicitly specified. Using assert for security validation creates checks that may be silently skipped in production.

**Severity:** Medium | **CWE:** [CWE-617 – Reachable Assertion](https://cwe.mitre.org/data/definitions/617.html)

## Why This Matters

A Docker image built with `python -O` or a JVM started without `-ea` will silently skip all `assert`-based security checks. The code looks secure in development but is unprotected in production. This has been the root cause of real authentication bypasses.

## What Gets Flagged

```python
# FLAGGED: Assert for authentication check
assert user.is_authenticated, "Must be logged in"

# FLAGGED: Assert for permission check
assert has_permission(user, 'admin'), "Admin required"
```

```java
// FLAGGED: Assert for security check
assert user.isAuthenticated() : "Must be authenticated";
```

## Remediation

Use explicit conditional checks with proper exceptions:

```python
# SAFE: Explicit check that cannot be optimized away
if not user.is_authenticated:
    raise PermissionError("Authentication required")

if not has_permission(user, 'admin'):
    raise PermissionError("Admin permission required")
```
