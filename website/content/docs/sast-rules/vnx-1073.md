## Overview

Detects functions that make multiple distinct data resource accesses (file reads, API calls, cache accesses) in a single invocation. Excessive data accesses in one function increases attack surface, complicates error handling, and can exhaust I/O resources.

**Severity:** Low | **CWE:** [CWE-1073](https://cwe.mitre.org/data/definitions/1073.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
