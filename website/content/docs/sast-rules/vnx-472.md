## Overview

VNX-472 maps to [CWE-472: External Control of Assumed-Immutable Web Parameter](https://cwe.mitre.org/data/definitions/472.html). Detects source patterns associated with CWE-472 (External Control of Assumed-Immutable Web Parameter). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-472](https://cwe.mitre.org/data/definitions/472.html) | **Languages:** node

## Why This Matters

This weakness class (External Control of Assumed-Immutable Web Parameter) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-472 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```javascript
// FLAGGED: contains '<input type="hidden' pattern
<input type="hidden
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-472 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-472` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
