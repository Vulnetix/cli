---
title: "VNX-1288 – Improper Validation of Consistency within Input"
description: "Detect missing cross-field consistency validation such as start/end date pairs, min/max value ranges, and from/to fields that must have a logical ordering relationship."
---

## Overview

Multi-field inputs often have logical consistency requirements: a start date must be before an end date, a minimum price must be less than the maximum price, a from-index must be less than a to-index. When these cross-field invariants are not validated, attackers can supply inverted ranges that cause empty result sets (hiding records), incorrect billing calculations, or logic bypass. **Severity:** Low | **CWE:** [CWE-1288 – Improper Validation of Consistency within Input](https://cwe.mitre.org/data/definitions/1288.html)

## Why This Matters

An inverted date range in a financial report query might return zero records, making audit trails appear empty. An inverted price range in a product filter might expose all products when the attacker provides `min_price=9999&max_price=0`. In some multi-step workflows, inconsistent inputs can skip mandatory processing steps.

## What Gets Flagged

```python
# FLAGGED: start_date/end_date used without ordering check
start_date = request.GET['start_date']
end_date = request.GET['end_date']
records = Record.objects.filter(date__range=(start_date, end_date))
# If start_date > end_date, Django returns empty queryset silently
```

```java
// FLAGGED: date range without comparison
LocalDate startDate = LocalDate.parse(request.getParameter("start"));
LocalDate endDate = LocalDate.parse(request.getParameter("end"));
List<Order> orders = orderRepo.findByDateBetween(startDate, endDate);
```

## Remediation

```python
# SAFE: Validate date ordering
from datetime import date

start_date = date.fromisoformat(request.GET['start_date'])
end_date = date.fromisoformat(request.GET['end_date'])

if start_date > end_date:
    return HttpResponseBadRequest("start_date must be before end_date")

records = Record.objects.filter(date__range=(start_date, end_date))
```

```java
// SAFE: Validate ordering before query
LocalDate startDate = LocalDate.parse(request.getParameter("start"));
LocalDate endDate = LocalDate.parse(request.getParameter("end"));

if (startDate.isAfter(endDate)) {
    throw new IllegalArgumentException("start must be before end");
}
```

```javascript
// SAFE: Cross-field validation with Joi or Zod
const schema = z.object({
    startDate: z.coerce.date(),
    endDate: z.coerce.date(),
}).refine(data => data.startDate <= data.endDate, {
    message: "startDate must be before endDate",
    path: ["endDate"],
});
```
