## Overview

Detects self-modifying code patterns such as eval(), exec(), dynamic code compilation, and runtime code generation. Self-modifying code is difficult to audit, can introduce code injection vulnerabilities, and bypasses static security analysis.

**Severity:** Low | **CWE:** [CWE-1123](https://cwe.mitre.org/data/definitions/1123.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1123](https://cwe.mitre.org/data/definitions/1123.html) for authoritative guidance.
