## Overview

Detects source files with TODO/FIXME markers indicating incomplete design decisions, especially in security-relevant components. Incomplete design documentation increases the risk that security assumptions are not understood or enforced.

**Severity:** Low | **CWE:** [CWE-1110](https://cwe.mitre.org/data/definitions/1110.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1110](https://cwe.mitre.org/data/definitions/1110.html) for authoritative guidance.
