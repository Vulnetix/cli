---
title: "VNX-1286 – Improper Validation of Syntactic Correctness of Input"
description: "Detect JSON, XML, YAML, and other structured inputs parsed without schema validation, which can allow malformed or injected structures to bypass business logic."
---

## Overview

Parsing structured input (JSON, XML, YAML) without validating it against a schema allows malformed payloads, unexpected extra fields, wrong types, and missing required fields to reach business logic unchecked. Attackers can supply minimal or malformed inputs that trigger edge cases, bypass authorization checks, or exploit parser-specific behaviours. **Severity:** Medium | **CWE:** [CWE-1286 – Improper Validation of Syntactic Correctness of Input](https://cwe.mitre.org/data/definitions/1286.html)

## Why This Matters

JSON parsers in different languages handle duplicate keys differently — some take the first, others the last — allowing parameter pollution attacks. XML parsers without explicit DTD/XSD validation are vulnerable to XXE. YAML `!!python/object` tags allow arbitrary object deserialization in Python's default loader. Schema validation catches all these at the boundary.

## What Gets Flagged

```python
# FLAGGED: No schema validation after parsing
data = json.loads(request.body)
user_id = data['user_id']  # could be missing, wrong type, or injected key

# FLAGGED: YAML without schema validation
config = yaml.load(raw_config)  # dangerous — allows object tags
```

```javascript
// FLAGGED: JSON.parse without validation or try/catch
const payload = JSON.parse(req.body);
processPayment(payload.amount, payload.currency);
```

```java
// FLAGGED: ObjectMapper without schema validation
User user = objectMapper.readValue(requestBody, User.class);
// Bean Validation annotations not enforced without @Valid
```

## Remediation

```python
# SAFE: Validate with pydantic or jsonschema
from pydantic import BaseModel, ValidationError

class PaymentRequest(BaseModel):
    amount: float
    currency: str
    recipient_id: int

try:
    data = PaymentRequest.model_validate_json(request.body)
except ValidationError as e:
    return HttpResponseBadRequest(str(e))
```

```javascript
// SAFE: Validate with Zod or Joi
const { z } = require('zod');
const schema = z.object({
    amount: z.number().positive(),
    currency: z.string().length(3),
});
const result = schema.safeParse(JSON.parse(req.body));
if (!result.success) return res.status(400).json(result.error);
```
