title: "VNX-837 – Improper Enforcement of a Single, Unique Action"
description: "Detect OTP/token validation without token invalidation, and payment processing without idempotency checks."
---

## Overview

This rule flags one-time token or OTP validation that does not include code to invalidate the token after use, and payment or order creation logic that does not include an idempotency check. Reusable one-time tokens allow attackers to replay authentication or password-reset flows. Non-idempotent payments can be charged multiple times by replaying requests. This maps to [CWE-837: Improper Enforcement of a Single, Unique Action](https://cwe.mitre.org/data/definitions/837.html).

**Severity:** High | **CWE:** [CWE-837](https://cwe.mitre.org/data/definitions/837.html)

## Why This Matters

A password reset link that remains valid after use allows an attacker to capture the link (from browser history, proxy logs, or email forwarding) and reset the password again days later. An OTP that can be replayed enables an attacker who observes one valid OTP submission to authenticate again using the same code. Non-idempotent payment APIs double-charge customers when network errors cause the client to retry.

## What Gets Flagged

```python
# FLAGGED: OTP validation without invalidation
def verify_otp(user_id, submitted_otp):
    token = OTPToken.objects.get(user_id=user_id)
    if token.value == submitted_otp:
        return True
    return False
    # Missing: token.delete() or token.mark_used()
```

```javascript
// FLAGGED: token comparison without deletion
async function verifyToken(token) {
    const record = await db.tokens.findOne({ value: token });
    if (record && record.expiresAt > Date.now()) {
        return true;  // token not deleted/invalidated
    }
    return false;
}
```

```python
# FLAGGED: payment processing without idempotency key check
def process_payment(amount, card_token):
    charge = stripe.Charge.create(
        amount=amount,
        currency="usd",
        source=card_token,
    )
    return charge
    # Missing: idempotency_key parameter
```

## Remediation

```python
# SAFE: invalidate OTP token after successful use
def verify_otp(user_id, submitted_otp):
    try:
        token = OTPToken.objects.get(
            user_id=user_id,
            value=submitted_otp,
            used=False,
            expires_at__gt=timezone.now()
        )
    except OTPToken.DoesNotExist:
        return False
    
    token.used = True
    token.save()
    return True
```

```javascript
// SAFE: delete token after use
async function verifyAndConsumeToken(token) {
    const record = await db.tokens.findOne({ value: token, used: false });
    if (!record || record.expiresAt <= Date.now()) {
        return false;
    }
    await db.tokens.deleteOne({ _id: record._id });  // invalidate immediately
    return true;
}
```

```python
# SAFE: use idempotency key for payments
import uuid

def process_payment(amount, card_token, idempotency_key=None):
    if idempotency_key is None:
        idempotency_key = str(uuid.uuid4())
    
    charge = stripe.Charge.create(
        amount=amount,
        currency="usd",
        source=card_token,
        idempotency_key=idempotency_key  # prevents duplicate charges on retry
    )
    return charge
```
