---
title: "VNX-170 – Improper Null Termination"
description: "Detect C string operations that may fail to null-terminate the destination buffer."
---

## Overview

This rule flags C functions that do not guarantee null termination: `strncpy()` does not null-terminate when the source is longer than the count; `strncat()` needs careful count management; `sprintf()` has no bounds check. An improperly terminated string causes undefined behavior when subsequently passed to string functions. This maps to [CWE-170: Improper Null Termination](https://cwe.mitre.org/data/definitions/170.html).

**Severity:** High | **CWE:** [CWE-170](https://cwe.mitre.org/data/definitions/170.html)

## What Gets Flagged

```c
// FLAGGED: strncpy without manual null termination
strncpy(dest, src, sizeof(dest));
// dest may not be null-terminated if src >= sizeof(dest)
```

## Remediation

```c
// SAFE: use strlcpy (always null-terminates) or add explicit null byte
strlcpy(dest, src, sizeof(dest));

// Or with strncpy:
strncpy(dest, src, sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';

// Or use snprintf:
snprintf(dest, sizeof(dest), "%s", src);
```
