description: "Detect security API calls where the return value is not checked or preconditions are not met."
---

## Overview

Security APIs have documented contracts (preconditions, return value semantics). Not checking the return value of `SSL_CTX_new()` or `RAND_bytes()` means a NULL or failure result is silently used, causing crashes or weak cryptography. This maps to [CWE-227: Improper Fulfillment of API Contract](https://cwe.mitre.org/data/definitions/227.html).

**Severity:** Low | **CWE:** [CWE-227](https://cwe.mitre.org/data/definitions/227.html)

## Remediation

Always check return values of security-critical API calls and abort or return an error if the call fails.
