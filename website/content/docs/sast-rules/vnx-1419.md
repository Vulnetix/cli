---
title: "VNX-1419 – Incorrect Initialization of Resource"
description: "Detect insecure TLS/SSL context initialization: deprecated protocol versions (SSLv3, TLS 1.0/1.1), disabled certificate verification, trust-all managers, and InsecureSkipVerify patterns."
---

## Overview

TLS/SSL contexts initialized with insecure defaults create encrypted connections that provide no real security. Disabling certificate verification (`verify=False`, `InsecureSkipVerify: true`, `rejectUnauthorized: false`) prevents verification of the server's identity, making the connection trivially MitM-able. Allowing SSLv3 or TLS 1.0/1.1 exposes connections to POODLE, BEAST, and SWEET32 attacks. **Severity:** High | **CWE:** [CWE-1419 – Incorrect Initialization of Resource](https://cwe.mitre.org/data/definitions/1419.html)

## Why This Matters

The `verify=False` anti-pattern is extremely common in Python code — developers add it when they encounter a certificate error and never remove it. In production this creates an illusion of security: data is encrypted but can be read by any attacker on the path who presents any certificate. This pattern was present in the infamous 2014 Heartbleed response scripts.

## What Gets Flagged

```python
# FLAGGED: Disabling SSL verification
import requests
response = requests.get(url, verify=False)

# FLAGGED: Insecure SSL context
import ssl
ctx = ssl.create_default_context()
ctx.check_hostname = False
ctx.verify_mode = ssl.CERT_NONE

# FLAGGED: Deprecated protocol
ctx = ssl.SSLContext(ssl.PROTOCOL_TLSv1)
```

```java
// FLAGGED: Trust-all X509TrustManager
new X509TrustManager() {
    public void checkServerTrusted(X509Certificate[] certs, String authType) {}
    // Empty method — trusts everything
};

// FLAGGED: ALLOW_ALL_HOSTNAME_VERIFIER
connection.setHostnameVerifier(SSLSocketFactory.ALLOW_ALL_HOSTNAME_VERIFIER);
```

```go
// FLAGGED: InsecureSkipVerify
tlsConfig := &tls.Config{InsecureSkipVerify: true}
```

```javascript
// FLAGGED: rejectUnauthorized: false
const https = require('https');
https.request({ hostname: 'example.com', rejectUnauthorized: false }, cb);
```

## Remediation

```python
# SAFE: Always verify certificates
import requests
response = requests.get(url)  # verify=True is the default

# SAFE: Secure SSL context
ctx = ssl.create_default_context()
# Do not set check_hostname or verify_mode — defaults are secure
```

```go
// SAFE: TLS with proper certificate verification
tlsConfig := &tls.Config{
    MinVersion: tls.VersionTLS12,
    // InsecureSkipVerify defaults to false — do not set it
}
```

```javascript
// SAFE: Use default (rejectUnauthorized defaults to true)
https.request({ hostname: 'example.com' }, cb);
```
