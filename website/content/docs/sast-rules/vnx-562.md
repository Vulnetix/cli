---
title: "VNX-562 – Return of Stack Variable Address"
description: "Detect C/C++ functions that return a pointer to a local (stack-allocated) variable."
---

## Overview

Returning the address of a local variable (`return &x;`) or a local array (`char buf[100]; return buf;`) produces a dangling pointer. After the function returns, the stack frame is deallocated and the memory can be overwritten by subsequent function calls, leading to undefined behavior and potential exploitation.

**Severity:** High | **CWE:** [CWE-562 – Return of Stack Variable Address](https://cwe.mitre.org/data/definitions/562.html)

## Why This Matters

Using a dangling pointer is undefined behavior in C/C++. In practice it leads to reading stale data, writing to memory now owned by another stack frame, and crashes. Controlled stack grooming can turn these into exploitable vulnerabilities.

## What Gets Flagged

```c
// FLAGGED: Returning address of local variable
int* getCount() {
    int count = 42;
    return &count;  // count is gone after return
}

// FLAGGED: Returning local array
char* getBuffer() {
    char buf[100];
    snprintf(buf, sizeof(buf), "data");
    return buf;  // buf is on the stack
}
```

## Remediation

Use heap allocation or caller-provided buffers:

```c
// SAFE: Caller provides buffer
void getBuffer(char* buf, size_t len) {
    snprintf(buf, len, "data");
}

// SAFE: Heap allocation (caller must free)
char* getBuffer() {
    char* buf = malloc(100);
    if (buf) snprintf(buf, 100, "data");
    return buf;
}
```
