---

## Overview

This rule flags type casts performed without prior type validation. In Java, casting an object to a type it doesn't implement throws `ClassCastException` at runtime. In C/C++, casting between incompatible pointer types violates the strict aliasing rule and causes undefined behaviour including memory corruption. This maps to [CWE-704: Incorrect Type Conversion or Cast](https://cwe.mitre.org/data/definitions/704.html).

**Severity:** Medium | **CWE:** [CWE-704](https://cwe.mitre.org/data/definitions/704.html)

## Why This Matters

ClassCastExceptions in Java can terminate request handlers, potentially causing denial of service or bypassing security checks in error-handling paths. In C, casting between incompatible pointer types (e.g., `(int*)charptr`) triggers undefined behaviour that compilers may exploit during optimisation.

## What Gets Flagged

```java
// FLAGGED: cast without instanceof check
Object obj = getFromCache("user");
String name = (String) obj;  // throws ClassCastException if not a String
```

```c
// FLAGGED: incompatible pointer cast
char *buf = get_buffer();
int *values = (int*) buf;  // undefined behaviour if buf is not int-aligned
```

## Remediation

```java
// SAFE: check type before casting
if (obj instanceof String) {
    String name = (String) obj;
}
// Java 16+: pattern matching instanceof
if (obj instanceof String name) { ... }
```

```c
// SAFE: use memcpy for type punning
int result;
memcpy(&result, buf, sizeof(int));
```
