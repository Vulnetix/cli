## Overview

Acquiring multiple locks in inconsistent orders, or failing to release a lock, can cause threads to wait indefinitely for each other. Deadlocks halt program execution and are often exploitable as denial-of-service vectors.

**Severity:** Medium | **CWE:** [CWE-833 – Deadlock](https://cwe.mitre.org/data/definitions/833.html)

## Why This Matters

Violations of CWE-833 can expose the application to exploitation. This rule detects source code patterns associated with this weakness across multiple languages and frameworks.

## What Gets Flagged

This rule fires when source code contains patterns indicative of CWE-833. Review the flagged code to confirm whether the identified pattern represents a genuine vulnerability in context.

## Remediation

1. Review the flagged code and understand the specific weakness described in CWE-833.
2. Apply the recommended remediation for your language/framework.
3. Add appropriate input validation, access controls, or safe API usage as applicable.
4. Add a test case that verifies the fix.
