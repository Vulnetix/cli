## Overview

Detects loop counter or condition variables that are modified inside the loop body in ways other than the expected increment. Unexpected loop variable modification can cause infinite loops, skip security iterations, or process items out of expected bounds.

**Severity:** Low | **CWE:** [CWE-1095](https://cwe.mitre.org/data/definitions/1095.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1095](https://cwe.mitre.org/data/definitions/1095.html) for authoritative guidance.
