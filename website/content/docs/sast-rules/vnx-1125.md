## Overview

Detects patterns that unnecessarily expand the attack surface, including wildcard CORS origins, permissive Content Security Policy directives, and open API configurations. Each unnecessary exposure increases the risk of exploitation.

**Severity:** Medium | **CWE:** [CWE-1125 – Excessive Attack Surface](https://cwe.mitre.org/data/definitions/1125.html)

## Why This Matters

`Access-Control-Allow-Origin: *` allows any website to make cross-origin requests to your API with the user's credentials (if using non-simple requests with `withCredentials`). `unsafe-inline` in CSP allows XSS payloads to execute even with CSP enabled.

## What Gets Flagged

```go
// FLAGGED: wildcard CORS
w.Header().Set("Access-Control-Allow-Origin", "*")
```

```python
# FLAGGED: Django CORS
CORS_ALLOW_ALL_ORIGINS = True
```

```javascript
// FLAGGED: permissive CSP
res.setHeader('Content-Security-Policy', "script-src 'unsafe-inline'");
```

## Remediation

```go
// SAFE: restrict to known origins
allowedOrigins := map[string]bool{
    "https://app.example.com": true,
}
origin := r.Header.Get("Origin")
if allowedOrigins[origin] {
    w.Header().Set("Access-Control-Allow-Origin", origin)
}
```
