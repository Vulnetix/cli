---
title: "VNX-653 – Insufficient Compartmentalization"
description: "Detect global authentication and authorization state that is shared across contexts, enabling state confusion attacks."
---

## Overview

Using global variables to track authentication or authorization state (e.g., `global is_authenticated`, `global current_user_role`) means that all concurrent requests share this state. A race condition or logic error can cause one user's authentication to bleed into another request, or a single authenticated request to set a global flag that persists across all subsequent requests.

**Severity:** Low | **CWE:** [CWE-653 – Insufficient Compartmentalization](https://cwe.mitre.org/data/definitions/653.html)

## Why This Matters

In Python web frameworks like Flask with threaded or async servers, a `global is_admin = True` set during one request can remain set for subsequent requests if the reset path is missed (e.g., due to an exception). Multi-threaded environments have race conditions: Thread A sets `is_authenticated = True`, Thread B reads it before Thread A resets it, and Thread B processes its request as if authenticated.

## What Gets Flagged

```python
# FLAGGED: Global authentication state
global is_authenticated
is_authenticated = True

# FLAGGED: Global admin flag
global is_admin
is_admin = check_admin(user)

# FLAGGED: Global role — shared across requests
global current_role
current_role = 'admin'
```

```javascript
// FLAGGED: Module-level auth state
let isAuthenticated = false;
// ... later in a request handler:
isAuthenticated = true;
```

## Remediation

Use request-scoped or thread-local storage for authentication and authorization state:

```python
# SAFE: Flask's request context (request-scoped)
from flask import g, request

@app.before_request
def load_user():
    g.user = get_user_from_token(request.headers.get('Authorization'))
    g.is_admin = g.user.role == 'admin' if g.user else False

# Access via g.is_admin — scoped to current request
```

```python
# SAFE: Thread-local storage
import threading
_local = threading.local()

def set_current_user(user):
    _local.current_user = user

def get_current_user():
    return getattr(_local, 'current_user', None)
```
