---
title: "VNX-222 – Truncation of Security-relevant Information"
description: "Detect log statements that truncate usernames, tokens, or IP addresses before writing to audit logs."
---

## Overview

Truncating the username, token, or IP address in a security log entry hides the full context of the event. A 32-character username sliced to 20 characters may not uniquely identify the attacker. Short substrings of tokens prevent correlation with other events. This maps to [CWE-222: Truncation of Security-relevant Information](https://cwe.mitre.org/data/definitions/222.html).

**Severity:** Low | **CWE:** [CWE-222](https://cwe.mitre.org/data/definitions/222.html)

## Remediation

Log the complete value for security events, or use a structured logging format. If output length is a concern, hash long values (e.g., SHA-256 of the token) rather than truncating.
