## Overview

VNX-825 maps to [CWE-825: Expired Pointer Dereference](https://cwe.mitre.org/data/definitions/825.html). Detects source patterns associated with CWE-825 (Expired Pointer Dereference). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-825](https://cwe.mitre.org/data/definitions/825.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Expired Pointer Dereference) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-825 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'return &' pattern
return &
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-825 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-825` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
