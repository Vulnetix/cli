---
title: "VNX-825 – Expired Pointer Dereference"
description: "Detect use-after-free patterns where a pointer is dereferenced after being freed, or a dangling pointer to a local variable is returned."
---

## Overview

This rule flags two related expired pointer patterns: (1) a pointer that is `free()`d and then dereferenced within 10 lines without being set to `NULL`, and (2) returning the address of a local (stack-allocated) variable, which becomes invalid when the function returns. Both are undefined behavior that attackers exploit for heap-based arbitrary code execution or stack data disclosure. This maps to [CWE-825: Expired Pointer Dereference](https://cwe.mitre.org/data/definitions/825.html).

**Severity:** Critical | **CWE:** [CWE-825](https://cwe.mitre.org/data/definitions/825.html)

## Why This Matters

Use-after-free (UAF) vulnerabilities are consistently among the most exploited memory safety issues. After `free(ptr)`, the heap allocator can reuse that memory for another allocation. If an attacker can control what gets allocated in that space, they can place a crafted object there and control program execution when the original code dereferences `ptr`. Returning a pointer to a local variable produces a dangling pointer that points to overwritten stack space.

## What Gets Flagged

```c
// FLAGGED: dereferencing freed pointer
free(node);
printf("%d\n", node->value);  // use-after-free

// FLAGGED: calling function on freed pointer
free(buf);
process(buf);  // buf is freed, but passed to function

// FLAGGED: returning address of local variable
int *get_value() {
    int x = 42;
    return &x;  // x is destroyed when function returns
}

// FLAGGED: returning pointer to local array
char *get_name() {
    char name[64] = "test";
    return name;  // dangling pointer after return
}
```

## Remediation

```c
// SAFE: set pointer to NULL immediately after free
free(node);
node = NULL;
// Now attempting to dereference node will crash cleanly (SIGSEGV)
// rather than exhibiting undefined behavior

// SAFE: check before use, NULL after free
free(buf);
buf = NULL;
// ... later
if (buf != NULL) {
    process(buf);
}
```

```c
// SAFE: return heap-allocated value instead of stack address
int *get_value() {
    int *x = malloc(sizeof(int));
    if (x != NULL) *x = 42;
    return x;  // caller must free
}

// SAFE: return static variable (be careful of thread safety)
const char *get_name() {
    static char name[64] = "test";
    return name;
}

// SAFE: pass caller-provided buffer instead
void get_name(char *buf, size_t size) {
    strncpy(buf, "test", size - 1);
    buf[size - 1] = '\0';
}
```

Adopt the pattern of setting all pointers to `NULL` immediately after `free()`. Enable AddressSanitizer (`-fsanitize=address`) during development and testing to detect UAF at runtime.
