## Overview

Detects Unicode homoglyph characters (visually similar to ASCII but different code points) and bidirectional control characters. This class of attack, known as Trojan Source, allows attackers to create code that appears safe to reviewers but executes differently.

**Severity:** Medium | **CWE:** [CWE-1007 – Insufficient Visual Distinction of Homoglyphs](https://cwe.mitre.org/data/definitions/1007.html)

## Why This Matters

The 2021 Trojan Source research (CVE-2021-42574) demonstrated that Unicode bidirectional control characters can cause compilers and interpreters to execute code in a completely different order than what appears in a text editor. Homoglyph substitutions (e.g., Cyrillic `а` replacing Latin `a`) create visually identical but semantically different identifiers.

## What Gets Flagged

- Cyrillic characters that look like Latin letters (U+0430 `а`, U+0435 `е`, U+043E `о`)
- Zero-width characters (U+200B, U+200C, U+200D)
- Bidirectional control characters (U+202A–U+202E, U+2066–U+2069)
- BOM character (U+FEFF) in unexpected positions

## Remediation

1. Configure your editor and CI pipeline to flag non-ASCII characters in source files.
2. Add a pre-commit hook to detect these characters.
3. Review all flagged files carefully — legitimate i18n files may contain these characters in string literals.
