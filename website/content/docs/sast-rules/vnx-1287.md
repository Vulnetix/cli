title: "VNX-1287 – Improper Validation of Specified Type of Input"
description: "Detect input used without type validation — loose equality in JavaScript, missing isinstance() in Python, no is_numeric() in PHP — that allows type-coercion bypasses."
---

## Overview

Type confusion occurs when input is processed assuming a specific type without verifying it. JavaScript's loose equality `==` coerces types, allowing `'0' == false` and `[] == false`. PHP's type juggling notoriously causes `"0e12345" == "0e999"` (both evaluate to 0.0 in scientific notation). Python's duck typing means a string used where an int is expected may silently succeed in some operations. **Severity:** Medium | **CWE:** [CWE-1287 – Improper Validation of Specified Type of Input](https://cwe.mitre.org/data/definitions/1287.html)

## Why This Matters

PHP's type juggling has been exploited in authentication bypasses: `"00e00" == 0` in early PHP allowed bypassing MD5 hash comparisons. JavaScript `== null` matches both `null` and `undefined`. Type confusion in authentication or authorization logic can allow attackers to pass unexpected types that evaluate to truthy/falsy in ways that bypass security checks.

## What Gets Flagged

```javascript
// FLAGGED: Loose equality with request data
if (req.body.userId == currentUser.id) {  // type coercion risk
    grantAccess();
}

// FLAGGED: No typeof check before use
const age = req.query.age;
if (age < 18) { reject(); }  // string comparison '17' < 18 is numeric, but '9' < 18 is false
```

```python
# FLAGGED: Request data used without type check
def view(request):
    value = request.GET.get('limit')  # returns str or None
    items = get_items(value)  # if function expects int, may fail silently
```

```php
// FLAGGED: Superglobal without type validation
$age = $_GET['age'];
if ($age >= 18) { allow(); }  // string comparison, no is_numeric check
```

## Remediation

```javascript
// SAFE: Strict equality and typeof validation
if (typeof req.body.userId === 'number' && req.body.userId === currentUser.id) {
    grantAccess();
}
// Or use a validation library
const schema = z.object({ userId: z.number().int() });
```

```python
# SAFE: Explicit type cast and validation
try:
    limit = int(request.GET.get('limit', 10))
except (TypeError, ValueError):
    return HttpResponseBadRequest("limit must be a number")
```

```php
// SAFE: Type validation with filter_var
$age = filter_input(INPUT_GET, 'age', FILTER_VALIDATE_INT);
if ($age === false || $age === null || $age < 0) {
    http_response_code(400);
    exit('Invalid age');
}
```
