title: "VNX-547 – Use of Hard-coded Security-relevant Constants"
description: "Detect magic numbers used as security parameters such as token lengths, session timeouts, and cryptographic key sizes."
---

## Overview

Hard-coding security parameters (token length = 4, expiry = 31536000 seconds, key size = 128 bits) makes them difficult to change without code modification and creates risk that insecure defaults persist in production. CWE-547 covers security-relevant constants that should be configurable named values.

**Severity:** Medium | **CWE:** [CWE-547 – Use of Hard-coded, Security-relevant Constants](https://cwe.mitre.org/data/definitions/547.html)

## Why This Matters

A 4-digit OTP token has only 10,000 combinations and can be brute-forced in seconds. A session that expires in one year is effectively permanent. Hard-coded values prevent easy remediation when security requirements evolve.

## What Gets Flagged

```python
# FLAGGED: Token length too short and hardcoded
token_length = 4
otp_length = 4

# FLAGGED: One-year session expiry
expiry = 86400 * 365
SESSION_COOKIE_AGE = 31536000
```

## Remediation

Use named constants with documented security rationale:

```python
# SAFE: Meaningful named constant with appropriate value
OTP_LENGTH = 6           # TOTP standard is 6 digits
TOKEN_LENGTH = 32        # 256 bits of entropy
SESSION_TIMEOUT = 3600   # 1 hour inactivity timeout
```

Reference NIST SP 800-63B for OTP length requirements (minimum 6 digits).
