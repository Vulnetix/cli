## Overview

VNX-204 maps to [CWE-204: Observable Response Discrepancy](https://cwe.mitre.org/data/definitions/204.html). Detects source patterns associated with CWE-204 (Observable Response Discrepancy). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-204](https://cwe.mitre.org/data/definitions/204.html) | **Languages:** go, java, node, python

## Why This Matters

This weakness class (Observable Response Discrepancy) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-204 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'if user is None' pattern
if user is None
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-204 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-204` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
