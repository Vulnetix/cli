title: "VNX-708 – Incorrect Ownership Assignment"
description: "Detect file ownership changes that may assign resources to incorrect owners, granting unintended elevated privileges."
---

## Overview

This rule flags calls to `chown`, `os.chown`, and `os.Chown` that change file ownership. Assigning file ownership to root (UID 0) or another privileged user without verification can grant attackers elevated access to sensitive files. This maps to [CWE-708: Incorrect Ownership Assignment](https://cwe.mitre.org/data/definitions/708.html).

**Severity:** Low | **CWE:** [CWE-708](https://cwe.mitre.org/data/definitions/708.html)

## Why This Matters

Files owned by root or privileged UIDs may be executed with elevated privileges (setuid), read by processes running as those users, or bypass access control checks. Incorrect ownership assignment from user-controlled paths is a common privilege escalation vector.

## What Gets Flagged

```c
// FLAGGED: chown to UID 0 (root)
chown(user_path, 0, 0);
```

```python
# FLAGGED: chown with potentially wrong UID
os.chown(path, uid_from_request, gid_from_request)
```

```go
// FLAGGED: Chown with potentially wrong UID
os.Chown(path, 0, 0)
```

## Remediation

- Verify that the target UID/GID is the correct owner for the resource.
- Never use UIDs/GIDs derived from user input.
- Prefer the least-privilege UID that provides necessary access.
- After chown, verify the operation succeeded and the result is expected.
