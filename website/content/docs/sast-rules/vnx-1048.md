## Overview

Functions with high cyclomatic complexity (many `if`, `for`, `while`, `catch`, `||`, `&&` branches) are difficult to reason about, test, and audit for security properties. Security-critical functions — authentication handlers, authorisation checks, input validators — with high complexity frequently contain untested code paths that bypass intended controls.

**Severity:** Low | **CWE:** [CWE-1048 – Invokable Control Element with Large Number of Outgoing Control Flow Transfers](https://cwe.mitre.org/data/definitions/1048.html)

## Why This Matters

Security testing coverage drops sharply as complexity rises. An untested branch in an authentication function may skip a required check. Code review becomes impractical at high complexity.

## What Gets Flagged

Functions with approximately 15 or more control flow branches in their body are flagged as potentially too complex for reliable security auditing.

## Remediation

Refactor complex functions into smaller, single-purpose functions. Extract validation logic into dedicated validators, and extract authorization logic into decorators or middleware.
