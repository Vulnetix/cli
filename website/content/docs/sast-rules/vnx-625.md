## Overview

VNX-625 maps to [CWE-625: Permissive Regular Expression](https://cwe.mitre.org/data/definitions/625.html). Detects source patterns associated with CWE-625 (Permissive Regular Expression). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-625](https://cwe.mitre.org/data/definitions/625.html) | **Languages:** python

## Why This Matters

This weakness class (Permissive Regular Expression) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-625 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains '.*' pattern
.*
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-625 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-625` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
