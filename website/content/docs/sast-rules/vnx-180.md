---
title: "VNX-180 – Validate Before Canonicalize"
description: "Detect security checks applied before URL decoding or path normalization."
---

## Overview

Security validation must occur after canonicalization. If a traversal check runs on the raw URL before `decodeURIComponent()`, an attacker supplies `%2e%2e%2f` (URL-encoded `../`) which bypasses the check but decodes to a traversal after the fact. This maps to [CWE-180](https://cwe.mitre.org/data/definitions/180.html).

**Severity:** Medium | **CWE:** [CWE-180](https://cwe.mitre.org/data/definitions/180.html)

## Remediation

Always decode/normalize input before applying security validation. Treat the normalized form as the canonical value for all downstream checks.
