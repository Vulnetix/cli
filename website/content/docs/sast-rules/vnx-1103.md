## Why This Matters

This weakness class (Use of Platform-Dependent Third Party Components) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1103 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'sun.misc' pattern
sun.misc
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1103 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1103` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
