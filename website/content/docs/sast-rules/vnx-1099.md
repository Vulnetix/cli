## Overview

Detects inconsistent naming conventions for security-sensitive identifiers such as mixing camelCase and snake_case for authentication/authorisation variables in the same file. Inconsistent naming makes security auditing harder and increases the risk of using the wrong variable.

**Severity:** Low | **CWE:** [CWE-1099](https://cwe.mitre.org/data/definitions/1099.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1099](https://cwe.mitre.org/data/definitions/1099.html) for authoritative guidance.
