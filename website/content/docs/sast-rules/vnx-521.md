## Overview

VNX-521 maps to [CWE-521: Weak Password Requirements](https://cwe.mitre.org/data/definitions/521.html). Detects source patterns associated with CWE-521 (Weak Password Requirements). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-521](https://cwe.mitre.org/data/definitions/521.html) | **Languages:** java, node, python

## Why This Matters

This weakness class (Weak Password Requirements) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-521 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'minLength: 1' pattern
minLength: 1
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-521 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-521` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
