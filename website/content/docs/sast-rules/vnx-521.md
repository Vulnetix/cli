---
title: "VNX-521 – Weak Password Requirements"
description: "Detect missing or trivially weak password validation that fails to enforce minimum strength requirements."
---

## Overview

Applications that do not enforce minimum password strength allow users to set trivially weak passwords that can be guessed or brute-forced. CWE-521 covers missing validators, empty validator lists, and trivial checks that only verify non-empty input.

**Severity:** Medium | **CWE:** [CWE-521 – Weak Password Requirements](https://cwe.mitre.org/data/definitions/521.html)

## Why This Matters

Weak passwords are the primary vector for credential-stuffing, brute-force, and account-takeover attacks. A password of "a" satisfies a check of `strlen($password) > 0` but can be cracked instantly. NIST SP 800-63B requires a minimum of 8 characters and recommends checking against known-breached password lists.

## What Gets Flagged

```python
# FLAGGED: Django with no password validators
AUTH_PASSWORD_VALIDATORS = []
```

```php
// FLAGGED: Only checking non-empty
if (strlen($password) > 0) {
    // accept password
}
```

```javascript
// FLAGGED: Trivial JS check
if (password.length > 0) { /* accept */ }
```

## Remediation

Enforce meaningful password policies:

```python
# SAFE: Django with validators
AUTH_PASSWORD_VALIDATORS = [
    {"NAME": "django.contrib.auth.password_validation.MinimumLengthValidator",
     "OPTIONS": {"min_length": 12}},
    {"NAME": "django.contrib.auth.password_validation.CommonPasswordValidator"},
    {"NAME": "django.contrib.auth.password_validation.NumericPasswordValidator"},
]
```

```php
// SAFE: minimum length + complexity
if (strlen($password) >= 12 && preg_match('/[A-Z]/', $password) && preg_match('/[0-9]/', $password)) {
    // accept
}
```
