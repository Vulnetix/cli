## Overview

VNX-409 maps to [CWE-409: Improper Handling of Highly Compressed Data (Data Amplification)](https://cwe.mitre.org/data/definitions/409.html). **Severity:** Medium | **CWE:** [CWE-409](https://cwe.mitre.org/data/definitions/409.html) | **Languages:** node, python

## Why This Matters

This weakness class (Improper Handling of Highly Compressed Data (Data Amplification)) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-409 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'zipfile.ZipFile' pattern

```

```javascript
// FLAGGED: contains 'zlib.gunzip' pattern

```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-409 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-409` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
