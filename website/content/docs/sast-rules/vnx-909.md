---
title: "VNX-909 – Missing Initialization of Resource"
description: "Detect variables and class attributes declared without initialization that may hold undefined or garbage values."
---

## Overview

A resource variable declared without initialization holds an undefined value (garbage data in C/C++, null in Java, or None in Python). When these uninitialized values reach security-sensitive operations — credentials checks, buffer operations, or pointer dereferences — the results are unpredictable and potentially exploitable.

**Severity:** Medium | **CWE:** [CWE-909 – Missing Initialization of Resource](https://cwe.mitre.org/data/definitions/909.html)

## Why This Matters

In C, an uninitialized pointer may point to arbitrary memory. In Java, an uninitialized field causes `NullPointerException`. In Python, accessing an attribute not set in `__init__` raises `AttributeError` at an unpredictable point during execution.

## What Gets Flagged

```c
// FLAGGED: Uninitialized pointer
char *buf;  // no = NULL
FILE *fp;   // no = NULL
```

```java
// FLAGGED: Uninitialized field
private Connection connection;  // should be = null
```

## Remediation

Initialize all variables at declaration:

```c
// SAFE
char *buf = NULL;
FILE *fp = NULL;
```

```java
// SAFE
private Connection connection = null;
```

```python
# SAFE: always initialize in __init__
def __init__(self):
    self.connection = None
    self.token = None
```
