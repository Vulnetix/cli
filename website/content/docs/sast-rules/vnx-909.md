## Overview

VNX-909 maps to [CWE-909: Missing Initialization of Resource](https://cwe.mitre.org/data/definitions/909.html). Detects source patterns associated with CWE-909 (Missing Initialization of Resource). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-909](https://cwe.mitre.org/data/definitions/909.html) | **Languages:** java

## Why This Matters

This weakness class (Missing Initialization of Resource) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-909 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'private int' pattern
private int
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-909 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-909` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
