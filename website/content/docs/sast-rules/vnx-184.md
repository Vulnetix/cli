---
title: "VNX-184 – Incomplete Denylist"
description: "Detect blocklist-based HTML sanitization that only filters specific known-bad strings, missing variants."
---

## Overview

Blocklist sanitization that removes only `<script>` (lowercase) still allows `<SCRIPT>`, `<Script>`, event handlers like `onerror`, and `javascript:` URI schemes. An attacker chooses a variant not in the blocklist. This maps to [CWE-184: Incomplete List of Disallowed Inputs](https://cwe.mitre.org/data/definitions/184.html).

**Severity:** Medium | **CWE:** [CWE-184](https://cwe.mitre.org/data/definitions/184.html)

## Remediation

Replace denylist sanitization with allowlist-based HTML parsers:

```python
# SAFE: bleach with explicit allowlist
import bleach
clean = bleach.clean(user_input, tags=['b', 'i', 'em', 'strong'], attributes={})
```

```php
<?php
// SAFE: HTMLPurifier
$config = HTMLPurifier_Config::createDefault();
$purifier = new HTMLPurifier($config);
$clean = $purifier->purify($_POST['content']);
```
