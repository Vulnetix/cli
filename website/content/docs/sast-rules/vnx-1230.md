## Overview

Files carry metadata beyond their visible content. EXIF data in JPEG images can contain GPS coordinates, device serial numbers, and timestamps. Accessible `.git` directories expose full source history including deleted secrets. PDF and Office documents embed author names, company names, revision history, and internal directory paths. **Severity:** Medium | **CWE:** [CWE-1230 – Exposure of Sensitive Information Through Metadata](https://cwe.mitre.org/data/definitions/1230.html)

## Why This Matters

In 2010, a UN document's metadata revealed the author's name, undercutting anonymity. Multiple data breaches have begun with source code recovered from accessible `.git` directories. EXIF GPS data in photos shared by journalists or activists has compromised their locations. User-uploaded images served without EXIF stripping expose device and location data.

## What Gets Flagged

```python
# FLAGGED: Reading EXIF data without stripping before serving
import exifread
tags = exifread.process_file(uploaded_image)
# If GPS data is in tags, it should be stripped before storing/serving

# FLAGGED: PDF metadata with sensitive fields
from reportlab.lib.pagesizes import A4
from reportlab.pdfgen import canvas
c = canvas.Canvas("output.pdf", pagesize=A4)
# /Author and /Creator fields should be cleared or anonymised
```

```nginx

location /.git/ {
    # Missing: deny all;
}
```

## Remediation

```python
# SAFE: Strip EXIF data before storing user-uploaded images
from PIL import Image
import io

def strip_exif(image_data):
    img = Image.open(io.BytesIO(image_data))
    # Create new image without metadata
    output = io.BytesIO()
    # Copy only pixel data, not metadata
    clean = Image.new(img.mode, img.size)
    clean.putdata(list(img.getdata()))
    clean.save(output, format=img.format)
    return output.getvalue()
```

```nginx
# SAFE: Deny .git directory access
location ~ /\.git {
    deny all;
    return 404;
}
```
