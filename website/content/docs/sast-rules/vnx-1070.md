## Overview

Detects Java Serializable classes that contain fields of types that are commonly non-serializable (threads, sockets, streams, loggers). Serializing objects with non-serializable fields causes runtime NotSerializableException, can leak internal state, and may expose sensitive objects during deserialization attacks.

**Severity:** Low | **CWE:** [CWE-1070](https://cwe.mitre.org/data/definitions/1070.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.
