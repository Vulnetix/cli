---
title: "VNX-643 – XPath Injection"
description: "Detect user-controlled data concatenated into XPath expressions, enabling attackers to manipulate XML document queries."
---

## Overview

XPath injection occurs when user-supplied data is concatenated into an XPath expression without sanitisation. Unlike SQL injection, there is no native parameterisation in XPath 1.0, making string concatenation especially dangerous. An attacker can break out of the intended query, enumerate the full XML document, or bypass authentication.

**Severity:** High | **CWE:** [CWE-643 – Improper Neutralization of Data within XPath Expressions (XPath Injection)](https://cwe.mitre.org/data/definitions/643.html)

## Why This Matters

Authentication systems that look up users in XML files are particularly vulnerable. An attacker can inject `' or '1'='1` to match any node, bypassing the password check entirely.

## What Gets Flagged

```java
// FLAGGED: Java XPath with string concatenation
xpath.evaluate("//user[name='" + userInput + "']", doc, XPathConstants.NODE);
```

```python
# FLAGGED: Python lxml with f-string
result = tree.xpath(f"//user[@name='{user}']")
```

## Remediation

Use variable binding or allow-listed values instead of concatenation:

```python
# SAFE: lxml variable substitution
result = tree.xpath("//user[@name=$name]", name=user)
```

```java
// SAFE: XPathVariableResolver for parameterised XPath
```
