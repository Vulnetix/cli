---
title: "VNX-1021 – Improper Restriction of Rendered UI Layers (Clickjacking)"
description: "Detect missing or disabled X-Frame-Options and frame-ancestors CSP headers that allow clickjacking attacks."
---

## Overview

When a web application can be embedded in an iframe by arbitrary sites, attackers can overlay a transparent frame over a legitimate UI element and trick users into clicking buttons they cannot see — a clickjacking attack. Setting `X-Frame-Options: DENY` or `Content-Security-Policy: frame-ancestors 'none'` prevents this.

**Severity:** Medium | **CWE:** [CWE-1021 – Improper Restriction of Rendered UI Layers or Frames](https://cwe.mitre.org/data/definitions/1021.html)

## Why This Matters

Clickjacking has been used to steal social media likes, trigger fund transfers, enable camera/microphone access, and bypass CSRF protections. It is invisible to the victim and works even on well-hardened applications.

## What Gets Flagged

```javascript
// FLAGGED: frameguard disabled
app.use(helmet({ frameguard: false }));
```

```python
# FLAGGED: Django X_FRAME_OPTIONS weakened
X_FRAME_OPTIONS = 'ALLOWALL'

# FLAGGED: View exempted from frame protection
@xframe_options_exempt
def sensitive_view(request): ...
```

```java
// FLAGGED: Spring Security frameOptions disabled
http.headers().frameOptions().disable();
```

## Remediation

Enable and enforce frame restrictions:

```python
# SAFE: Django
X_FRAME_OPTIONS = 'DENY'
MIDDLEWARE = ['django.middleware.clickjacking.XFrameOptionsMiddleware', ...]
```

```javascript
// SAFE: Express with helmet
app.use(helmet.frameguard({ action: 'deny' }));
```
