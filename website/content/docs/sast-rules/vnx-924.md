---
title: "VNX-924 – Improper Enforcement of Message Integrity During Transmission"
description: "Detect HTTP requests and serialized data sent without HMAC or digital signature integrity protection."
---

## Overview

Without integrity protection, an attacker positioned between sender and receiver (or with access to logs/message queues) can modify messages. The application cannot detect that the data has been tampered with, enabling privilege escalation, injecting malicious payloads, or replaying old messages.

**Severity:** Medium | **CWE:** [CWE-924 – Improper Enforcement of Message Integrity During Transmission in a Communication Channel](https://cwe.mitre.org/data/definitions/924.html)

## Why This Matters

APIs that process webhooks, inter-service messages, or serialized payloads without signature verification are vulnerable to message injection and replay attacks.

## What Gets Flagged

```python
# FLAGGED: POST without HMAC signature
requests.post(url, json={"action": "transfer", "amount": 1000})

# FLAGGED: Serialization without MAC
payload = pickle.dumps(user_object)
send_to_queue(payload)
```

## Remediation

Add HMAC or digital signatures to all sensitive messages:

```python
# SAFE: Add HMAC header
import hmac, hashlib
mac = hmac.new(SECRET_KEY, body, hashlib.sha256).hexdigest()
requests.post(url, json=body, headers={"X-HMAC": mac})
```
