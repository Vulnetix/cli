---
title: "VNX-663 – Use of a Non-reentrant Function in a Multithreaded Context"
description: "Detect calls to non-reentrant C library functions (strtok, localtime, rand) in files that also use threading primitives."
---

## Overview

Non-reentrant functions maintain internal state in static or global buffers. When called from multiple threads simultaneously, threads overwrite each other's state, causing corrupted results. Security-relevant functions like `rand()` (used for token generation), `strtok()` (used for parsing), and `localtime()` (used for audit timestamps) are particularly dangerous when called from concurrent code.

**Severity:** Low | **CWE:** [CWE-663 – Use of a Non-reentrant Function in a Multithreaded Context](https://cwe.mitre.org/data/definitions/663.html)

## Why This Matters

A multithreaded server calling `rand()` for session token generation has all threads sharing the same PRNG state. Concurrent calls produce non-random, correlated values that are predictable — effectively reducing the token entropy to zero. `strtok()` called from two threads parsing different strings will corrupt both results, potentially bypassing authentication parsing logic.

## What Gets Flagged

```c
// FLAGGED: strtok in a file that also uses pthread
#include <pthread.h>
// ...
char *token = strtok(input, ",");
while (token != NULL) {
    process(token);
    token = strtok(NULL, ",");  // uses static internal state
}
```

```c
// FLAGGED: rand() for security in threaded code
#include <pthread.h>
// ...
int session_id = rand();  // shared state across threads
```

```c
// FLAGGED: localtime() for audit in threaded code
#include <pthread.h>
struct tm *t = localtime(&timestamp);  // returns pointer to static struct
```

## Remediation

Use the reentrant `_r` variants on POSIX systems:

```c
// SAFE: strtok_r with caller-provided state
char *saveptr;
char *token = strtok_r(input, ",", &saveptr);
while (token != NULL) {
    process(token);
    token = strtok_r(NULL, ",", &saveptr);
}

// SAFE: localtime_r writes to caller-provided struct
struct tm result;
localtime_r(&timestamp, &result);

// SAFE: rand_r with per-thread seed
unsigned int seed = get_thread_seed();
int session_id = rand_r(&seed);

// BETTER: Use crypto-secure random source
int fd = open("/dev/urandom", O_RDONLY);
read(fd, &session_id, sizeof(session_id));
```
