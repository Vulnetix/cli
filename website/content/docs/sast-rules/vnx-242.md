---

## Overview

Certain functions are inherently unsafe: `gets()` reads unlimited input with no bounds; `strcpy()` and `strcat()` have no length parameters; `scanf("%s")` fills a buffer without limit; PHP `eval()`, `exec()`, `system()`, and `passthru()` execute arbitrary code or commands. There is no safe way to use these functions with untrusted input. This maps to [CWE-242: Use of Inherently Dangerous Function](https://cwe.mitre.org/data/definitions/242.html).

**Severity:** High | **CWE:** [CWE-242](https://cwe.mitre.org/data/definitions/242.html)

## Remediation

| Dangerous | Safe Alternative |
|-----------|-----------------|
| `gets(buf)` | `fgets(buf, sizeof(buf), stdin)` |
| `strcpy(d, s)` | `strlcpy(d, s, sizeof(d))` |
| `strcat(d, s)` | `strlcat(d, s, sizeof(d))` |
| `sprintf(buf, fmt, ...)` | `snprintf(buf, sizeof(buf), fmt, ...)` |
| PHP `eval($input)` | Remove; use data structures |
| PHP `system($cmd)` | Avoid; use parameterised APIs |
