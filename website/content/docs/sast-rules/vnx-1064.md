## Overview

Detects functions and methods with an excessive number of parameters (7 or more). Functions with many parameters are hard to call correctly, easy to pass arguments in the wrong order, and difficult to validate — increasing the risk that security-critical parameters are omitted or misused.

**Severity:** Low | **CWE:** [CWE-1064](https://cwe.mitre.org/data/definitions/1064.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
