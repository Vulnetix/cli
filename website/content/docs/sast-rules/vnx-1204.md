## Overview

VNX-1204 maps to [CWE-1204: Generation of Weak Initialization Vector (IV)](https://cwe.mitre.org/data/definitions/1204.html). Detects source patterns associated with CWE-1204 (Generation of Weak Initialization Vector (IV)). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-1204](https://cwe.mitre.org/data/definitions/1204.html) | **Languages:** java

## Why This Matters

This weakness class (Generation of Weak Initialization Vector (IV)) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1204 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'byte[] iv = new byte[16]' pattern
byte[] iv = new byte[16]
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1204 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1204` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
