---
title: "VNX-836 – Use of Password Hash Instead of Password for Authentication"
description: "Detect authentication logic that compares a user-submitted hash directly against a stored hash, allowing hash-as-password attacks."
---

## Overview

This rule flags authentication code that compares a user-submitted value labeled as a hash (`request.body.hash`, `request.POST.get('hash')`, `req.body.passwordHash`) directly against a stored hash value. When an application accepts a hash as a credential and compares it to `stored_hash`, an attacker who obtains the hash database can authenticate by submitting the hash directly — no password cracking required. This maps to [CWE-836: Use of Password Hash Instead of Password for Authentication](https://cwe.mitre.org/data/definitions/836.html).

**Severity:** High | **CWE:** [CWE-836](https://cwe.mitre.org/data/definitions/836.html)

## Why This Matters

The entire purpose of hashing passwords is to protect credentials at rest: even if the hash database is stolen, attackers must crack the hashes. If the application accepts the hash itself as the credential, the hash IS the password. An attacker who compromises the database (via SQL injection, a backup leak, or insider access) can authenticate as any user immediately, without cracking.

## What Gets Flagged

```python
# FLAGGED: comparing submitted hash against stored hash
submitted_hash = request.POST.get('hash')
if submitted_hash == stored_hash:
    grant_access(user)

# FLAGGED: accepting password_hash from request
password_hash = request.POST.get('password_hash')
if password_hash == user.password_hash:
    login(user)
```

```javascript
// FLAGGED: comparing submitted hash directly
const hash = req.body.hash;
if (hash === storedHash) {
    authenticateUser(userId);
}

// FLAGGED: hash comparison in auth
if (req.body.passwordHash === user.passwordHash) {
    session.userId = user.id;
}
```

```php
// FLAGGED: submitting and comparing hash
$hash = $_POST['hash'];
if ($hash === $stored_hash) {
    $_SESSION['authenticated'] = true;
}
```

## Remediation

```python
# SAFE: accept the plain password and verify against the hash
import bcrypt

submitted_password = request.POST.get('password').encode('utf-8')
if bcrypt.checkpw(submitted_password, user.password_hash):
    grant_access(user)

# SAFE: using Django's built-in auth
from django.contrib.auth import authenticate
user = authenticate(request, username=username, password=password)
if user is not None:
    login(request, user)
```

```javascript
// SAFE: accept plain password, verify with bcrypt
const bcrypt = require('bcrypt');
const password = req.body.password;
const match = await bcrypt.compare(password, user.passwordHash);
if (match) {
    req.session.userId = user.id;
}
```

```php
// SAFE: accept password, verify with password_verify
$password = $_POST['password'];
if (password_verify($password, $user['password_hash'])) {
    $_SESSION['authenticated'] = true;
}
```

Always accept the plaintext password from the client, hash it server-side, and compare using a constant-time comparison function. Never accept a pre-computed hash as the authentication credential.
