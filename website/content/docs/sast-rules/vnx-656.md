description: "Detect patterns where security relies on hiding endpoints, variable names, or implementation details rather than proper access controls."
---

## Overview

Security through obscurity is the practice of protecting systems by concealing information about their design — using unpredictable URL paths instead of proper authentication, hiding admin panels with non-obvious names, or assuming attackers won't find undocumented API endpoints. This provides no real security because determined attackers use automated tools, source code leaks, or JavaScript analysis to discover these "hidden" features.

**Severity:** Medium | **CWE:** [CWE-656 – Reliance on Security Through Obscurity](https://cwe.mitre.org/data/definitions/656.html)

## Why This Matters

Secret endpoints documented in JavaScript bundles, mobile app binaries, or source code leaks are trivially discovered. Attack tools like Dirbuster, ffuf, and Burp Suite actively enumerate common admin path variations. Once found, an endpoint protected only by obscurity is completely open to exploitation.

## What Gets Flagged

```python
# FLAGGED: Secret endpoint — obscurity as access control
SECRET_ENDPOINT = '/admin_xK7mP9'
# No authentication check in the handler

# FLAGGED: Hidden endpoint with no auth
@app.route('/hidden_admin_panel')
def admin():
    return render_template('admin.html')  # no auth decorator
```

```javascript
// FLAGGED: Secret path in client-side code
const SECRET_ADMIN_PATH = '/secret_admin_k9x7';
```

```python
# FLAGGED: Comment indicating security relies on obscurity
# Security: only known internally, don't document this endpoint
```

## Remediation

Replace obscurity with proper access controls:

```python
# SAFE: Proper authentication and authorization, no secret paths
@app.route('/admin')
@login_required
@permission_required('admin')
def admin_panel():
    return render_template('admin.html')
```

The path `/admin` is obvious and discoverable — that is acceptable because the route is protected by `@login_required` and `@permission_required`. Security comes from the access control mechanism, not the URL's obscurity.
