---
title: "VNX-826 – Premature Release of Resource That Is Still Required"
description: "Detect resources freed or closed while still in use: free() followed by buffer reuse, early close(), or defer Close() inside loops."
---

## Overview

This rule flags patterns where a resource is released while it may still be needed: a `free()`d buffer that is subsequently accessed, an early `.close()` call not at the end of resource usage, or `defer file.Close()` placed inside a loop (which defers until the enclosing function returns, not the loop iteration). This maps to [CWE-826: Premature Release of Resource That Is Still Required](https://cwe.mitre.org/data/definitions/826.html).

**Severity:** High | **CWE:** [CWE-826](https://cwe.mitre.org/data/definitions/826.html)

## Why This Matters

Releasing a resource prematurely and then using it leads to use-after-free memory corruption, reading from closed file descriptors (returning errors or reading unrelated data), or file descriptor exhaustion when `defer` in a loop accumulates thousands of deferred closes before the function returns.

## What Gets Flagged

```c
// FLAGGED: free followed by use of same buffer
free(buffer);
memcpy(buffer, src, len);  // buffer is freed — use-after-free

// FLAGGED: close called then file used again
fclose(fp);
fprintf(fp, "more data");  // fp is closed
```

```go
// FLAGGED: defer Close() inside loop — defers until function return, not iteration
func processFiles(paths []string) error {
    for _, path := range paths {
        f, err := os.Open(path)
        if err != nil {
            return err
        }
        defer f.Close()  // accumulates all closes until processFiles returns
        process(f)
    }
    return nil
}
```

## Remediation

```c
// SAFE: do not use buffer after free
process(buffer);  // use before freeing
free(buffer);
buffer = NULL;

// SAFE: close at end of usage, not in the middle
fprintf(fp, "data");
// ... all use of fp ...
fclose(fp);
```

```go
// SAFE: close explicitly within the loop iteration
func processFiles(paths []string) error {
    for _, path := range paths {
        if err := processFile(path); err != nil {
            return err
        }
    }
    return nil
}

func processFile(path string) error {
    f, err := os.Open(path)
    if err != nil {
        return err
    }
    defer f.Close()  // correct: defers to end of processFile, not the loop
    return process(f)
}
```

```go
// SAFE alternative: close explicitly in loop without helper function
func processFiles(paths []string) error {
    for _, path := range paths {
        f, err := os.Open(path)
        if err != nil {
            return err
        }
        err = process(f)
        f.Close()  // explicit close at end of iteration
        if err != nil {
            return err
        }
    }
    return nil
}
```
