## Overview

Detects missing documentation for program execution flow, particularly around security-critical paths such as authentication and authorisation flows. Undocumented execution paths make it difficult to verify that security controls are correctly applied.

**Severity:** Low | **CWE:** [CWE-1111](https://cwe.mitre.org/data/definitions/1111.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1111](https://cwe.mitre.org/data/definitions/1111.html) for authoritative guidance.
