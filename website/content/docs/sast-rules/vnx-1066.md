---
title: "VNX-1066 – Missing Serialization Control Element"
description: "Detect Java Serializable classes without serialVersionUID, causing unpredictable deserialization."
---

## Overview

Detects Java classes that implement `Serializable` without declaring a `serialVersionUID`. Without an explicit `serialVersionUID`, the JVM auto-generates one based on class structure; any class change can break deserialization compatibility.

**Severity:** Low | **CWE:** [CWE-1066 – Missing Serialization Control Element](https://cwe.mitre.org/data/definitions/1066.html)

## Why This Matters

Without `serialVersionUID`, adding a new field to a serialized class causes `InvalidClassException` when deserializing old data. More critically, explicit version control helps detect when serialized data from an unexpected version arrives — a sign of deserialization attack.

## What Gets Flagged

```java
// FLAGGED: Serializable without serialVersionUID
public class User implements Serializable {
    private String username;
    private String email;
    // no serialVersionUID!
}
```

## Remediation

```java
public class User implements Serializable {
    private static final long serialVersionUID = 1L;
    private String username;
    private String email;
}
```

Use `serialver` tool or IDE to generate a unique value based on class structure.
