---
title: "VNX-835 – Loop with Unreachable Exit Condition"
description: "Detect infinite loops (while True, while(1), for(;;), for{}) without a visible break, return, or exit statement in the loop body."
---

## Overview

This rule flags infinite loop constructs — `while True:` in Python, `while(1)`/`for(;;)` in C/C++, `for {}` in Go, `while (true)` in JavaScript/Java — that have no `break`, `return`, `raise`, `panic`, `exit`, or similar statement in the loop body within the next 20 lines. Unintentional infinite loops cause 100% CPU usage and render the application unresponsive. This maps to [CWE-835: Loop with Unreachable Exit Condition (Infinite Loop)](https://cwe.mitre.org/data/definitions/835.html).

**Severity:** Medium | **CWE:** [CWE-835](https://cwe.mitre.org/data/definitions/835.html)

## Why This Matters

An infinite loop is a denial of service vulnerability. In a server context, a single request that triggers an infinite loop consumes a worker thread/process indefinitely. An attacker who can trigger the code path repeatedly exhausts all available workers, making the service unavailable to legitimate users. Even in non-adversarial contexts, accidental infinite loops are a common source of production incidents.

## What Gets Flagged

```python
# FLAGGED: while True without break/return in next 20 lines
while True:
    data = process_queue()
    log(data)
    # No break, return, raise, or exit visible
```

```c
// FLAGGED: while(1) without exit in body
while(1) {
    handle_event();
    update_state();
    // No break, return, exit, or goto visible
}

// FLAGGED: for(;;) without exit
for(;;) {
    poll_sensors();
}
```

```go
// FLAGGED: for {} without break/return/panic
for {
    processMessage()
    updateCounter()
    // No break, return, panic visible
}
```

```javascript
// FLAGGED: while (true) without break/return/throw
while (true) {
    processNextItem();
    // No break, return, or throw visible
}
```

## Remediation

```python
# SAFE: include explicit exit condition
while True:
    data = process_queue()
    if data is None:
        break  # exit when queue is empty
    process(data)

# SAFE: use condition in while expression
running = True
while running:
    event = wait_for_event()
    if event.type == QUIT:
        running = False
    handle(event)
```

```c
// SAFE: event loop with signal-controlled exit
while (1) {
    handle_event();
    if (shutdown_requested) {
        break;
    }
}

// SAFE: explicit iteration limit for retry loops
for (int attempt = 0; attempt < MAX_RETRIES; attempt++) {
    if (try_connect()) break;
    sleep(1);
}
```

```go
// SAFE: loop with break condition
for {
    msg, ok := <-ch
    if !ok {
        break  // channel closed
    }
    process(msg)
}
```

Ensure every infinite loop has at least one reachable exit path. Use bounded loops (`for i := 0; i < maxIter; i++`) when the number of iterations is known, and include explicit exit conditions in event loops.
