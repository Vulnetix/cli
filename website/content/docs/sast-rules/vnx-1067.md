## Overview

Detects patterns that perform sequential (linear) searches on large data collections instead of using indexed lookups, hash maps, or binary search. Sequential searches on large datasets cause O(n) performance that can be exploited to cause denial of service.

**Severity:** Low | **CWE:** [CWE-1067](https://cwe.mitre.org/data/definitions/1067.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1067](https://cwe.mitre.org/data/definitions/1067.html) for authoritative guidance.
