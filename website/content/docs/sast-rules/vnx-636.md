---
title: "VNX-636 – Not Failing Securely (Fail Open)"
description: "Detect exception handlers that grant access or return success when an error occurs, causing the system to fail open."
---

## Overview

A security control that fails open grants access when it should deny. When an authentication or authorization check raises an exception, catching that exception and returning `True` (or continuing as if the check passed) means any error condition — network timeout, database outage, malformed token — becomes a free pass for attackers.

**Severity:** High | **CWE:** [CWE-636 – Not Failing Securely (Fail Open)](https://cwe.mitre.org/data/definitions/636.html)

## Why This Matters

An attacker who can trigger an exception in an authentication check (e.g., by providing a malformed JWT, causing a database connection error, or exploiting an integer overflow) gains unauthorized access. This transforms any error condition into an authentication bypass, making the security control unreliable precisely when it is most needed.

## What Gets Flagged

```python
# FLAGGED: Exception returns True — grants access on any error
def is_authorized(user, resource):
    try:
        return check_permission(user, resource)
    except:
        return True  # WRONG: fail open

# FLAGGED: Auth check swallows exception and continues
try:
    verify_token(token)
except Exception:
    pass  # WRONG: execution continues as if token was valid
```

```javascript
// FLAGGED: catch block returns true (authorized)
function checkAuth(token) {
    try {
        return verifyJWT(token);
    } catch (e) {
        return true;  // WRONG: fail open
    }
}
```

## Remediation

Always fail closed — deny access on any error, and propagate or log the exception:

```python
# SAFE: fail closed — deny on exception
def is_authorized(user, resource):
    try:
        return check_permission(user, resource)
    except PermissionError:
        return False
    except Exception as e:
        logger.error("Permission check failed: %s", e)
        return False  # deny by default

# SAFE: re-raise so the caller handles it
def verify_auth(token):
    try:
        return decode_token(token)
    except TokenExpiredError:
        raise AuthenticationError("Token expired")
    except Exception as e:
        logger.error("Token verification error: %s", e)
        raise AuthenticationError("Authentication failed")
```
