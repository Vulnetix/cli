## Overview

VNX-1071 maps to [CWE-1071: Empty Code Block](https://cwe.mitre.org/data/definitions/1071.html). Detects source patterns associated with CWE-1071 (Empty Code Block). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1071](https://cwe.mitre.org/data/definitions/1071.html) | **Languages:** java

## Why This Matters

This weakness class (Empty Code Block) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1071 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'void ' pattern
void 
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1071 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1071` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
