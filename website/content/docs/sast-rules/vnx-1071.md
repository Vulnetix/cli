## Overview

Detects empty code blocks including empty if/for/while bodies and empty function implementations. Empty blocks can indicate unimplemented security controls, skipped validation, or dead code that creates confusion about intent.

**Severity:** Low | **CWE:** [CWE-1071](https://cwe.mitre.org/data/definitions/1071.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1071](https://cwe.mitre.org/data/definitions/1071.html) for authoritative guidance.
