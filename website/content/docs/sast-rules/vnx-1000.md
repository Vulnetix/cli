## Overview

This rule flags broadly common insecure coding patterns that span multiple CWE categories. CWE-1000 is the root node of the CWE Research Concepts view and covers fundamental vulnerability patterns. This rule specifically targets hardcoded credentials, unsafe eval/exec usage, and explicitly disabled security controls.

**Severity:** Info | **CWE:** [CWE-1000 – Research Concepts](https://cwe.mitre.org/data/definitions/1000.html)

## Why This Matters

The patterns detected by this rule represent foundational security mistakes. Hardcoded credentials allow attackers who obtain source code to gain immediate access to systems. Disabled SSL verification and unsafe evaluation are common shortcuts that introduce severe vulnerabilities.

## What Gets Flagged

```python
# FLAGGED: Hardcoded password
password = "mysecretpassword"

# FLAGGED: SSL verification disabled
requests.get(url, verify=False)
```

```php
<?php
// FLAGGED: Hardcoded secret
$secret = "my_api_secret_key";
```

## Remediation

1. Use environment variables or secrets managers for credentials.
2. Never disable SSL/TLS verification in production code.
3. Replace `eval()` and `exec()` with safe static alternatives.
