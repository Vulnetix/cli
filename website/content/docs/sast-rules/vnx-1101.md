## Overview

VNX-1101 maps to [CWE-1101: Reliance on Runtime Component in Generated Code](https://cwe.mitre.org/data/definitions/1101.html). Detects source patterns associated with CWE-1101 (Reliance on Runtime Component in Generated Code). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1101](https://cwe.mitre.org/data/definitions/1101.html) | **Languages:** java

## Why This Matters

This weakness class (Reliance on Runtime Component in Generated Code) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1101 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'Runtime.getRuntime(' pattern
Runtime.getRuntime(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1101 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1101` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
