---
title: "VNX-252 – Unchecked Return Value"
description: "Detect security-relevant function calls whose return values are explicitly discarded."
---

## Overview

Security-critical functions (privilege drops, file writes, cryptographic operations) signal failure via their return value. Ignoring these signals leaves the application in a potentially compromised state — a failed `setuid()` leaves the process as root; a failed `write()` means data was not saved. This maps to [CWE-252: Unchecked Return Value](https://cwe.mitre.org/data/definitions/252.html).

**Severity:** Medium | **CWE:** [CWE-252](https://cwe.mitre.org/data/definitions/252.html)

## What Gets Flagged

```go
// FLAGGED: discarded db error
_ = db.Exec("DELETE FROM sessions WHERE user=?", userID)

// FLAGGED: discarded file removal error
_ = os.Remove(tempFile)
```

## Remediation

```go
// SAFE: check all errors
if err := db.Exec("DELETE FROM sessions WHERE user=?", userID); err != nil {
    log.Printf("session cleanup failed: %v", err)
}
```
