## Overview

Detects patterns where variables are reused for different purposes, particularly when sensitive data (passwords, tokens) is stored in a variable that is later reused for a different purpose. Variable reuse with sensitive data extends the lifetime of sensitive values in memory.

**Severity:** Low | **CWE:** [CWE-1109](https://cwe.mitre.org/data/definitions/1109.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1109](https://cwe.mitre.org/data/definitions/1109.html) for authoritative guidance.
