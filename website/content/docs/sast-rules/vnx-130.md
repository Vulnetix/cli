description: "Detect memory operations that use length values from untrusted network headers or user input without validation."
---

## Overview

This rule flags memory operations (`memcpy`, `memmove`, `recv`, `read`) that use a length value from an untrusted source such as a network packet header field (`ntohl`, `hdr->len`, `pkt->len`) without first validating the length against the actual buffer size. An attacker crafts a packet with a length field larger than the allocated buffer, causing a heap or stack buffer overflow. This maps to [CWE-130: Improper Handling of Length Parameter Inconsistency](https://cwe.mitre.org/data/definitions/130.html).

**Severity:** Medium | **CWE:** [CWE-130 – Improper Handling of Length Parameter Inconsistency](https://cwe.mitre.org/data/definitions/130.html)

## What Gets Flagged

```c
// FLAGGED: memcpy with length from network header
uint32_t recv_len = ntohl(hdr->len);
memcpy(dest, src, recv_len);  // recv_len could be >> sizeof(dest)
```

## Remediation

```c
// SAFE: validate length before use
uint32_t recv_len = ntohl(hdr->len);
if (recv_len > sizeof(dest)) {
    return -1;  // reject oversized message
}
memcpy(dest, src, recv_len);
```
