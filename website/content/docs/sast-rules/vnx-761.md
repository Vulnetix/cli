---
title: "VNX-761 – Free of Pointer not at Start of Buffer"
description: "Detect C free() calls on pointers that have been incremented or offset from their original allocation address, causing heap corruption."
---

## Overview

This rule flags `free()` calls where the pointer argument has been offset from the original allocation address using arithmetic or increment operators. `free()` must receive the exact pointer returned by `malloc`/`calloc`/`realloc` — passing a different address corrupts the heap allocator's internal bookkeeping structures, typically resulting in a crash or exploitable heap corruption. This maps to [CWE-761: Free of Pointer not at Start of Buffer](https://cwe.mitre.org/data/definitions/761.html).

**Severity:** High | **CWE:** [CWE-761](https://cwe.mitre.org/data/definitions/761.html)

## Why This Matters

Heap corruption is one of the most exploitable vulnerability classes. Modern exploits routinely turn heap corruption into arbitrary code execution by overwriting allocator metadata. This specific bug is common in string parsing code where a pointer is advanced through a buffer and then freed at the advanced position.

## What Gets Flagged

```c
// FLAGGED: free of pointer + offset
char *buf = malloc(256);
// ... advance buf through the buffer ...
buf += header_size;  // or: buf = buf + offset;
free(buf + offset);  // wrong: not the original malloc address

// FLAGGED: free of incremented pointer
char *ptr = malloc(256);
ptr++;           // advance
free(ptr++);     // wrong: incremented before and during free call
free(++ptr);     // wrong: pre-incremented pointer
```

## Remediation

```c
// SAFE: keep the original pointer and use a separate pointer for traversal
char *buf = malloc(256);
char *cursor = buf;  // use cursor for traversal
while (*cursor != '\0') {
    process(*cursor);
    cursor++;
}
free(buf);  // free the original allocation address
buf = NULL;

// SAFE: if you must advance the original pointer, save a copy
char *orig = malloc(256);
char *ptr = orig;
ptr += offset;
// ... use ptr ...
free(orig);  // free the saved original
```
