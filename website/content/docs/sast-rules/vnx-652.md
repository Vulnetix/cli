title: "VNX-652 – Improper Neutralization of Data within XPath Expressions (XPath Injection)"
description: "Detect XPath query construction using string concatenation with unsanitized user input."
---

## Overview

XPath injection occurs when user-supplied data is concatenated directly into an XPath expression, allowing an attacker to modify the query structure. Since XPath 1.0 has no parameterized query support in many implementations, developers often build queries via string concatenation, creating injection vulnerabilities analogous to SQL injection.

**Severity:** High | **CWE:** [CWE-652 – Improper Neutralization of Data within XPath Expressions (XPath Injection)](https://cwe.mitre.org/data/definitions/652.html)

## Why This Matters

An authentication XPath query like `//users/user[username/text()='$user' and password/text()='$pass']` can be bypassed with `username = ' or '1'='1` which evaluates to `//users/user[username/text()='' or '1'='1' and ...]`, returning all users. This is equivalent to SQL injection — a complete authentication bypass.

## What Gets Flagged

```java
// FLAGGED: XPath query built with string concatenation
String query = "//users/user[@username='" + username + "' and @password='" + password + "']";
NodeList result = (NodeList) xpath.evaluate(query, document, XPathConstants.NODESET);
```

```python
# FLAGGED: f-string XPath expression
query = f"//users[@name='{username}']"
results = tree.xpath(query)

# FLAGGED: String concatenation in xpath()
nodes = doc.xpath("//item[@id='" + item_id + "']")
```

```php
<?php
// FLAGGED: XPath with user input concatenation
$query = "//user[@name='" . $_GET['name'] . "']";
$result = $xpath->query($query);
```

## Remediation

Use parameterized XPath where available, or sanitize input by escaping single quotes:

```java
// SAFE: Use XPath variables (parameterized)
XPathVariableResolver resolver = new XPathVariableResolver() {
    public Object resolveVariable(QName name) {
        if ("username".equals(name.getLocalPart())) return username;
        return null;
    }
};
xpath.setXPathVariableResolver(resolver);
NodeList result = (NodeList) xpath.evaluate(
    "//users/user[@username=$username]", document, XPathConstants.NODESET);
```

```python
# SAFE: Use lxml's XPath variable binding
results = tree.xpath("//users[@name=$name]", name=username)
```
