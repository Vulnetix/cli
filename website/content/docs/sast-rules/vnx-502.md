---
title: "VNX-502 – Deserialization of Untrusted Data"
description: "Detect use of unsafe deserialization functions across Python, Java, PHP, Ruby, Node.js, and Go that can enable remote code execution when processing attacker-controlled data."
---

## Overview

This rule flags calls to unsafe deserialization functions that process arbitrary data streams into live objects without type-safe constraints. Deserializing attacker-controlled data can instantiate arbitrary classes, invoke constructors and finalizers, and ultimately achieve remote code execution — all before any application-level validation runs. This maps to [CWE-502: Deserialization of Untrusted Data](https://cwe.mitre.org/data/definitions/502.html).

**Severity:** High | **CWE:** [CWE-502 – Deserialization of Untrusted Data](https://cwe.mitre.org/data/definitions/502.html)

## Why This Matters

Deserialization vulnerabilities have caused some of the most severe breaches of the last decade, including the Apache Struts vulnerability exploited in the Equifax breach (CVE-2017-5638) and widespread Java deserialization attacks via `ObjectInputStream`. Unlike SQL injection, which requires attacker-controlled values to reach a specific sink, deserialization attacks often bypass all application logic — the payload is evaluated during the deserialization step itself, before your code runs. Python's `pickle` module explicitly warns in its documentation that it is not secure against erroneous or maliciously constructed data. MITRE ATT&CK T1059 covers the execution of attacker-supplied code via interpreter abuse, which deserialization gadget chains facilitate.

## What Gets Flagged

The rule fires when any of these patterns appear in source files:

```python
# FLAGGED: pickle deserializes arbitrary Python objects — enables RCE
import pickle
data = pickle.loads(request.body)  # attacker controls request.body

# FLAGGED: yaml.load without SafeLoader allows Python object construction
import yaml
config = yaml.load(user_input)  # use yaml.safe_load() instead

# FLAGGED: marshal is not secure against malicious input
import marshal
obj = marshal.loads(untrusted_bytes)
```

```java
// FLAGGED: Java native deserialization — gadget chains in classpath enable RCE
ObjectInputStream ois = new ObjectInputStream(request.getInputStream());
Object obj = ois.readObject();  // dangerous: deserializes arbitrary classes

// FLAGGED: XStream deserialization — many CVEs for arbitrary code execution
XStream xstream = new XStream();
Object obj = xstream.fromXML(userInput);
```

```php
<?php
// FLAGGED: PHP unserialize with user input — __wakeup and __destruct gadgets
$data = unserialize($_POST['data']);
```

```ruby
# FLAGGED: Ruby Marshal can execute arbitrary code during load
obj = Marshal.load(user_data)

# FLAGGED: YAML.load (not safe_load) can instantiate arbitrary Ruby objects
config = YAML.load(params[:config])
```

```go
// FLAGGED: gob.Decode with untrusted network input
dec := gob.NewDecoder(conn)
dec.Decode(&untrustedStruct)
```

## Remediation

### Python

```python
# SAFE: Use json for data interchange — no object instantiation
import json
data = json.loads(request.body)

# SAFE: Use yaml.safe_load — restricts to basic YAML types
import yaml
config = yaml.safe_load(user_input)

# SAFE: If pickle is truly necessary, sign the payload with HMAC and verify before loading
import pickle, hmac, hashlib

SECRET = b"your-secret-key"

def safe_loads(payload: bytes, signature: bytes) -> object:
    expected = hmac.new(SECRET, payload, hashlib.sha256).digest()
    if not hmac.compare_digest(expected, signature):
        raise ValueError("Invalid signature")
    return pickle.loads(payload)
```

### Java

```java
// SAFE: Use Jackson with strict type binding — no polymorphic deserialization
ObjectMapper mapper = new ObjectMapper();
// Disable default typing — never call enableDefaultTyping()
mapper.deactivateDefaultTyping();
MyDto dto = mapper.readValue(json, MyDto.class);

// SAFE: Use a deserialization filter for ObjectInputStream (Java 9+)
ObjectInputStream ois = new ObjectInputStream(inputStream);
ois.setObjectInputFilter(info -> {
    if (info.serialClass() != null && !ALLOWED_CLASSES.contains(info.serialClass().getName())) {
        return ObjectInputFilter.Status.REJECTED;
    }
    return ObjectInputFilter.Status.ALLOWED;
});
```

### PHP

```php
<?php
// SAFE: Use JSON instead of unserialize
$data = json_decode($_POST['data'], true);

// SAFE: If you must use serialize/unserialize, sign with HMAC
function safe_unserialize(string $payload, string $hmac): mixed {
    $expected = hash_hmac('sha256', $payload, SECRET_KEY);
    if (!hash_equals($expected, $hmac)) {
        throw new RuntimeException('Tampered payload');
    }
    return unserialize($payload, ['allowed_classes' => [MyClass::class]]);
}
```

### Ruby

```ruby
# SAFE: Use JSON for data exchange
require 'json'
data = JSON.parse(params[:data])

# SAFE: Use YAML.safe_load — restricts to basic types
require 'yaml'
config = YAML.safe_load(params[:config])
```

### Go

```go
// SAFE: Use encoding/json with a known struct type
import "encoding/json"

var dto MyDTO
if err := json.Unmarshal(untrustedData, &dto); err != nil {
    return err
}

// SAFE: If using gob, only decode into concrete types from trusted sources
// and apply size limits to prevent DoS
```
