---
title: "VNX-223 – Omission of Security-relevant Information"
description: "Detect authentication and authorization failure paths that do not log the security event."
---

## Overview

Authentication failures, authorization denials, and security exceptions that are silently swallowed leave no audit trail. Without logs, intrusion detection is impossible and forensic investigation is severely hampered. This maps to [CWE-223: Omission of Security-relevant Information](https://cwe.mitre.org/data/definitions/223.html).

**Severity:** Low | **CWE:** [CWE-223](https://cwe.mitre.org/data/definitions/223.html)

## Remediation

Log all authentication failures with username, timestamp, and source IP. Never use empty `catch` blocks for security exceptions.

```python
# SAFE: log failed auth
except AuthenticationError as e:
    logger.warning("Auth failure for user %s from %s: %s",
                   username, request.remote_addr, str(e))
    return False
```
