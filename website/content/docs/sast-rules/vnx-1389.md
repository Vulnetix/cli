description: "Detect parseInt() without explicit radix in JavaScript, Python int() on octal-looking strings, and Go strconv.ParseInt with base 0, which can cause radix-confusion leading to unexpected numeric values."
---

## Overview

Numbers with leading zeros are historically parsed as octal in C-derived languages. In JavaScript, `parseInt('08')` returned `0` in pre-ES5 engines (8 is not a valid octal digit). In Go, `strconv.ParseInt("010", 0, 64)` returns `8`, not `10`. These radix surprises cause silent logic errors in security checks, access control comparisons, and arithmetic. **Severity:** Medium | **CWE:** [CWE-1389 – Incorrect Parsing of Numbers with Different Radices](https://cwe.mitre.org/data/definitions/1389.html)

## Why This Matters

User-supplied numbers in URLs or API inputs often carry leading zeros (e.g. postal codes `01234`, phone extensions `0800`, time values `09:00`). If these are parsed as octal, the resulting integer is wrong. An authorization check comparing parsed user ID `010` (expecting decimal 10) against stored ID `8` (parsed as octal) would succeed for the wrong user.

## What Gets Flagged

```javascript
// FLAGGED: parseInt without radix
const port = parseInt(req.query.port);  // '08' -> 0 in older engines
const userId = parseInt(params.id);     // ambiguous
```

```python
# FLAGGED: int() on potential octal-looking string
user_id = int("010")  # returns 10 (decimal) in Python 3, but intent may be 8
permissions = int(octal_string)  # unclear base
```

```go
// FLAGGED: ParseInt with base 0 (auto-detect)
n, _ := strconv.ParseInt(userInput, 0, 64)  // "010" = 8 (octal), not 10
```

## Remediation

```javascript
// SAFE: Always specify radix 10
const port = parseInt(req.query.port, 10);
const userId = parseInt(params.id, 10);

// Or use Number() which always parses as decimal (but doesn't handle hex)
const userId = Number(params.id);
```

```python
# SAFE: Explicit base for ambiguous strings
user_id = int("010", 10)   # explicitly decimal = 10
octal_val = int("010", 8)  # explicitly octal = 8

# For validated decimal input from users, always use base 10
user_id = int(request.GET['id'], 10)
```

```go
// SAFE: Use explicit base 10
n, err := strconv.ParseInt(userInput, 10, 64)
if err != nil {
    return fmt.Errorf("invalid number: %w", err)
}
```
