---
title: "VNX-838 – Inappropriate Encoding for Output Context"
description: "Detect HTML encoding applied in a SQL context or URL encoding applied in an HTML rendering context, where the encoding does not match the output sink."
---

## Overview

This rule flags encoding mismatches where the encoding function used does not match the output context: HTML-encoding functions (`htmlspecialchars`, `html.escape`) used in code that appears to be building SQL queries, or URL-encoding functions (`urllib.parse.quote`, `encodeURIComponent`) used near HTML template rendering. Using the wrong encoding for the output context provides no protection against injection in that context. This maps to [CWE-838: Inappropriate Encoding for Output Context](https://cwe.mitre.org/data/definitions/838.html).

**Severity:** Medium | **CWE:** [CWE-838](https://cwe.mitre.org/data/definitions/838.html)

## Why This Matters

`htmlspecialchars()` converts `<`, `>`, `&` and `"` to HTML entities, which protects against HTML/XSS injection but does nothing against SQL injection. If a developer applies HTML escaping before inserting data into a SQL query, the SQL metacharacters (`'`, `--`, `;`) are not neutralized. Similarly, URL percent-encoding does nothing to prevent XSS in an HTML context.

## What Gets Flagged

```php
// FLAGGED: HTML encoding used in SQL context
$name = htmlspecialchars($_POST['name']);
$query = "SELECT * FROM users WHERE name = '$name'";  // SQL injection still possible
```

```python
# FLAGGED: html.escape used near SQL query building
import html
username = html.escape(request.POST.get('username'))
query = f"SELECT * FROM users WHERE username = '{username}'"  # SQL injection
```

```javascript
// FLAGGED: encodeURIComponent used in HTML rendering context
const username = encodeURIComponent(req.query.user);
res.send(`<div>Hello, ${username}</div>`);  // XSS: % encoding != HTML encoding
```

```python
# FLAGGED: URL encoding near HTML template rendering
from urllib.parse import quote
username = quote(user_input)
html_output = f"<p>Welcome, {username}</p>"  // URL encoding doesn't prevent XSS
```

## Remediation

```php
// SAFE: use parameterized queries for SQL (not HTML encoding)
$stmt = $pdo->prepare("SELECT * FROM users WHERE name = ?");
$stmt->execute([$_POST['name']]);

// SAFE: use htmlspecialchars() for HTML OUTPUT context
$name = htmlspecialchars($user_input, ENT_QUOTES, 'UTF-8');
echo "<p>Hello, $name</p>";
```

```python
# SAFE: use parameterized queries for SQL
cursor.execute("SELECT * FROM users WHERE username = %s", (username,))

# SAFE: use template engine auto-escaping for HTML
from markupsafe import escape
safe_name = escape(user_input)
html_output = f"<p>Welcome, {safe_name}</p>"
```

```javascript
// SAFE: use parameterized queries for SQL
const result = await db.query('SELECT * FROM users WHERE name = $1', [username]);

// SAFE: use template engine HTML escaping for HTML output
const DOMPurify = require('dompurify');
const safeUsername = DOMPurify.sanitize(req.query.user);
res.render('profile', { username: safeUsername });
```

Match the encoding to the output context: parameterized queries for SQL, HTML entity encoding for HTML, JSON encoding for JSON, and URL encoding only for URL components.
