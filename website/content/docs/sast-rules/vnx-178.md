## Overview

VNX-178 maps to [CWE-178: Improper Handling of Case Sensitivity](https://cwe.mitre.org/data/definitions/178.html). Detects source patterns associated with CWE-178 (Improper Handling of Case Sensitivity). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-178](https://cwe.mitre.org/data/definitions/178.html) | **Languages:** go, java, node, python

## Why This Matters

This weakness class (Improper Handling of Case Sensitivity) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-178 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'equals("admin"' pattern
equals("admin"
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-178 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-178` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
