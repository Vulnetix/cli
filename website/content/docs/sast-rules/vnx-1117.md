## Overview

Detects exported/public functions and methods in security-relevant files that lack adequate documentation describing their security contract, expected inputs, and trust assumptions. Without a clear behavioural summary, callers may misuse security-critical functions.

**Severity:** Low | **CWE:** [CWE-1117](https://cwe.mitre.org/data/definitions/1117.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1117](https://cwe.mitre.org/data/definitions/1117.html) for authoritative guidance.
