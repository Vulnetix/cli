---
title: "VNX-628 – Function Call with Incorrectly Specified Arguments (Wrong Crypto Key/Message Order)"
description: "Detect HMAC and cryptographic function calls where the key and message arguments are swapped."
---

## Overview

Cryptographic functions like HMAC have strict argument order requirements: the secret key must be the first argument and the message to be authenticated is the second. Swapping these arguments produces a valid-looking HMAC that can be forged by anyone who knows the (publicly visible) message, completely defeating the authentication guarantee.

**Severity:** Medium | **CWE:** [CWE-628 – Function Call with Incorrectly Specified Arguments](https://cwe.mitre.org/data/definitions/628.html)

## Why This Matters

`hmac.new(message, key)` in Python uses `message` as the key and `key` as the message. Since the message is typically attacker-controlled, the attacker can generate valid HMACs for arbitrary messages, bypassing signature verification on JWT tokens, webhook payloads, and API request signing.

## What Gets Flagged

```python
# FLAGGED: message and key swapped — message is used as HMAC key
import hmac
mac = hmac.new(message, key, hashlib.sha256)

# FLAGGED: Wrong order — should be hmac.new(key, message, ...)
signature = hmac.new(payload, secret_key, hashlib.sha256).hexdigest()
```

```javascript
// FLAGGED: createHmac(algorithm, key) — second arg should be the key
// Here message is passed where key is expected
const mac = crypto.createHmac('sha256', message).update(key).digest('hex');
```

## Remediation

Always put the secret key first and the message second:

```python
# SAFE: key first, message second
import hmac, hashlib
mac = hmac.new(secret_key, message, hashlib.sha256)
signature = mac.hexdigest()
```

```javascript
// SAFE: key is second argument to createHmac
const mac = crypto.createHmac('sha256', secretKey).update(message).digest('hex');
```

Refer to official documentation and use named parameters or typed wrappers to prevent argument confusion.
