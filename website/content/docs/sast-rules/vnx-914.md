title: "VNX-914 – Improper Control of Dynamically-Identified Variables"
description: "Detect PHP variable-variables and Python locals()/globals() lookups with user-controlled keys that allow arbitrary variable overwrite."
---

## Overview

Dynamic variable access allows code to look up or set a variable whose name is determined at runtime. When the variable name comes from user input, an attacker can target any in-scope variable, including authentication flags, file paths, configuration values, and SQL query fragments.

**Severity:** High | **CWE:** [CWE-914 – Improper Control of Dynamically-Identified Variables](https://cwe.mitre.org/data/definitions/914.html)

## Why This Matters

PHP's variable-variable syntax (`$$varName`) and Python's `locals()[user_key]` allow attackers to overwrite security-critical variables. A single `$$_GET['var']` call can set `$is_admin`, `$authenticated`, or any other variable to any value.

## What Gets Flagged

```php
<?php
// FLAGGED: Variable variable with GET param
$varName = $_GET['name'];
$$varName = $_GET['value'];  // can overwrite $is_admin, $query, etc.
```

```python
# FLAGGED: Dynamic local variable access
locals()[request.GET.get('var')]
globals()[user_input]
getattr(self, request.data['field'])
```

## Remediation

Use an explicit allowlist of permitted variable names:

```php
<?php
// SAFE: Allowlist
$allowed = ['theme', 'language', 'timezone'];
if (in_array($_GET['setting'], $allowed)) {
    $$_GET['setting'] = $_GET['value'];
}
```
