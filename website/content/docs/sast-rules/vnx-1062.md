---
title: "VNX-1062 – Parent Class with References to Child Class"
description: "Detect parent classes that reference specific subclasses, creating circular dependencies."
---

## Overview

Detects parent/base classes that explicitly reference or test for their subclasses (`isinstance`, `instanceof`). This creates circular dependencies that prevent proper encapsulation and can break security inheritance hierarchies.

**Severity:** Low | **CWE:** [CWE-1062 – Parent Class with References to Child Class](https://cwe.mitre.org/data/definitions/1062.html)

## Why This Matters

A base `User` class that checks `isinstance(self, AdminUser)` creates a tight coupling where the base class must know about all subclasses. Adding new subclasses can inadvertently bypass security checks in the base class.

## What Gets Flagged

```python
# FLAGGED: base class checking for specific subtype
class User:
    def get_data(self):
        if isinstance(self, AdminUser):  # base knows about child
            return self._admin_data
```

## Remediation

Use polymorphism — define abstract methods in the base class and let each subclass implement them:

```python
class User:
    def get_accessible_data(self):
        raise NotImplementedError

class AdminUser(User):
    def get_accessible_data(self):
        return self._admin_data
```
