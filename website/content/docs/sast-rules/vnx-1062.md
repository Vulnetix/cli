## Overview

VNX-1062 maps to [CWE-1062: Parent Class with References to Child Class](https://cwe.mitre.org/data/definitions/1062.html). Detects source patterns associated with CWE-1062 (Parent Class with References to Child Class). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1062](https://cwe.mitre.org/data/definitions/1062.html) | **Languages:** java

## Why This Matters

This weakness class (Parent Class with References to Child Class) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1062 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'extends ' pattern
extends 
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1062 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1062` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
