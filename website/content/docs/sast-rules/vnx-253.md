---
title: "VNX-253 – Incorrect Check of Function Return Value"
description: "Detect write()/send() checked against -1 instead of the expected byte count, missing partial write detection."
---

## Overview

`write()` and `send()` return the number of bytes written, which may be less than requested (partial write). Checking only `!= -1` treats a partial write as success. `strcmp()` returns 0 for equality but is commonly misused as a boolean (where 0 is falsy). This maps to [CWE-253: Incorrect Check of Function Return Value](https://cwe.mitre.org/data/definitions/253.html).

**Severity:** Medium | **CWE:** [CWE-253](https://cwe.mitre.org/data/definitions/253.html)

## Remediation

```c
// SAFE: check against expected count
ssize_t written = write(fd, buf, n);
if (written != (ssize_t)n) {
    // handle partial write or error
}

// SAFE: strcmp equality
if (strcmp(a, b) == 0) { /* strings are equal */ }
```
