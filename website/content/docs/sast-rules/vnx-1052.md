title: "VNX-1052 – Excessive Use of Hard-coded Literals in Initialization"
description: "Detect magic numbers and string literals used for security-sensitive configuration values without named constants."
---

## Overview

Using numeric or string literals directly in security-sensitive configuration (session timeouts, token expiry, port numbers, iteration counts) rather than named constants makes the code hard to review and the values easy to misconfigure. During code review it is impossible to tell whether `86400` is a session timeout or an unrelated calculation.

**Severity:** Low | **CWE:** [CWE-1052 – Excessive Use of Hard-Coded Literals in Initialization](https://cwe.mitre.org/data/definitions/1052.html)

## Why This Matters

Magic numbers in security configuration are frequently copied incorrectly. A `3600` (1 hour) session timeout copied from one project may need to be `900` (15 minutes) in a higher-security context but the intent is invisible without a constant name.

## What Gets Flagged

```python
# FLAGGED: Magic number without name
session['expires'] = time.time() + 86400

# FLAGGED: Hardcoded port without constant
server.listen(3306)
```

## Remediation

Define named constants:

```python
# SAFE
SESSION_TIMEOUT_SECONDS = 3600  # 1 hour
session['expires'] = time.time() + SESSION_TIMEOUT_SECONDS

DATABASE_PORT = 3306
server.listen(DATABASE_PORT)
```
