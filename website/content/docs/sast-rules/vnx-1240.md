## Overview

Implementing cryptographic primitives from scratch — custom ciphers, custom hash functions, custom key derivation — is almost always insecure. Even experienced cryptographers make subtle mistakes that are invisible to code review but exploitable in practice. Similarly, using deprecated algorithms like DES, 3DES, RC4, or Blowfish provides false security. **Severity:** High | **CWE:** [CWE-1240 – Use of a Cryptographic Primitive with a Risky Implementation](https://cwe.mitre.org/data/definitions/1240.html)

## Why This Matters

The history of cryptography is littered with "obviously correct" custom implementations that were broken: WEP's RC4 misuse, the Debian OpenSSL random number generator bug, Sony's PS3 ECDSA constant-nonce bug. XOR with a static key is trivially reversible with known plaintext. DES has a 56-bit key that can be brute-forced in hours.

## What Gets Flagged

```python
# FLAGGED: Custom encryption function
def encrypt(data, key):
    return bytes([b ^ key[i % len(key)] for i, b in enumerate(data)])

# FLAGGED: XOR cipher
def xor_encrypt(plaintext, key):
    return ''.join(chr(ord(c) ^ ord(k)) for c, k in zip(plaintext, cycle(key)))
```

```java
// FLAGGED: DES cipher
Cipher cipher = Cipher.getInstance("DES/CBC/PKCS5Padding");

// FLAGGED: RC4
Cipher cipher = Cipher.getInstance("RC4");
```

## Remediation

```python
# SAFE: Use cryptography library with AES-GCM
from cryptography.hazmat.primitives.ciphers.aead import AESGCM
import os

key = AESGCM.generate_key(bit_length=256)
aesgcm = AESGCM(key)
nonce = os.urandom(12)
ciphertext = aesgcm.encrypt(nonce, plaintext, associated_data)
```

```java
// SAFE: AES-GCM via JCE
Cipher cipher = Cipher.getInstance("AES/GCM/NoPadding");
byte[] nonce = new byte[12];
new SecureRandom().nextBytes(nonce);
cipher.init(Cipher.ENCRYPT_MODE, secretKey, new GCMParameterSpec(128, nonce));
byte[] ciphertext = cipher.doFinal(plaintext);
```
