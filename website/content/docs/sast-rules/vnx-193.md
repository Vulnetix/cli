---
title: "VNX-193 – Off-by-one Error"
description: "Detect loop bounds using <= instead of <, and malloc(strlen(s)) without +1 for null terminator."
---

## Overview

An off-by-one error occurs when a loop iterates one time too many, writing one element past the end of a buffer, or when `malloc(strlen(s))` allocates no room for the null terminator. These single-byte overflows are exploitable for stack/heap corruption. This maps to [CWE-193: Off-by-one Error](https://cwe.mitre.org/data/definitions/193.html).

**Severity:** High | **CWE:** [CWE-193](https://cwe.mitre.org/data/definitions/193.html)

## What Gets Flagged

```c
// FLAGGED: <= n writes buf[n] which is out of bounds
for (int i = 0; i <= n; i++) buf[i] = data[i];

// FLAGGED: no room for null terminator
char *copy = malloc(strlen(src));  // needs +1
```

## Remediation

```c
// SAFE: use < for zero-indexed loops
for (int i = 0; i < n; i++) buf[i] = data[i];

// SAFE: allocate +1 for null terminator
char *copy = malloc(strlen(src) + 1);
strcpy(copy, src);
```
