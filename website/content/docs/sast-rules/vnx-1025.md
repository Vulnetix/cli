---
title: "VNX-1025 – Comparison Using Wrong Factors"
description: "Detect security comparisons using reference equality or timing-unsafe equality that can be bypassed."
---

## Overview

Comparing security values (passwords, tokens, hashes, MACs) using the wrong method leads to exploitable weaknesses. Java's `==` on Strings compares object references, not values. Python's `==` on hash digests is susceptible to timing attacks. Only constant-time comparison prevents timing side-channel attacks.

**Severity:** Low | **CWE:** [CWE-1025 – Comparison Using Wrong Factors](https://cwe.mitre.org/data/definitions/1025.html)

## Why This Matters

Timing attacks on password comparison can leak whether a submitted value matches a stored hash by measuring how long the comparison takes. Short-circuit equality returns early when the first differing byte is found, leaking positional information.

## What Gets Flagged

```java
// FLAGGED: String == comparison (reference, not value)
if (password == expectedPassword) { ... }
```

```python
# FLAGGED: Timing-unsafe comparison
if token == stored_token: ...
if digest == expected_digest: ...
```

## Remediation

Use constant-time comparison for all security-sensitive values:

```python
# SAFE
import hmac
if hmac.compare_digest(token.encode(), stored_token.encode()):
    ...
```

```java
// SAFE: MessageDigest.isEqual is constant-time
if (MessageDigest.isEqual(submittedHash, storedHash)) { ... }
```
