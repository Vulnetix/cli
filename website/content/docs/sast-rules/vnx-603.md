---
title: "VNX-603 – Use of Client-Side Authentication"
description: "Detect authentication logic implemented in localStorage/sessionStorage that can be forged by any user."
---

## Overview

Authentication implemented entirely in client-side JavaScript — checking `localStorage.getItem('authenticated') === 'true'` — provides no security. Any user can set this value in browser developer tools in seconds. Authentication state must be validated server-side on every request.

**Severity:** High | **CWE:** [CWE-603 – Use of Client-Side Authentication](https://cwe.mitre.org/data/definitions/603.html)

## Why This Matters

`localStorage` and `sessionStorage` are writable by any JavaScript running in the page, including by the user themselves via developer tools. Basing authentication on these values means anyone can authenticate as anyone else.

## What Gets Flagged

```javascript
// FLAGGED: Authentication from localStorage
if (localStorage.getItem('authenticated') === 'true') {
    loadUserDashboard();
}

// FLAGGED: Role from sessionStorage
if (sessionStorage.role === 'admin') {
    showAdminMenu();
}
```

## Remediation

Validate authentication server-side on every request:

```javascript
// SAFE: Server validates the session token
async function loadDashboard() {
    const resp = await fetch('/api/me', { credentials: 'include' });
    if (!resp.ok) {
        window.location = '/login';
        return;
    }
    const user = await resp.json();
    renderDashboard(user);
}
```
