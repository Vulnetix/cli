---
title: "VNX-822 – Untrusted Pointer Dereference"
description: "Detect C code that casts network-received buffers directly to struct pointers or dereferences pointer-to-pointer values from external sources without validation."
---

## Overview

This rule flags C code that receives data from an untrusted source (via `recv()`, `read()`, sockets) and immediately casts the buffer to a struct pointer, or that dereferences a pointer-to-pointer (`**`) from an external source. Attackers can craft malicious input that places arbitrary values in the received buffer, causing the cast pointer to point anywhere in memory. This maps to [CWE-822: Untrusted Pointer Dereference](https://cwe.mitre.org/data/definitions/822.html).

**Severity:** Critical | **CWE:** [CWE-822](https://cwe.mitre.org/data/definitions/822.html)

## Why This Matters

Casting a network buffer directly to a struct pointer and then accessing its fields dereferences attacker-controlled memory addresses. If the struct contains function pointers, the attacker achieves arbitrary code execution. If it contains data pointers, they can read or write arbitrary memory. This class of vulnerability has been exploited in numerous high-severity CVEs in network daemons and protocol parsers.

## What Gets Flagged

```c
// FLAGGED: casting recv buffer directly to struct pointer
char buf[1024];
recv(sock, buf, sizeof(buf), 0);
struct Command *cmd = (struct Command *)buf;  // attacker controls all fields
cmd->handler();  // function pointer from attacker-controlled data
```

```c
// FLAGGED: read into buffer then cast
char data[512];
read(fd, data, sizeof(data));
struct Header *hdr = (struct Header *)data;
process(hdr->data_ptr);  // attacker-controlled pointer
```

```c
// FLAGGED: pointer-to-pointer parameter from external source
void process(char **pptr) {
    char *p = *pptr;  // dereferencing attacker-controlled pointer
    memcpy(dest, p, len);
}
```

## Remediation

```c
// SAFE: validate buffer contents before interpreting as struct
struct Command cmd;
char buf[sizeof(cmd)];
ssize_t n = recv(sock, buf, sizeof(buf), 0);
if (n != sizeof(cmd)) {
    // Reject malformed message
    return -1;
}
memcpy(&cmd, buf, sizeof(cmd));

// SAFE: validate each field individually
if (cmd.type > MAX_COMMAND_TYPE) {
    return -1;  // reject unknown command type
}
if (cmd.length > MAX_DATA_LENGTH) {
    return -1;  // reject oversized data
}
// Now safe to process cmd
```

```c
// SAFE: use a deserializer / parser instead of direct cast
// Use a library like Protocol Buffers, MessagePack, or a hand-written parser
// that validates all fields before creating the struct
struct Command *parse_command(const char *buf, size_t len) {
    if (len < MIN_COMMAND_SIZE) return NULL;
    struct Command *cmd = malloc(sizeof(*cmd));
    // Deserialize and validate each field from buf
    uint16_t type;
    memcpy(&type, buf, sizeof(type));
    cmd->type = ntohs(type);  // network-to-host byte order
    if (cmd->type > MAX_COMMAND_TYPE) {
        free(cmd);
        return NULL;
    }
    // ... validate remaining fields
    return cmd;
}
```

Never cast received buffers directly to struct pointers. Always deserialize with explicit field-by-field parsing and validation.
