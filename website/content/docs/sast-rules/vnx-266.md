## Overview

VNX-266 maps to [CWE-266: Incorrect Privilege Assignment](https://cwe.mitre.org/data/definitions/266.html). Detects source patterns associated with CWE-266 (Incorrect Privilege Assignment). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-266](https://cwe.mitre.org/data/definitions/266.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Incorrect Privilege Assignment) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-266 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'setuid(0)' pattern
setuid(0)
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-266 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-266` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
