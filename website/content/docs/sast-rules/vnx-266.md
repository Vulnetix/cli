---
title: "VNX-266 – Incorrect Privilege Assignment"
description: "Detect chmod() calls setting world-writable (0777) or world-readable/writable (0666) permissions on sensitive files."
---

## Overview

Setting permissions to 0777 or 0666 on files grants write (and execute) access to all system users. Sensitive files like configuration, credentials, private keys, or scripts should never be world-writable, as any local user can modify or replace them. This maps to [CWE-266: Incorrect Privilege Assignment](https://cwe.mitre.org/data/definitions/266.html).

**Severity:** High | **CWE:** [CWE-266](https://cwe.mitre.org/data/definitions/266.html)

## Remediation

| File Type | Recommended Mode |
|-----------|-----------------|
| Private key / credential | 0600 (owner read/write only) |
| Config with secrets | 0640 (owner rw, group r) |
| Regular script | 0755 (owner rwx, others rx) |
| Log files | 0644 |

```python
# SAFE: restrictive permissions
os.chmod('/etc/app/config.ini', 0o640)
```
