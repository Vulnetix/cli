## Overview

Directly converting user input to numeric types without range validation leads to unexpected values being used in calculations, array accesses, or allocations. Negative numbers, values exceeding `INT_MAX`, or non-numeric strings cause crashes or logic errors. This maps to [CWE-229: Improper Handling of Values](https://cwe.mitre.org/data/definitions/229.html).

**Severity:** Low | **CWE:** [CWE-229](https://cwe.mitre.org/data/definitions/229.html)

## Remediation

```python
# SAFE: bounded conversion
try:
    count = int(request.args.get('count', 0))
    if count < 0 or count > 1000:
        raise ValueError
except (ValueError, TypeError):
    abort(400)
```
