## Overview

VNX-1086 maps to [CWE-1086: Class with Excessive Number of Child Classes](https://cwe.mitre.org/data/definitions/1086.html). Detects source patterns associated with CWE-1086 (Class with Excessive Number of Child Classes). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1086](https://cwe.mitre.org/data/definitions/1086.html) | **Languages:** python

## Why This Matters

This weakness class (Class with Excessive Number of Child Classes) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1086 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'def ' pattern
def 
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1086 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1086` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
