## Overview

Detects patterns indicating a single class is subclassed excessively, suggesting it acts as a god class. Classes with too many children are often the result of poor abstraction and make it hard to reason about which subclass handles which security responsibility.

**Severity:** Low | **CWE:** [CWE-1086](https://cwe.mitre.org/data/definitions/1086.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1086](https://cwe.mitre.org/data/definitions/1086.html) for authoritative guidance.
