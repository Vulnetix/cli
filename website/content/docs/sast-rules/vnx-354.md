## Overview

VNX-354 maps to [CWE-354: Improper Validation of Integrity Check Value](https://cwe.mitre.org/data/definitions/354.html). Detects source patterns associated with CWE-354 (Improper Validation of Integrity Check Value). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-354](https://cwe.mitre.org/data/definitions/354.html) | **Languages:** java, python

## Why This Matters

This weakness class (Improper Validation of Integrity Check Value) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-354 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'MessageDigest.getInstance' pattern
MessageDigest.getInstance
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-354 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-354` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
