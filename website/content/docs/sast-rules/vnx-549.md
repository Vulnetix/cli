---
title: "VNX-549 – Missing Password Field Masking"
description: "Detect HTML password fields that use type='text' or have autocomplete='on', exposing passwords in plaintext."
---

## Overview

HTML `<input>` fields for passwords must use `type="password"` to enable browser masking. Using `type="text"` displays the password in plaintext. Setting `autocomplete="on"` stores the password in browser history accessible on shared or compromised devices.

**Severity:** Medium | **CWE:** [CWE-549 – Missing Password Field Masking](https://cwe.mitre.org/data/definitions/549.html)

## Why This Matters

A password displayed in cleartext is visible to anyone who can see the screen (shoulder surfing, screen recording, shared monitors). Browser-stored passwords are accessible to other users of the same browser profile and to malicious browser extensions.

## What Gets Flagged

```html
<!-- FLAGGED: type=text for password field -->
<input type="text" name="password" />

<!-- FLAGGED: autocomplete on for password -->
<input type="password" name="password" autocomplete="on" />
```

## Remediation

Use the correct input type and autocomplete attributes:

```html
<!-- SAFE: Masked password field -->
<input type="password" name="password" autocomplete="current-password" />

<!-- SAFE: New password field -->
<input type="password" name="new_password" autocomplete="new-password" />
```

Use `autocomplete="current-password"` for login forms (allows password manager integration) and `autocomplete="new-password"` for registration/change forms.
