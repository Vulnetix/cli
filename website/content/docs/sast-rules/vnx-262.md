---
title: "VNX-262 – Not Using Password Aging"
description: "Detect user models that store passwords without a password_changed_at or expiry timestamp."
---

## Overview

Credentials that never expire remain valid indefinitely after compromise. Password aging — tracking when a password was last changed and enforcing periodic rotation — limits the attacker's window of opportunity. This maps to [CWE-262: Not Using Password Aging](https://cwe.mitre.org/data/definitions/262.html).

**Severity:** Low | **CWE:** [CWE-262](https://cwe.mitre.org/data/definitions/262.html)

## Remediation

Add a `password_changed_at` timestamp to user records and enforce rotation at login:

```python
if (datetime.now() - user.password_changed_at).days > 90:
    return redirect('/change-password')
```
