## Overview

VNX-1053 maps to [CWE-1053: Missing Documentation for Design](https://cwe.mitre.org/data/definitions/1053.html). Detects source patterns associated with CWE-1053 (Missing Documentation for Design). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1053](https://cwe.mitre.org/data/definitions/1053.html) | **Languages:** java, python

## Why This Matters

This weakness class (Missing Documentation for Design) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1053 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'TODO: design' pattern
TODO: design
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1053 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1053` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
