## Overview

Detects public API functions and security-relevant methods that lack documentation comments. Missing design documentation makes it harder to understand security assumptions, expected inputs, and trust boundaries, increasing the risk of misuse.

**Severity:** Low | **CWE:** [CWE-1053 – Missing Documentation for Design](https://cwe.mitre.org/data/definitions/1053.html)

## Why This Matters

Security functions without documentation are dangerous because callers cannot know what validation has already been performed, what inputs are trusted, and what the security guarantees are. A missing docstring on `authenticate()` can lead to callers bypassing it or calling it incorrectly.

## What Gets Flagged

```python
# FLAGGED: security function without docstring
def authenticate(username, password):
    return check_credentials(username, password)
```

## Remediation

```python
def authenticate(username: str, password: str) -> bool:
    """
    Verify user credentials against the database.

    Args:
        username: User-supplied username (untrusted input, validated here).
        password: User-supplied plaintext password (not logged, compared via bcrypt).

    Returns:
        True if credentials are valid, False otherwise.
        Never raises exceptions — catches all errors and returns False.
    """
    return check_credentials(username, password)
```
