title: "VNX-1098 – Data Element containing Pointer Item without Proper Copy Control Element"
description: "Detect C++ classes with raw pointer members that lack copy constructor/assignment operator (Rule of Three/Five violation)."
---

## Overview

Detects C++ classes that own raw pointer members but do not implement copy constructor, copy assignment, or use smart pointers. Without these controls, shallow copies share raw pointers, leading to double-free, use-after-free, and dangling pointer vulnerabilities.

**Severity:** Medium | **CWE:** [CWE-1098 – Data Element containing Pointer Item without Proper Copy Control Element](https://cwe.mitre.org/data/definitions/1098.html)

## Why This Matters

Double-free vulnerabilities are exploitable for heap corruption and arbitrary code execution. When a class containing a raw pointer is copied (passed by value, stored in a container), both copies point to the same memory. When the first is destroyed, the memory is freed. When the second is destroyed, the already-freed memory is freed again.

## What Gets Flagged

```cpp
// FLAGGED: raw pointer without copy controls, no smart pointer
class SessionStore {
    Session* sessions;  // raw pointer
    // no copy constructor
    // no operator=
    // no unique_ptr/shared_ptr
};
```

## Remediation

```cpp
// OPTION 1: Use smart pointers (preferred)
class SessionStore {
    std::unique_ptr<Session[]> sessions;
    // Rule of Five automatically handled
};

// OPTION 2: Implement Rule of Five explicitly
class SessionStore {
    Session* sessions;
public:
    SessionStore(const SessionStore& other);           // copy ctor
    SessionStore& operator=(const SessionStore& other); // copy assign
    SessionStore(SessionStore&& other) noexcept;       // move ctor
    SessionStore& operator=(SessionStore&& other) noexcept; // move assign
    ~SessionStore();
};
```
