---
title: "VNX-823 – Out-of-range Pointer Offset"
description: "Detect pointer arithmetic using user-controlled values from atoi/strtol/argv without preceding bounds validation."
---

## Overview

This rule flags C pointer arithmetic where the offset is derived from user-controlled input (`atoi()`, `strtol()`, `argv[]`) without a preceding bounds check. Adding an arbitrary integer offset to a pointer can place the resulting pointer anywhere in the address space, enabling arbitrary reads or writes. This maps to [CWE-823: Use of Out-of-range Pointer Offset](https://cwe.mitre.org/data/definitions/823.html).

**Severity:** High | **CWE:** [CWE-823](https://cwe.mitre.org/data/definitions/823.html)

## Why This Matters

Pointer arithmetic with unvalidated offsets is equivalent to an arbitrary memory access primitive. If an attacker controls the offset, they can read sensitive data from adjacent memory (information disclosure) or write to attacker-chosen memory locations (code execution). Even a read-only out-of-bounds access can crash the process.

## What Gets Flagged

```c
// FLAGGED: pointer offset from atoi without bounds check
char buf[256];
int offset = atoi(user_input);
char *ptr = buf + offset;  // offset could be negative or beyond buf
memcpy(dest, ptr, len);

// FLAGGED: strtol offset without bounds check
long idx = strtol(argv[2], NULL, 10);
char *entry = table + idx;  // idx could be arbitrary
```

```c
// FLAGGED: array index from argv without validation
int i = atoi(argv[1]);
process(data[i]);  // i could be out of bounds
```

## Remediation

```c
// SAFE: validate offset before pointer arithmetic
#define BUF_SIZE 256
char buf[BUF_SIZE];
int offset = atoi(user_input);
if (offset < 0 || offset >= BUF_SIZE) {
    fprintf(stderr, "Invalid offset: %d\n", offset);
    return -1;
}
char *ptr = buf + offset;
```

```c
// SAFE: validate array index before access
long idx = strtol(argv[2], NULL, 10);
if (idx < 0 || idx >= TABLE_SIZE) {
    fprintf(stderr, "Index out of range: %ld\n", idx);
    return -1;
}
char *entry = table[idx];
```

```c
// SAFE: use size_t for indices, check before use
size_t i;
char *endptr;
long raw = strtol(argv[1], &endptr, 10);
if (*endptr != '\0' || raw < 0 || (size_t)raw >= data_len) {
    return -1;
}
i = (size_t)raw;
process(data[i]);
```

Always validate that an offset or index is within `[0, size)` before using it in pointer arithmetic or array access. Use `size_t` or `ptrdiff_t` with appropriate range checks.
