## Overview

This rule specifically targets the direct use of non-cryptographically-secure pseudo-random number generators (PRNGs). Unlike a CSPRNG, a standard PRNG uses a deterministic algorithm seeded with a small value (often a timestamp). An attacker who observes a few outputs can reconstruct the internal state and predict all future values. This maps to [CWE-338: Use of Cryptographically Weak Pseudo-Random Number Generator (PRNG)](https://cwe.mitre.org/data/definitions/338.html).

**Severity:** High | **CWE:** [CWE-338 – Use of Cryptographically Weak PRNG](https://cwe.mitre.org/data/definitions/338.html)

## Why This Matters

A PRNG is not random — it is a mathematical function that produces a deterministic sequence from a seed. Standard PRNGs (Mersenne Twister, linear congruential generators) are designed for statistical uniformity in simulations, games, and testing, not for security. Java's `java.util.Random` is a 48-bit LCG: after two calls to `nextInt()`, the seed can be fully recovered. Python's `random` module uses Mersenne Twister with a 19,937-bit state, but even this is recoverable from 624 consecutive 32-bit outputs. Go's `math/rand` package documentation explicitly warns: "Top-level functions, such as Float64 and Int, use a default shared Source that produces a deterministic sequence of values each time a program is run." If the seed is not set (or set from a weak source like the current time), the sequence is trivially reproducible.

## What Gets Flagged

- **Go**: `"math/rand"` import, `rand.New(rand.NewSource(`, `rand.Seed(`
- **Java**: `new Random(`, `new java.util.Random(`, `ThreadLocalRandom.current(`
- **JavaScript**: `Math.random(`
- **Python**: `import random`, `from random import`, `random.seed(`, `random.getstate(`
- **PHP**: `srand(`, `mt_srand(`, `lcg_value(`
- **Ruby**: `Random.new(`, `srand(`

```go
// FLAGGED: math/rand seeded with time
import "math/rand"
rand.Seed(time.Now().UnixNano())
id := rand.Int63()
```

```python
# FLAGGED: random module imported
import random
token = random.getrandbits(128)
```

```java
// FLAGGED: java.util.Random
Random r = new Random(System.currentTimeMillis());
long sessionId = r.nextLong();
```

## Remediation

Replace every security-sensitive use of a weak PRNG with a CSPRNG.

```go
// SAFE: crypto/rand — reads from OS entropy pool
import (
    "crypto/rand"
    "math/big"
)

// Generate a random integer in [0, max)
func secureRandInt(max int64) (int64, error) {
    n, err := rand.Int(rand.Reader, big.NewInt(max))
    if err != nil {
        return 0, err
    }
    return n.Int64(), nil
}
```

```python
# SAFE: secrets module (Python 3.6+)
import secrets
token = secrets.token_bytes(32)
otp = secrets.randbelow(1000000)
```

```java
// SAFE: SecureRandom
import java.security.SecureRandom;
SecureRandom csprng = new SecureRandom(); // seeded from OS entropy
long sessionId = csprng.nextLong();
```

Keep weak PRNGs for non-security purposes (test data generation, simulations) but never mix them with security-sensitive code paths in the same module without clear demarcation.
