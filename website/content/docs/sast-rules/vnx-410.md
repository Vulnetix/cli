## Overview

VNX-410 maps to [CWE-410: Insufficient Resource Pool](https://cwe.mitre.org/data/definitions/410.html). Detects source patterns associated with CWE-410 (Insufficient Resource Pool). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-410](https://cwe.mitre.org/data/definitions/410.html) | **Languages:** java

## Why This Matters

This weakness class (Insufficient Resource Pool) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-410 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-410 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-410` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
