---
title: "VNX-1235 – Incorrect Use of Autoboxing and Unboxing for Performance"
description: "Detect Java autoboxing of wrapper types (Integer, Long, Double) in loop variables and accumulators, which causes unnecessary heap allocations and risks NullPointerException on unboxing."
---

## Overview

Java's autoboxing transparently converts between primitive types (`int`, `long`, `double`) and their wrapper classes (`Integer`, `Long`, `Double`). Using wrapper types in tight loops creates thousands of short-lived heap objects per second, increasing GC pressure and reducing throughput. Unboxing a null wrapper reference throws `NullPointerException`. **Severity:** Low | **CWE:** [CWE-1235 – Incorrect Use of Autoboxing and Unboxing for Performance](https://cwe.mitre.org/data/definitions/1235.html)

## Why This Matters

While not a security vulnerability in the traditional sense, CWE-1235 weaknesses can contribute to denial-of-service conditions under load when GC pauses become severe. They also introduce NullPointerException paths that can cause unexpected application crashes — which may be exploitable in contexts where the crash produces useful error output.

## What Gets Flagged

```java
// FLAGGED: Wrapper type in for-each loop variable
for (Integer i = 0; i < list.size(); i++) {
    process(list.get(i));
}

// FLAGGED: Accumulator with wrapper type
Integer sum = 0;
for (int val : values) {
    sum += val;  // unboxes sum, adds val, reboxes result — every iteration
}

// FLAGGED: for-each over collection of Integer
for (Integer item : integerList) {
    total += item;  // unboxes item each iteration
}
```

## Remediation

```java
// SAFE: Use primitive types in loops
for (int i = 0; i < list.size(); i++) {
    process(list.get(i));
}

// SAFE: Primitive accumulator
int sum = 0;
for (int val : values) {
    sum += val;  // pure primitive arithmetic, no boxing
}

// SAFE: Use IntStream for sum
int sum = intStream.sum();

// SAFE: Use primitive collections (e.g. Eclipse Collections, Trove)
IntList intList = IntLists.mutable.of(1, 2, 3);
```
