## Overview

This rule detects security debt markers — comments that indicate deferred security implementation, suppressed analysis warnings, or acknowledged bypasses. CWE-1003 covers weaknesses used in simplified vulnerability mapping, and these markers often indicate the most commonly mapped vulnerability categories.

**Severity:** Info | **CWE:** [CWE-1003 – Weaknesses for Simplified Mapping](https://cwe.mitre.org/data/definitions/1003.html)

## Why This Matters

Security debt markers like `nosec`, `// nolint:gosec`, `TODO: validate`, and `HACK: bypass` represent acknowledged but unresolved security issues. Each represents a known gap in the security posture that attackers can exploit.

## What Gets Flagged

```python
# FLAGGED: suppressed check
except:
    pass  # nosec

# FLAGGED: deferred validation
def login(user, password):
    # TODO: validate input
    return True
```

## Remediation

1. Resolve all `TODO: security` and `FIXME: security` comments before release.
2. Never use `nosec` or `nolint:gosec` as permanent suppressions without documented justification.
3. Track security debt in a formal issue tracker with assigned owners.
