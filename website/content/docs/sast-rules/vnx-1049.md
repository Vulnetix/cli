## Overview

Detects `SELECT *` usage and other patterns that fetch all columns or all rows from large tables unnecessarily. These patterns cause excessive data transfer, prevent index optimisation, and can exhaust server resources.

**Severity:** Low | **CWE:** [CWE-1049 – Excessive Data Query Operations in a Large Data Table](https://cwe.mitre.org/data/definitions/1049.html)

## Why This Matters

`SELECT *` fetches all columns including sensitive ones (password hashes, PII) that may not be needed. On large tables it prevents index-only scans, increases memory pressure, and slows query execution — all exploitable for denial of service.

## What Gets Flagged

```sql
-- FLAGGED
SELECT * FROM users WHERE id = ?

-- FLAGGED
SELECT * FROM audit_log
```

## Remediation

```sql
-- SAFE: select only needed columns
SELECT id, username, email FROM users WHERE id = ?
```

Add appropriate indices for columns used in WHERE clauses.
