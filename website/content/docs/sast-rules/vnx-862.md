title: "VNX-862 – Missing Authorization"
description: "Detect view methods, API endpoints, and controller actions that may be missing authorization checks."
---

## Overview

Missing authorization occurs when the application does not verify that the currently authenticated user has permission to perform the requested action or access the requested resource. Any user (or even unauthenticated users) may access functionality intended for privileged roles.

**Severity:** High | **CWE:** [CWE-862 – Missing Authorization](https://cwe.mitre.org/data/definitions/862.html)

## Why This Matters

Broken access control is consistently the top-ranked OWASP risk. Missing authorization checks allow attackers to access admin functionality, read other users' data, or perform destructive operations.

## What Gets Flagged

```python
# FLAGGED: Django view without permission decorator
class UserDataView(View):
    def get(self, request):   # no @login_required or @permission_required
        return JsonResponse(User.objects.all().values())
```

```java
// FLAGGED: Spring endpoint without security annotation
@GetMapping("/admin/users")
public List<User> getAllUsers() { ... }
```

## Remediation

Apply authorization checks to every sensitive endpoint:

```python
# SAFE
@permission_required('app.view_userdata')
def get(self, request):
    ...
```

```java
// SAFE
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/admin/users")
public List<User> getAllUsers() { ... }
```
