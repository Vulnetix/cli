---
title: "VNX-565 – Reliance on Cookies without Validation"
description: "Detect authorization decisions based on unverified cookie values that clients can forge."
---

## Overview

Cookies are sent by the client and can contain any value the client chooses. Making access control decisions based on cookie values (e.g., `if (req.cookies.admin === 'true')`) without cryptographic signature verification allows any user to elevate their privileges by modifying the cookie.

**Severity:** High | **CWE:** [CWE-565 – Reliance on Cookies without Validation in Security Decision](https://cwe.mitre.org/data/definitions/565.html)

## Why This Matters

Cookie values can be edited in browser developer tools in seconds. Any cookie-based authorization check that does not verify a server-side signature (HMAC or similar) can be bypassed trivially.

## What Gets Flagged

```javascript
// FLAGGED: Auth decision from unsigned cookie
if (req.cookies.admin === 'true') {
    serveAdminPanel(req, res);
}
```

```php
// FLAGGED: PHP role from unverified cookie
if ($_COOKIE['role'] == 'admin') {
    showAdminPage();
}
```

```python
# FLAGGED: Django cookie without verification
if request.COOKIES.get('is_admin') == '1':
    return admin_view(request)
```

## Remediation

Use server-side session data or cryptographically signed tokens:

```javascript
// SAFE: Use session data (server-side)
if (req.session.isAdmin === true) {
    serveAdminPanel(req, res);
}
```
