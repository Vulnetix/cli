title: "VNX-605 – Multiple Binds to the Same Port"
description: "Detect socket bind() calls without SO_REUSEADDR that may cause multiple bind failures or port-stealing."
---

## Overview

Binding a socket to a port without setting `SO_REUSEADDR` causes `Address already in use` errors on restart. Incorrectly relying on `SO_REUSEPORT` without proper handling can allow another process to steal connections on the same port.

**Severity:** Low | **CWE:** [CWE-605 – Multiple Binds to the Same Port](https://cwe.mitre.org/data/definitions/605.html)

## Why This Matters

Services that cannot rebind their port on restart are vulnerable to downtime. More critically, `SO_REUSEPORT` without proper process ownership validation can allow a malicious process to receive traffic intended for a legitimate service.

## What Gets Flagged

```python
# FLAGGED: bind without SO_REUSEADDR
import socket
sock = socket.socket()
sock.bind(('', 8080))  # Will fail if port is in TIME_WAIT
```

```c
// FLAGGED: bind without setsockopt SO_REUSEADDR
int fd = socket(AF_INET, SOCK_STREAM, 0);
bind(fd, (struct sockaddr*)&addr, sizeof(addr));
```

## Remediation

Set `SO_REUSEADDR` before binding:

```python
# SAFE: Set SO_REUSEADDR before bind
sock = socket.socket()
sock.setsockopt(socket.SOL_SOCKET, socket.SO_REUSEADDR, 1)
sock.bind(('', 8080))
```
