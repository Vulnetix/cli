---
title: "VNX-624 – Executable Regular Expression (preg_replace /e modifier)"
description: "Detect PHP preg_replace() calls using the /e modifier which executes the replacement string as PHP code."
---

## Overview

PHP's `preg_replace()` function with the `/e` (eval) modifier evaluates the replacement string as PHP code after performing substitution. If user input controls the replacement string, this is a direct remote code execution (RCE) vulnerability. The `/e` modifier was deprecated in PHP 5.5 and removed in PHP 7.0.

**Severity:** Critical | **CWE:** [CWE-624 – Executable Regular Expression Error](https://cwe.mitre.org/data/definitions/624.html)

## Why This Matters

`preg_replace('/pattern/e', $user_input, $subject)` executes `$user_input` as PHP code. An attacker supplying `system('whoami')` achieves command execution. This is one of the most severe PHP vulnerability classes.

## What Gets Flagged

```php
// FLAGGED: /e modifier enables code execution
$result = preg_replace('/\[b\](.*)\[\/b\]/e', "'<b>' . strtoupper('\\1') . '</b>'", $input);

// FLAGGED: /e with user-controlled replacement
$result = preg_replace('/' . $pattern . '/e', $replacement, $subject);
```

## Remediation

Use `preg_replace_callback()` instead:

```php
// SAFE: Callback function, not eval
$result = preg_replace_callback('/\[b\](.*)\[\/b\]/', function($matches) {
    return '<b>' . htmlspecialchars(strtoupper($matches[1])) . '</b>';
}, $input);
```

Upgrade to PHP 7.0+ where the `/e` modifier is completely removed.
