---
title: "VNX-183 – Permissive Allowlist"
description: "Detect overly broad regex patterns used for input validation that allow dangerous characters."
---

## Overview

An allowlist is only as good as its specificity. Patterns like `.*`, `[a-zA-Z0-9\s\S]`, or `[[:print:]]` permit all printable characters including HTML metacharacters and SQL special characters. This creates a false sense of security. This maps to [CWE-183: Permissive List of Allowed Inputs](https://cwe.mitre.org/data/definitions/183.html).

**Severity:** Medium | **CWE:** [CWE-183](https://cwe.mitre.org/data/definitions/183.html)

## Remediation

Replace broad patterns with specific ones appropriate to the field context:

```python
# SAFE: strict username validation
import re
if not re.fullmatch(r'^[a-zA-Z0-9_-]{3,32}$', username):
    raise ValueError("Invalid username")
```
