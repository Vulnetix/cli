---
title: "VNX-668 – Exposure of Resource to Wrong Sphere"
description: "Detect sensitive files written to world-readable locations, temporary files with predictable names, and sensitive data in public directories."
---

## Overview

Sensitive data (credentials, private keys, session data, PII) written to world-readable directories, publicly accessible web directories, or predictably named temporary files can be read by unauthorized users or web clients. The resource is "exposed to the wrong sphere" — it crosses a trust boundary and becomes accessible to principals who should not have access.

**Severity:** Medium | **CWE:** [CWE-668 – Exposure of Resource to Wrong Sphere](https://cwe.mitre.org/data/definitions/668.html)

## Why This Matters

A private key written to `/tmp/key_12345.pem` with `tempfile.mktemp()` (which creates predictably guessable names) can be read by other processes or users on the same system. Sensitive data written to `public/`, `static/`, or `MEDIA_ROOT` becomes directly downloadable by web clients. These exposures often lead to credential theft and full system compromise.

## What Gets Flagged

```python
# FLAGGED: World-readable permissions on sensitive file
import os
os.chmod('/var/app/private_key.pem', 0o644)  # group/world readable

# FLAGGED: Sensitive data written to public directory
with open('/var/www/static/backup.sql', 'w') as f:
    f.write(database_dump)

# FLAGGED: Predictable temporary file (race condition + enumerable)
import tempfile
filename = tempfile.mktemp(suffix='.key')  # deprecated, insecure
```

```python
# FLAGGED: Writing credentials to /tmp
with open('/tmp/secret_token', 'w') as f:
    f.write(api_token)
```

## Remediation

Use private directories, secure temporary file creation, and appropriate permissions:

```python
# SAFE: Secure temporary files (not guessable, auto-deleted)
import tempfile
with tempfile.NamedTemporaryFile(mode='w', suffix='.key', delete=True) as f:
    f.write(private_key_data)
    process_key(f.name)
# File is deleted when the with block exits

# SAFE: Private permissions for sensitive files
os.chmod('/var/app/private_key.pem', 0o600)  # owner read/write only

# SAFE: Write sensitive data outside web root
PRIVATE_STORAGE = '/var/app/private/'
with open(os.path.join(PRIVATE_STORAGE, 'key.pem'), 'w') as f:
    f.write(private_key_data)
```
