## Overview

VNX-468 maps to [CWE-468: Incorrect Pointer Scaling](https://cwe.mitre.org/data/definitions/468.html). Detects source patterns associated with CWE-468 (Incorrect Pointer Scaling). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-468](https://cwe.mitre.org/data/definitions/468.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Incorrect Pointer Scaling) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-468 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'sizeof(' pattern
sizeof(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-468 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-468` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
