## Overview

Detects utility functions that are called across multiple architectural layers (controller, service, repository) without proper abstractions. Reusing the same low-level function across layers bypasses layer-specific security controls like input validation (controller layer) and authorisation (service layer).

**Severity:** Low | **CWE:** [CWE-1092](https://cwe.mitre.org/data/definitions/1092.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1092](https://cwe.mitre.org/data/definitions/1092.html) for authoritative guidance.
