## Overview

Detects assumptions about platform-specific data type sizes (e.g., sizeof(int)==4, sizeof(long)==8) without static assertions. Code that assumes specific integer widths can cause integer overflow or truncation vulnerabilities on platforms with different word sizes.

**Severity:** Low | **CWE:** [CWE-1102](https://cwe.mitre.org/data/definitions/1102.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1102](https://cwe.mitre.org/data/definitions/1102.html) for authoritative guidance.
