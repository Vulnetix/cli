---
title: "VNX-917 – Expression Language (EL) Injection"
description: "Detect user-controlled data being embedded into Expression Language contexts (SpEL, JSTL, Thymeleaf, Jinja2) where it will be evaluated, enabling arbitrary code or object graph traversal."
---

## Overview

Expression Language injection occurs when untrusted input is embedded into an EL or template expression that is subsequently evaluated by the runtime. The expression engine has access to the application context, beans, class loaders, and sometimes reflection APIs, enabling attackers to execute arbitrary methods, read environment variables, traverse the file system, and achieve remote code execution. This maps to [CWE-917: Improper Neutralization of Special Elements used in an Expression Language Statement](https://cwe.mitre.org/data/definitions/917.html).

**Severity:** High | **CWE:** [CWE-917 – EL Injection](https://cwe.mitre.org/data/definitions/917.html)

## Why This Matters

EL injection vulnerabilities are particularly insidious because the code paths are often in template rendering logic rather than business logic, and the injection point may be several calls removed from the user input. Spring's SpEL has been exploited in numerous CVEs against Spring MVC and Spring Data REST (e.g., CVE-2018-1270, CVE-2022-22963) to achieve RCE on public-facing applications. Jinja2 Server-Side Template Injection (SSTI) is one of the most commonly exploited vulnerability classes in Python web applications (Django, Flask), with payloads as simple as `{{config.items()}}` exposing application secrets and `{{''.__class__.__mro__[1].__subclasses__()}}` enabling RCE. MITRE ATT&CK T1059 covers execution of attacker-supplied code via scripting interpreters.

## What Gets Flagged

```python
# FLAGGED: render_template_string with user input enables Jinja2 SSTI
from flask import render_template_string, request

@app.route('/greet')
def greet():
    name = request.args.get('name')
    # Attacker sends: name={{config.SECRET_KEY}} or RCE payload
    return render_template_string(f"Hello {name}!")

# FLAGGED: Jinja2.Template with user input
from jinja2 import Template
tmpl = Template(user_input)  # user controls the template itself
```

```java
// FLAGGED: SpEL parseExpression with user input
import org.springframework.expression.spel.standard.SpelExpressionParser;

SpelExpressionParser parser = new SpelExpressionParser();
// userInput = "T(java.lang.Runtime).getRuntime().exec('id')"
Expression expr = parser.parseExpression(userInput);
Object result = expr.getValue();

// FLAGGED: ExpressionParser with request parameter
String expression = request.getParameter("filter");
parser.parseExpression(expression).getValue(context);
```

```html
<!-- FLAGGED: Thymeleaf preprocessing with user-controlled value -->
<!-- Attacker injects: __${T(java.lang.Runtime).getRuntime().exec('id')}__ -->
<p th:text="${param.message}">Message here</p>
<!-- More dangerous: unescaped output -->
<p th:utext="${userMessage}"></p>

<!-- FLAGGED: Direct param use in href -->
<a th:href="${param.returnUrl}">Back</a>
```

```jsp
<%-- FLAGGED: JSTL EL with param directly in output without escaping --%>
<c:out value="${param.search}" escapeXml="false"/>
```

## Remediation

### Python / Jinja2

```python
# SAFE: Pass user data as template variables, never as template text
from flask import render_template, request

@app.route('/greet')
def greet():
    name = request.args.get('name', '')
    # The template is a static file — user controls only the value
    return render_template('greet.html', name=name)
```

`greet.html`:
```html
<!-- SAFE: Jinja2 auto-escapes values in HTML context -->
<p>Hello {{ name }}!</p>
```

```python
# SAFE: If dynamic templates are unavoidable, use a sandboxed environment
from jinja2.sandbox import SandboxedEnvironment

env = SandboxedEnvironment()
# Even in sandbox, prefer static templates. If the template itself must be user-supplied:
try:
    tmpl = env.from_string(user_template)
    result = tmpl.render(data=safe_data_dict)
except Exception:
    result = "Invalid template"
```

### Java / Spring SpEL

```java
// SAFE: Use SimpleEvaluationContext — restricts to property access only
import org.springframework.expression.EvaluationContext;
import org.springframework.expression.spel.support.SimpleEvaluationContext;

EvaluationContext context = SimpleEvaluationContext
    .forReadOnlyDataBinding()
    .withInstanceMethods()
    .build();

// SimpleEvaluationContext blocks method invocation, type access, and constructors
Expression expr = parser.parseExpression(userExpression);
Object result = expr.getValue(context, myBean);

// SAFER: Validate expression against allowlist before parsing
private static final Pattern SAFE_EXPRESSION = Pattern.compile("^[a-zA-Z0-9_.\\[\\]]+$");

if (!SAFE_EXPRESSION.matcher(userExpression).matches()) {
    throw new IllegalArgumentException("Invalid expression");
}
```

### Thymeleaf

```html
<!-- SAFE: Always use th:text (escapes HTML), never th:utext with user data -->
<p th:text="${userMessage}">Default</p>

<!-- SAFE: Never use Thymeleaf preprocessing (__${...}__) with user data -->
<!-- SAFE: For redirect URLs, validate on the server side -->
```

### JSP / JSTL

```jsp
<%-- SAFE: Always use escapeXml="true" (the default) --%>
<c:out value="${param.search}" escapeXml="true"/>

<%-- SAFE: Use fn:escapeXml for inline expressions --%>
${fn:escapeXml(param.search)}
```
