## Overview

A time-of-check to time-of-use (TOCTOU) race condition occurs when a program checks a property of a resource and then acts on it in a separate, non-atomic operation. Between the check and the use, another thread or process can change the resource, causing the program to operate on a different state than it verified. The classic form involves filesystem operations: checking whether a file exists or is accessible, then opening it — an attacker can swap the file for a symlink to a privileged target in the window between the two calls.

**Severity:** Medium | **CWE:** [CWE-362 – Concurrent Execution using Shared Resource with Improper Synchronization ('Race Condition')](https://cwe.mitre.org/data/definitions/362.html)

## Why This Matters

TOCTOU vulnerabilities are the basis of many local privilege escalation exploits. The `access()` + `open()` pattern in setuid programs has been exploited for decades. In Python web applications, checking `os.path.exists()` before opening a file allows a symlink race that can redirect file writes to arbitrary paths. In Java, `File.exists()` before `new FileOutputStream()` has the same problem. On modern multi-core systems the race window is smaller but the attack is still reliable with appropriate timing, especially in container environments where filesystem operations are slower. MITRE ATT&CK T1548 covers exploitation of TOCTOU for privilege escalation.

## What Gets Flagged

```c
/* FLAGGED: access() check followed by open() — race window between them */
if (access(filename, R_OK) == 0) {
    fd = open(filename, O_RDONLY);  // file may have been replaced
}
```

```python
# FLAGGED: existence check followed by open — symlink race possible
if os.path.exists(path):
    with open(path) as f:
        data = f.read()

# FLAGGED: isfile check is not atomic with subsequent open
if os.path.isfile(config_path):
    config = open(config_path).read()
```

```java
// FLAGGED: exists() is not atomic with new FileOutputStream
File f = new File(path);
if (f.exists()) {
    FileOutputStream out = new FileOutputStream(f);
}
```

## Remediation

Eliminate the check-then-act pattern by acting directly and handling the error:

```c
/* SAFE: open directly, handle ENOENT/EACCES rather than pre-checking */
int fd = open(filename, O_RDONLY | O_NOFOLLOW);
if (fd == -1) {
    perror("open");
    return -1;
}

/* SAFE: create exclusively — O_CREAT|O_EXCL is atomic */
int fd2 = open(tmpfile, O_WRONLY | O_CREAT | O_EXCL, 0600);
if (fd2 == -1 && errno == EEXIST) { /* file already exists */ }
```

```python
# SAFE: EAFP (Easier to Ask Forgiveness than Permission) pattern
try:
    with open(path) as f:
        data = f.read()
except FileNotFoundError:
    data = None
except PermissionError:
    raise
```

```java
// SAFE: use atomic NIO operations
try {
    Path p = Paths.get(path);
    // CREATE_NEW is atomic — fails if file already exists
    OutputStream out = Files.newOutputStream(p, StandardOpenOption.CREATE_NEW);
} catch (FileAlreadyExistsException e) {
    // handle
}
```
