description: "Detect direct database queries in controller/view layer files that bypass proper layer separation."
---

## Overview

Mixing presentation, business logic, and data access in a single layer creates security risks: SQL injection is more likely when raw queries appear in controllers, and security audits are harder when there are no clear layer boundaries.

**Severity:** Low | **CWE:** [CWE-1044 – Architecture with Number of Horizontal Layers Outside of Expected Range](https://cwe.mitre.org/data/definitions/1044.html)

## Why This Matters

When a controller directly executes raw SQL, there is no data access layer to enforce parameterisation, and developers are more likely to build queries via string concatenation with user input.

## What Gets Flagged

```python
# FLAGGED: Raw SQL in controller file
class UserController(View):
    def get(self, request):
        query = "SELECT * FROM users WHERE id=" + request.GET['id']
```

## Remediation

Move data access to a dedicated repository/DAO layer that enforces parameterised queries.
