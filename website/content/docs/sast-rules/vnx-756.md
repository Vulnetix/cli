---
title: "VNX-756 – Missing Custom Error Page"
description: "Detect debug modes and default error handlers that expose stack traces, server versions, and internal paths to clients."
---

## Overview

This rule flags debug mode settings (`DEBUG = True`, `debug=True`, `display_errors = On`) and default error handlers that send full stack traces to clients. In production, these expose server technology, file paths, database queries, and internal logic to any user who triggers an error. This maps to [CWE-756: Missing Custom Error Page](https://cwe.mitre.org/data/definitions/756.html).

**Severity:** Medium | **CWE:** [CWE-756](https://cwe.mitre.org/data/definitions/756.html)

## Why This Matters

Stack traces reveal framework versions, file paths, library names, database query structures, and sometimes sensitive configuration. This information directly assists attackers in identifying exploitable vulnerabilities. Debug modes also often enable additional attack surfaces such as interactive debuggers (Werkzeug debugger allows remote code execution).

## What Gets Flagged

```python
# FLAGGED: Django debug mode enabled (settings.py)
DEBUG = True

# FLAGGED: Flask running with debug=True
app.run(host='0.0.0.0', debug=True)
```

```javascript
// FLAGGED: Express default error handler exposing stack traces
app.use(errorHandler())
```

```php
// FLAGGED: PHP displaying errors to browser
ini_set('display_errors', 1);
error_reporting(E_ALL);
```

```ruby
# FLAGGED: Rails showing exceptions in browser in production
config.consider_all_requests_local = true
```

## Remediation

```python
# SAFE: Django — use environment variable to control debug
import os
DEBUG = os.environ.get('DJANGO_DEBUG', 'False') == 'True'

# SAFE: Flask — never pass debug=True in production
app.run(host='0.0.0.0', port=5000)
# Use FLASK_DEBUG=0 environment variable instead
```

```javascript
// SAFE: Express custom error handler
app.use((err, req, res, next) => {
    const statusCode = err.statusCode || 500;
    logger.error({ err, requestId: req.id });
    res.status(statusCode).json({
        error: 'An unexpected error occurred',
        requestId: req.id,
    });
});
```

```php
// SAFE: PHP — log errors, don't display them
ini_set('display_errors', 0);
ini_set('log_errors', 1);
ini_set('error_log', '/var/log/php_errors.log');
```
