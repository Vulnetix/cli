---
title: "VNX-915 – Mass Assignment / Improperly Controlled Object Attribute Modification"
description: "Detect patterns where unfiltered user request data is applied directly to model objects, enabling attackers to set privileged attributes such as admin flags or ownership fields."
---

## Overview

Mass assignment vulnerabilities occur when a web framework automatically binds request parameters to model object attributes, and the application passes user-controlled data to a bulk-update method without explicitly restricting which attributes may be set. Attackers can set attributes that should never be user-controlled, such as `is_admin`, `role`, `owner_id`, or `balance`. This maps to [CWE-915: Improperly Controlled Modification of Dynamically-Determined Object Attributes](https://cwe.mitre.org/data/definitions/915.html).

**Severity:** High | **CWE:** [CWE-915 – Improperly Controlled Modification of Dynamically-Determined Object Attributes](https://cwe.mitre.org/data/definitions/915.html)

## Why This Matters

The GitHub mass assignment vulnerability (2012) allowed any user to add their SSH key to any repository by exploiting Rails' `update_attributes` with unfiltered params, effectively granting write access to any public repository. This led to the `attr_accessible` (now Strong Parameters) pattern in Rails. Despite this high-profile incident, mass assignment vulnerabilities continue to appear in Express/Mongoose applications using `Object.assign(model, req.body)` and in PHP Laravel applications using `fill($request->all())` without `$fillable`. MITRE ATT&CK T1565 covers data manipulation, which privilege escalation through mass assignment enables.

## What Gets Flagged

```ruby
# FLAGGED: permit! allows all params — no restriction at all
def update
  @user.update_attributes(params.require(:user).permit!)
end

# FLAGGED: update_attributes with unfiltered params hash
@post.update_attributes(params)
```

```python
# FLAGGED: Django ModelForm saving all fields from POST without restriction
class UserForm(forms.ModelForm):
    class Meta:
        model = User
        fields = '__all__'  # allows setting is_staff, is_superuser, etc.

def update_user(request):
    form = UserForm(request.POST, instance=request.user)
    if form.is_valid():
        form.save()
```

```javascript
// FLAGGED: Object.assign with entire req.body — sets any field in the body
async function updateUser(req, res) {
    const user = await User.findById(req.params.id);
    Object.assign(user, req.body);  // attacker can set role, isAdmin, etc.
    await user.save();
}

// FLAGGED: lodash merge/extend with req.body
_.merge(model, req.body);
```

```php
<?php
// FLAGGED: fill() with all request data — bypasses $fillable if $guarded is empty
$user->fill($request->all());
$user->save();
```

## Remediation

### Ruby on Rails

```ruby
# SAFE: Explicit allowlist of permitted parameters (Strong Parameters)
def user_params
  params.require(:user).permit(:name, :email, :bio)
  # Never include: :is_admin, :role, :balance
end

def update
  @user.update(user_params)
end
```

### Python/Django

```python
# SAFE: Explicit field list in ModelForm — never use fields = '__all__'
class UserUpdateForm(forms.ModelForm):
    class Meta:
        model = User
        fields = ['first_name', 'last_name', 'email', 'bio']
        # Explicitly excludes: is_staff, is_superuser, password, groups

# SAFE: Use serializer allowlist in DRF
class UserSerializer(serializers.ModelSerializer):
    class Meta:
        model = User
        fields = ['id', 'username', 'email', 'bio']
        read_only_fields = ['id', 'is_staff', 'date_joined']
```

### Node.js / Express

```javascript
// SAFE: Extract only the fields you want to update
const ALLOWED_FIELDS = ['name', 'email', 'bio', 'preferences'];

async function updateUser(req, res) {
    const user = await User.findById(req.params.id);
    const updates = {};
    for (const field of ALLOWED_FIELDS) {
        if (field in req.body) {
            updates[field] = req.body[field];
        }
    }
    Object.assign(user, updates);
    await user.save();
}

// SAFE: Pick only allowed fields using lodash
const updates = _.pick(req.body, ['name', 'email', 'bio']);
```

### PHP / Laravel

```php
<?php
// SAFE: Use $fillable in the model to declare mass-assignable fields
class User extends Model
{
    protected $fillable = ['name', 'email', 'bio'];
    // $guarded ensures is_admin, role, etc. are never mass-assigned
    protected $guarded  = ['is_admin', 'role', 'balance'];
}

// SAFE: Pass validated-and-filtered data explicitly
$user->fill($request->only(['name', 'email', 'bio']));
$user->save();
```
