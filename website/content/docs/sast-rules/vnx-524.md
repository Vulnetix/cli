---
title: "VNX-524 – Use of Cache Containing Sensitive Information"
description: "Detect cached responses for user-specific or sensitive endpoints that lack appropriate Cache-Control directives."
---

## Overview

Web caches (CDNs, reverse proxies, browser caches) store responses for reuse. When sensitive or user-specific responses are cached without `Cache-Control: no-store, private`, they may be served to different users or stored in shared caches accessible to attackers.

**Severity:** Medium | **CWE:** [CWE-524 – Use of Cache Containing Sensitive Information](https://cwe.mitre.org/data/definitions/524.html)

## Why This Matters

A shared cache serving one user's account page to another is a direct data breach. CDN caches can store authenticated responses for minutes and serve them to any requester. Browser caches can expose sensitive data on shared computers.

## What Gets Flagged

```python
# FLAGGED: Django cache_page on user-specific view
@cache_page(60 * 15)
def user_dashboard(request):
    return render(request, 'dashboard.html', {'user': request.user})
```

```javascript
// FLAGGED: Cache-Control without no-store
res.setHeader("Cache-Control", "public, max-age=3600");
```

## Remediation

Add appropriate cache directives for sensitive endpoints:

```python
# SAFE: Explicitly prevent caching on user-specific views
from django.views.decorators.cache import never_cache

@never_cache
def user_dashboard(request):
    ...
```

```javascript
// SAFE: Prevent caching of authenticated responses
res.setHeader("Cache-Control", "no-store, no-cache, must-revalidate, private");
```
