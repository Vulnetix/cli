## Overview

VNX-788 maps to [CWE-788: Access of Memory Location After End of Buffer](https://cwe.mitre.org/data/definitions/788.html). Detects source patterns associated with CWE-788 (Access of Memory Location After End of Buffer). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-788](https://cwe.mitre.org/data/definitions/788.html) | **Languages:** c

## Why This Matters

This weakness class (Access of Memory Location After End of Buffer) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-788 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'buf[size]' pattern
buf[size]
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-788 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-788` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
