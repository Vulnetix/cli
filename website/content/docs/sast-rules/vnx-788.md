description: "Detect C off-by-one errors, strcpy, sprintf, and gets calls that write past the end of a buffer."
---

## Overview

This rule flags C functions that write to buffers without length limits (`strcpy`, `sprintf`, `gets`) and common off-by-one patterns like accessing `buf[size]` when valid indices are 0..size-1. These cause stack or heap buffer overflows that have been exploited for remote code execution since the 1980s. This maps to [CWE-788: Access of Memory Location After End of Buffer](https://cwe.mitre.org/data/definitions/788.html).

**Severity:** High | **CWE:** [CWE-788](https://cwe.mitre.org/data/definitions/788.html)

## Why This Matters

Buffer overflows remain one of the most exploited vulnerability classes, responsible for countless CVEs including many critical severity ones. Stack buffer overflows overwrite return addresses; heap buffer overflows overwrite adjacent allocation metadata or objects.

## What Gets Flagged

```c
// FLAGGED: strcpy has no length limit
char dest[64];
strcpy(dest, user_input);  // overflows if input > 63 bytes

// FLAGGED: sprintf has no length limit
char buf[256];
sprintf(buf, "Hello, %s!", username);  // overflows for long names

// FLAGGED: gets reads until newline with no limit
char buf[256];
gets(buf);  // always dangerous, removed from C11

// FLAGGED: off-by-one index
char buf[10];
buf[10] = '\0';  // index 10 is out of bounds for a 10-element array
```

## Remediation

```c
// SAFE: strncpy with destination size
char dest[64];
strncpy(dest, user_input, sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';

// SAFE: snprintf with buffer size
char buf[256];
snprintf(buf, sizeof(buf), "Hello, %s!", username);

// SAFE: fgets instead of gets
char buf[256];
fgets(buf, sizeof(buf), stdin);
buf[strcspn(buf, "\n")] = '\0';  // remove trailing newline

// SAFE: correct off-by-one
char buf[10];
buf[9] = '\0';  // last valid index is 9
```
