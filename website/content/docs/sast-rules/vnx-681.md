---
title: "VNX-681 – Incorrect Conversion Between Numeric Types"
description: "Detect silent numeric truncation when converting float/double/long values to smaller integer types without range validation."
---

## Overview

Narrowing numeric type conversions (float to int, double to int, long to int) silently discard the fractional part or high-order bits. When the converted value is used for array indexing, buffer sizing, or security checks, the unexpected truncation can produce buffer overflows, integer wrap-around, or logic bypasses.

**Severity:** Medium | **CWE:** [CWE-681 – Incorrect Conversion Between Numeric Types](https://cwe.mitre.org/data/definitions/681.html)

## Why This Matters

A value like `2147483648L` cast to `int` becomes `-2147483648` due to overflow. When used as an allocation size or array bound, this can produce a zero-byte allocation or a negative index, leading to heap corruption.

## What Gets Flagged

```java
// FLAGGED: long to int truncation
int size = (int) longValue;
int count = (int)(double) someCalculation;
```

```python
# FLAGGED: float truncation used as index
idx = int(float_result / divisor)
arr[int(math.sqrt(n))]
```

```c
// FLAGGED: narrowing cast without range check
int len = (int)largeUnsignedValue;
```

## Remediation

Validate the value fits within the target range before casting:

```java
// SAFE: throw if overflow
int size = Math.toIntExact(longValue);

// SAFE: explicit bounds check
if (longValue < Integer.MIN_VALUE || longValue > Integer.MAX_VALUE) {
    throw new ArithmeticException("Overflow");
}
```
