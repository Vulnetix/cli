title: "VNX-1024 – Comparison of Incompatible Types"
description: "Detect type-unsafe loose equality comparisons (==) in security-sensitive PHP and JavaScript code."
---

## Overview

Detects type-unsafe comparisons that may produce unexpected results and bypass security checks. PHP loose equality (`==`) performs type juggling that can allow `'0e123'` to equal `'0'`, and JavaScript `==` coerces types in surprising ways.

**Severity:** Medium | **CWE:** [CWE-1024 – Comparison of Incompatible Types](https://cwe.mitre.org/data/definitions/1024.html)

## Why This Matters

PHP's type juggling is a well-known source of authentication bypass. The comparison `"0e123" == "0e456"` evaluates to `true` because both are interpreted as scientific notation (zero to a power). Hash values starting with `0e` can be compared as equal using `==`.

## What Gets Flagged

```php
<?php
// FLAGGED: loose comparison for password check
if ($hash == $storedHash) {
    // may authenticate with hash that starts with 0e
}
```

```javascript
// FLAGGED: loose comparison for token validation
if (token == storedToken) {
    // type coercion can cause unexpected matches
}
```

## Remediation

```php
// SAFE: strict equality
if ($hash === $storedHash) { ... }
```

```javascript
// SAFE: strict equality
if (token === storedToken) { ... }
```
