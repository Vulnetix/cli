title: "VNX-434 – Unrestricted Upload of File with Dangerous Type"
description: "Detect file upload handlers that lack file type, extension, or MIME type validation, enabling web shell uploads and remote code execution."
---

## Overview

This rule flags file upload handling code that does not visibly validate the uploaded file's type, extension, or MIME type. An attacker who can upload an arbitrary file to a web-accessible location can upload a web shell (a PHP, JSP, or ASP file containing `system()` or `exec()` calls) and then request it directly to execute arbitrary commands on the server. This maps to [CWE-434: Unrestricted Upload of File with Dangerous Type](https://cwe.mitre.org/data/definitions/434.html).

**Severity:** High | **CWE:** [CWE-434 – Unrestricted Upload of File with Dangerous Type](https://cwe.mitre.org/data/definitions/434.html)

## Why This Matters

Web shell uploads are one of the most commonly observed post-exploitation techniques in breach reports. Once a web shell is in place, attackers have persistent, authenticated RCE with the privileges of the web server process. This typically leads to lateral movement, credential harvesting, and data exfiltration. The `multer` middleware for Express is widely used without the `fileFilter` option, and PHP applications commonly use `move_uploaded_file` without checking the file type beyond the untrusted `$_FILES['type']` field, which is sent by the client and entirely controllable by the attacker. MITRE ATT&CK T1505.003 covers web shell deployment as a server software component attack.

## What Gets Flagged

```php
<?php
// FLAGGED: Using $_FILES without extension or MIME type validation
if ($_FILES['upload']['error'] === UPLOAD_ERR_OK) {
    move_uploaded_file(
        $_FILES['upload']['tmp_name'],
        '/var/www/uploads/' . $_FILES['upload']['name']
    );
}
```

```python
# FLAGGED: Django file upload without content type check
def upload_view(request):
    if request.method == 'POST':
        uploaded_file = request.FILES['document']
        # No content_type or name validation
        with open(f'/uploads/{uploaded_file.name}', 'wb') as f:
            for chunk in uploaded_file.chunks():
                f.write(chunk)
```

```java
// FLAGGED: MultipartFile without content type or extension check
@PostMapping("/upload")
public ResponseEntity<?> upload(@RequestParam("file") MultipartFile file) {
    String filename = file.getOriginalFilename();
    // No validation of filename extension or content type
    Path target = Paths.get("/uploads/" + filename);
    Files.copy(file.getInputStream(), target);
    return ResponseEntity.ok().build();
}
```

```javascript
// FLAGGED: multer with no fileFilter — accepts everything
const upload = multer({ dest: 'uploads/' });
app.post('/upload', upload.single('file'), (req, res) => {
    res.json({ path: req.file.path });
});

// FLAGGED: Manual req.files handling without validation
app.post('/attach', (req, res) => {
    const file = req.files.attachment;
    file.mv('/uploads/' + file.name);
});
```

```ruby
# FLAGGED: Active Storage permit without content type constraint
# (bare permit allows any attachment)
def update
  @user.update(params.require(:user).permit(:avatar))
end
```

## Remediation

### PHP

```php
<?php
// SAFE: Validate extension against allowlist and verify magic bytes
$ALLOWED_EXTENSIONS = ['jpg', 'jpeg', 'png', 'gif', 'pdf'];
$ALLOWED_MIME_TYPES  = ['image/jpeg', 'image/png', 'image/gif', 'application/pdf'];

function safe_upload(array $file, string $upload_dir): string {
    $ext = strtolower(pathinfo($file['name'], PATHINFO_EXTENSION));
    if (!in_array($ext, $ALLOWED_EXTENSIONS, true)) {
        throw new RuntimeException("File type .$ext is not allowed");
    }
    // Verify MIME type from file contents, not from the client-supplied type
    $finfo = new finfo(FILEINFO_MIME_TYPE);
    $mime  = $finfo->file($file['tmp_name']);
    if (!in_array($mime, $ALLOWED_MIME_TYPES, true)) {
        throw new RuntimeException("MIME type $mime is not allowed");
    }
    // Use a random name to prevent path traversal and name-guessing
    $safe_name = bin2hex(random_bytes(16)) . '.' . $ext;
    move_uploaded_file($file['tmp_name'], $upload_dir . $safe_name);
    return $safe_name;
}
```

### Python/Django

```python
# SAFE: Validate content type and extension before saving
ALLOWED_CONTENT_TYPES = {'image/jpeg', 'image/png', 'image/gif', 'application/pdf'}
ALLOWED_EXTENSIONS    = {'.jpg', '.jpeg', '.png', '.gif', '.pdf'}

import os, secrets
from django.core.exceptions import ValidationError

def upload_view(request):
    uploaded_file = request.FILES['document']
    ext = os.path.splitext(uploaded_file.name)[1].lower()

    if uploaded_file.content_type not in ALLOWED_CONTENT_TYPES:
        raise ValidationError("File type not allowed")
    if ext not in ALLOWED_EXTENSIONS:
        raise ValidationError("File extension not allowed")

    safe_name = secrets.token_hex(16) + ext
    with open(f'/uploads/{safe_name}', 'wb') as f:
        for chunk in uploaded_file.chunks():
            f.write(chunk)
```

### Java (Spring)

```java
// SAFE: Check content type and extension allowlist
private static final Set<String> ALLOWED_TYPES = Set.of("image/jpeg", "image/png", "application/pdf");
private static final Set<String> ALLOWED_EXTS  = Set.of(".jpg", ".jpeg", ".png", ".pdf");

@PostMapping("/upload")
public ResponseEntity<?> upload(@RequestParam("file") MultipartFile file) {
    String originalName = file.getOriginalFilename();
    String ext = originalName != null
        ? originalName.substring(originalName.lastIndexOf('.')).toLowerCase()
        : "";

    if (!ALLOWED_TYPES.contains(file.getContentType())) {
        return ResponseEntity.badRequest().body("Content type not allowed");
    }
    if (!ALLOWED_EXTS.contains(ext)) {
        return ResponseEntity.badRequest().body("Extension not allowed");
    }

    String safeName = UUID.randomUUID() + ext;
    Path target = Paths.get("/uploads/" + safeName);
    Files.copy(file.getInputStream(), target, StandardCopyOption.REPLACE_EXISTING);
    return ResponseEntity.ok(safeName);
}
```

### Node.js / Express

```javascript
// SAFE: multer with fileFilter and size limit
const ALLOWED_TYPES = new Set(['image/jpeg', 'image/png', 'application/pdf']);

const upload = multer({
    dest: 'uploads/',
    limits: { fileSize: 5 * 1024 * 1024 },  // 5 MB
    fileFilter(req, file, cb) {
        if (!ALLOWED_TYPES.has(file.mimetype)) {
            return cb(new Error(`File type ${file.mimetype} not allowed`));
        }
        cb(null, true);
    }
});
```
