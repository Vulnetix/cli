---
title: "VNX-1045 – Parent Class with Virtual Destructor and Child Without Virtual Destructor"
description: "Detect C++ classes with virtual methods that have non-virtual destructors, causing incomplete cleanup when deleted through a base pointer."
---

## Overview

In C++, deleting an object through a base-class pointer invokes the base destructor only when the destructor is not declared virtual. Derived class destructors are never called, leaving resources (file handles, heap memory, locks) unreleased. This can lead to resource exhaustion or use-after-free conditions in destructor chains.

**Severity:** Medium | **CWE:** [CWE-1045 – Parent Class with a Virtual Destructor and a Child Class without a Virtual Destructor](https://cwe.mitre.org/data/definitions/1045.html)

## Why This Matters

Resource leaks from missing virtual destructors are a classic C++ bug. When the leaked resources include file descriptors, network connections, or cryptographic material, the security implications include exhausting available connections and leaving sensitive data in memory.

## What Gets Flagged

```cpp
// FLAGGED: Base class destructor not virtual, but class has virtual methods
class Base {
public:
    virtual void process();
    ~Base();  // should be virtual
};

class Derived : public Base {
    SecretKey *key;  // never freed if deleted via Base*
    ~Derived() { delete key; }
};
```

## Remediation

Declare base class destructors virtual when the class is intended to be subclassed:

```cpp
// SAFE
class Base {
public:
    virtual void process();
    virtual ~Base() = default;  // virtual destructor
};
```
