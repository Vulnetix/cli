title: "VNX-186 – Overly Restrictive Regular Expression"
description: "Detect validation regexes that are anchored only at one end or reject valid inputs while missing attack vectors."
---

## Overview

A regex anchored with `^` but not `$` (or vice versa) validates only a prefix, allowing an attacker to append malicious content after the matched portion. A regex that is too narrow for the field also rejects legitimate users while not blocking all attack variants. This maps to [CWE-186: Overly Restrictive Regular Expression](https://cwe.mitre.org/data/definitions/186.html).

**Severity:** Low | **CWE:** [CWE-186](https://cwe.mitre.org/data/definitions/186.html)

## Remediation

```python
# SAFE: anchor both ends
import re
if not re.fullmatch(r'[a-zA-Z0-9_]{3,32}', username):
    abort(400)
```
