## Overview

VNX-824 maps to [CWE-824: Access of Uninitialized Pointer](https://cwe.mitre.org/data/definitions/824.html). Detects source patterns associated with CWE-824 (Access of Uninitialized Pointer). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-824](https://cwe.mitre.org/data/definitions/824.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Access of Uninitialized Pointer) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-824 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'char *p;' pattern
char *p;
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-824 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-824` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
