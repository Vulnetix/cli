---
title: "VNX-1046 – Creation of Immutable Text Using String Concatenation"
description: "Detect Java String concatenation in loops that creates many intermediate immutable objects potentially containing sensitive data."
---

## Overview

Java's `String` type is immutable. Each `+` concatenation creates a new `String` object and the intermediate strings persist in the JVM string pool. When security-sensitive values (passwords, tokens) are concatenated in loops, those values remain accessible in memory via heap dumps for much longer than intended.

**Severity:** Low | **CWE:** [CWE-1046 – Creation of Immutable Text Using String Concatenation](https://cwe.mitre.org/data/definitions/1046.html)

## Why This Matters

Beyond performance concerns, strings in the JVM string pool cannot be explicitly zeroed. If heap dumps are captured (for debugging, monitoring, or by an attacker via a memory-read vulnerability), sensitive data may be readable in multiple copies.

## What Gets Flagged

```java
// FLAGGED: String concatenation in loop
String result = "";
for (String part : parts) {
    result = result + part;  // many intermediate String objects
}
```

## Remediation

Use `StringBuilder` which can be cleared after use:

```java
// SAFE
StringBuilder sb = new StringBuilder();
for (String part : parts) {
    sb.append(part);
}
String result = sb.toString();
Arrays.fill(sb.toString().toCharArray(), '\0');  // attempt to clear
```

For passwords specifically, use `char[]` instead of `String`.
