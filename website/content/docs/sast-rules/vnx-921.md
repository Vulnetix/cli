---

## Overview

This rule flags code that stores sensitive information — passwords, tokens, session identifiers, PII — in locations that lack proper access control. World-readable temporary directories (`/tmp`), browser client-side storage (`localStorage`, `sessionStorage`), and Android `MODE_WORLD_READABLE` shared preferences are all accessible to other processes, users, browser extensions, or JavaScript running in the same origin. This maps to [CWE-921: Storage of Sensitive Data in a Mechanism without Access Control](https://cwe.mitre.org/data/definitions/921.html).

**Severity:** Medium | **CWE:** [CWE-921 – Storage of Sensitive Data in a Mechanism without Access Control](https://cwe.mitre.org/data/definitions/921.html)

## Why This Matters

On multi-user systems and containers, `/tmp` is world-readable and writable by default. Any process running as any user can read files left there. In containerized environments, files in shared temp directories survive process restarts and may be read by sidecars or logging agents. In browsers, `localStorage` is accessible to any JavaScript running in the same origin, including third-party scripts loaded via CDN or ad networks. Malicious browser extensions and XSS attacks can trivially exfiltrate `localStorage` contents. Android's `MODE_WORLD_READABLE` was deprecated in API 17 and can expose data to other apps on the device. MITRE ATT&CK T1552.001 covers credentials in files.

## What Gets Flagged

```javascript
// FLAGGED: Storing passwords or tokens in localStorage
localStorage.setItem('password', userPassword);
localStorage.setItem('token', authToken);
localStorage.setItem('secret', apiSecret);

// FLAGGED: sessionStorage with sensitive keys
sessionStorage.setItem('password', password);
```

```python
# FLAGGED: Writing sensitive data to /tmp
with open('/tmp/auth_token.txt', 'w') as f:
    f.write(auth_token)

with open('/tmp/user_credentials', 'w') as f:
    f.write(f'{username}:{password}')
```

```php
<?php
// FLAGGED: Writing credentials to /tmp
fopen('/tmp/session_data', 'w');
file_put_contents('/tmp/api_key.txt', $apiKey);
```

```ruby
# FLAGGED: Writing sensitive data to world-readable temp
File.write('/tmp/oauth_token', token)
```

```java
// FLAGGED: Android world-readable shared preferences
SharedPreferences prefs = getSharedPreferences("auth", MODE_WORLD_READABLE);
prefs.edit().putString("password", password).apply();
```

## Remediation

### Browser / JavaScript

```javascript
// SAFE: Store session tokens in httpOnly cookies (server-set, not accessible to JS)
// This is set server-side:
// Set-Cookie: session=<token>; HttpOnly; Secure; SameSite=Strict

// SAFE: If client-side storage is unavoidable, use sessionStorage
// (scoped to the tab, cleared on close) and never store passwords
// Store only opaque, short-lived tokens — never raw passwords or long-lived secrets
sessionStorage.setItem('ephemeral_nonce', nonce);

// SAFE: For sensitive data that must persist, use the Web Crypto API
// to encrypt before storing, with the key derived from user interaction
const key = await crypto.subtle.generateKey(
    { name: 'AES-GCM', length: 256 },
    false,           // not extractable
    ['encrypt', 'decrypt']
);
```

### Python

```python
# SAFE: Use a restricted temporary file that only the current user can read
import tempfile, os

# tempfile.mkstemp creates a file with mode 0600 (owner-read/write only)
fd, path = tempfile.mkstemp(prefix='vulnetix_', suffix='.tmp')
try:
    os.write(fd, token.encode())
finally:
    os.close(fd)
    # Always clean up temp files when done
    os.unlink(path)

# SAFE: Use a secrets manager or environment variable instead of files
import os
token = os.environ['API_TOKEN']
```

### PHP

```php
<?php
// SAFE: Use session storage (server-side, not world-readable)
session_start();
$_SESSION['auth_token'] = $token;

// SAFE: Use environment variables for secrets
$apiKey = getenv('API_KEY');

// SAFE: If file storage is necessary, write to a non-web-accessible directory
// with restricted permissions
$path = sys_get_temp_dir() . '/app_' . session_id() . '.dat';
file_put_contents($path, $sensitiveData);
chmod($path, 0600);  // owner-read/write only
```

### Android

```java
// SAFE: Use EncryptedSharedPreferences (Jetpack Security)
import androidx.security.crypto.EncryptedSharedPreferences;
import androidx.security.crypto.MasterKey;

MasterKey masterKey = new MasterKey.Builder(context)
    .setKeyScheme(MasterKey.KeyScheme.AES256_GCM)
    .build();

SharedPreferences securePrefs = EncryptedSharedPreferences.create(
    context,
    "secure_prefs",
    masterKey,
    EncryptedSharedPreferences.PrefKeyEncryptionScheme.AES256_SIV,
    EncryptedSharedPreferences.PrefValueEncryptionScheme.AES256_GCM
);
securePrefs.edit().putString("token", authToken).apply();
```
