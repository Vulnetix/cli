---
title: "VNX-648 – Incorrect Use of Privileged APIs"
description: "Detect dangerous uses of privileged OS APIs such as world-writable permissions, setuid(0), and shell sudo invocations."
---

## Overview

Privileged APIs — `chmod` with world-writable permissions, `setuid(0)` to become root, or `exec('sudo ...')` — when used incorrectly expose the entire system to compromise. Overly permissive file modes allow any local user to modify critical files, while acquiring root privileges unnecessarily violates the principle of least privilege.

**Severity:** Medium | **CWE:** [CWE-648 – Incorrect Use of Privileged APIs](https://cwe.mitre.org/data/definitions/648.html)

## Why This Matters

A single `chmod 0777 /var/app/config.ini` on a configuration file containing database credentials gives every user on the system read and write access. `setuid(0)` before processing untrusted input means any code injection vulnerability in the same process now runs as root. Calling `exec('sudo ' + user_input)` in a web application is a direct command injection vector with root privileges.

## What Gets Flagged

```python
# FLAGGED: World-writable permission on file
import os
os.chmod('/var/app/config.ini', 0o777)

# FLAGGED: Setting effective UID to root
import ctypes
ctypes.CDLL(None).setuid(0)

# FLAGGED: Executing sudo in subprocess
import os
os.exec('sudo ' + user_command)
```

```c
// FLAGGED: setuid to root
setuid(0);
setgid(0);
```

## Remediation

Apply the principle of least privilege consistently:

```python
# SAFE: Owner-only read/write
os.chmod('/var/app/config.ini', 0o600)

# SAFE: Use dedicated service accounts instead of root
# In Docker: USER appuser
# In systemd: User=appuser

# SAFE: Avoid sudo in application code entirely
# Use proper service accounts with targeted sudo rules if needed
# Or use capabilities (CAP_NET_BIND_SERVICE) instead of full root
```

Grant only the specific capabilities required (e.g., `CAP_NET_BIND_SERVICE` to bind to port 80) rather than full root privileges. Use security frameworks that manage privilege escalation transparently.
