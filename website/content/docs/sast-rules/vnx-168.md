## Overview

This rule flags inconsistent encoding where some special characters are escaped but others are not, or where URL decoding occurs before security checks. Attackers use double-encoding or alternate encodings to bypass the partial check. This maps to [CWE-168: Improper Handling of Inconsistent Special Elements](https://cwe.mitre.org/data/definitions/168.html).

**Severity:** Low | **CWE:** [CWE-168](https://cwe.mitre.org/data/definitions/168.html)

## Remediation

Use an atomic HTML or URL sanitization library that handles all special characters consistently rather than chaining partial `replace()` calls.
