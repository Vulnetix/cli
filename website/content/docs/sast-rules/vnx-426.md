## Overview

VNX-426 maps to [CWE-426: Untrusted Search Path](https://cwe.mitre.org/data/definitions/426.html). Detects source patterns associated with CWE-426 (Untrusted Search Path). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-426](https://cwe.mitre.org/data/definitions/426.html) | **Languages:** python

## Why This Matters

This weakness class (Untrusted Search Path) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-426 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-426 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-426` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
