---
title: "VNX-570 – Expression is Always False"
description: "Detect conditions that always evaluate to false, creating dead code blocks and potentially disabled security checks."
---

## Overview

A condition that always evaluates to false (`if (false)`, `if (1 == 2)`, `if False:`) makes the enclosed code block permanently unreachable. In security contexts this most often means a check was disabled for debugging and never re-enabled.

**Severity:** Low | **CWE:** [CWE-570 – Expression is Always False](https://cwe.mitre.org/data/definitions/570.html)

## Why This Matters

An always-false security check is equivalent to no check at all. Unlike deleted code, the dead block is visually present and may pass code review, creating a false sense that the check exists.

## What Gets Flagged

```java
// FLAGGED: Security check in always-false block
if (false) {
    checkPermissions(user);
}
```

```python
# FLAGGED: Always-false Python condition
if False:
    verify_csrf_token(request)
```

## Remediation

Remove the always-false condition or restore the actual security check:

```java
// SAFE: Actually perform the check
checkPermissions(user);
```
