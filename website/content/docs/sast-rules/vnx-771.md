---
title: "VNX-771 – Missing Reference to Active Allocated Resource"
description: "Detect memory or resources allocated without storing the returned reference, making them impossible to free and causing memory leaks."
---

## Overview

This rule flags `malloc`/`calloc` calls whose return value is discarded, and Java/Node resource constructions where the object is never assigned to a variable. Without a reference to the allocation, the resource can never be freed or closed, causing memory/resource leaks that can be exploited for denial of service. This maps to [CWE-771: Missing Reference to Active Allocated Resource](https://cwe.mitre.org/data/definitions/771.html).

**Severity:** Medium | **CWE:** [CWE-771](https://cwe.mitre.org/data/definitions/771.html)

## Why This Matters

Memory leaks in long-running services gradually exhaust available memory, eventually causing the process to be killed by the OS OOM killer. Under attacker control, this can be triggered deliberately to cause denial of service, particularly if the allocation size is determined by user input.

## What Gets Flagged

```c
// FLAGGED: malloc result discarded
void process(int n) {
    malloc(n * sizeof(int));  // allocated but never stored or freed
}
```

```java
// FLAGGED: FileInputStream constructed but not stored
new FileInputStream("/etc/config");  // leaks file descriptor
```

## Remediation

```c
// SAFE: always store and later free the allocation
int *buf = malloc(n * sizeof(int));
if (buf == NULL) {
    handle_error();
    return;
}
// ... use buf ...
free(buf);
buf = NULL;
```

```java
// SAFE: assign to variable and use try-with-resources
try (FileInputStream fis = new FileInputStream("/etc/config")) {
    // ... use fis ...
}  // automatically closed
```
