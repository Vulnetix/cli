## Overview

VNX-771 maps to [CWE-771: Missing Reference to Active Allocated Resource](https://cwe.mitre.org/data/definitions/771.html). Detects source patterns associated with CWE-771 (Missing Reference to Active Allocated Resource). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-771](https://cwe.mitre.org/data/definitions/771.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Missing Reference to Active Allocated Resource) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-771 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'malloc(' pattern
malloc(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-771 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-771` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
