---
title: "VNX-601 – Open Redirect"
description: "Detect redirect calls that use unvalidated user input as the redirect destination, enabling phishing and OAuth token hijacking attacks."
---

## Overview

An open redirect vulnerability occurs when an application redirects a user's browser to a URL supplied as user input without validating that the destination is trusted. Attackers craft malicious links that appear to originate from a legitimate site (`https://trusted.example.com/login?next=https://evil.com`) but redirect to a phishing page or OAuth callback harvester. This maps to [CWE-601: URL Redirection to Untrusted Site ('Open Redirect')](https://cwe.mitre.org/data/definitions/601.html).

**Severity:** Medium | **CWE:** [CWE-601 – URL Redirection to Untrusted Site](https://cwe.mitre.org/data/definitions/601.html)

## Why This Matters

Open redirects are frequently dismissed as low-severity, but they are actively exploited in:

1. **Phishing campaigns** — the trusted domain lends credibility to the malicious URL, bypassing email spam filters and user suspicion.
2. **OAuth token theft** — OAuth 2.0 flows use `redirect_uri` parameters; open redirects on the authorization server allow token exfiltration to attacker-controlled URLs.
3. **Content Security Policy bypass** — CSP policies allowing redirects from trusted domains can be bypassed with open redirects on those domains.

Bug bounty programs routinely accept open redirect reports, particularly when chained with other issues. MITRE ATT&CK T1566 covers phishing, which open redirects directly enable.

## What Gets Flagged

```python
# FLAGGED: Django redirect from GET parameter
from django.shortcuts import redirect
from django.http import HttpResponseRedirect

def login_view(request):
    next_url = request.GET.get('next')
    return HttpResponseRedirect(next_url)  # attacker controls destination

def another_view(request):
    return redirect(request.POST['url'])  # open redirect via POST
```

```ruby
# FLAGGED: Rails redirect to params value
class SessionsController < ApplicationController
  def create
    redirect_to params[:redirect_url]  # open redirect
  end
end
```

```php
<?php
// FLAGGED: PHP header redirect from GET parameter
header("Location: " . $_GET['url']);  // open redirect
exit;
```

```java
// FLAGGED: Java Servlet redirect from request parameter
String redirectUrl = request.getParameter("url");
response.sendRedirect(redirectUrl);  // open redirect
```

```javascript
// FLAGGED: Express redirect from query parameter
app.get('/login', (req, res) => {
    res.redirect(req.query.redirect);  // open redirect
});
```

```go
// FLAGGED: Go net/http redirect from URL query
func loginHandler(w http.ResponseWriter, r *http.Request) {
    redirectTo := r.URL.Query().Get("url")
    http.Redirect(w, r, redirectTo, http.StatusFound)  // open redirect
}
```

## Remediation

The safest approach is to only allow relative paths (no scheme or host) and validate them against a known-good pattern. Where cross-domain redirects are genuinely needed, use a server-side allowlist.

### Python/Django

```python
from urllib.parse import urlparse, urljoin
from django.http import HttpResponseRedirect
from django.utils.http import url_has_allowed_host_and_scheme

def safe_redirect(request):
    next_url = request.GET.get('next', '/')
    # Only allow redirects to the same host
    if not url_has_allowed_host_and_scheme(
        url=next_url,
        allowed_hosts={request.get_host()},
        require_https=request.is_secure(),
    ):
        next_url = '/'
    return HttpResponseRedirect(next_url)
```

### Ruby on Rails

```ruby
# SAFE: Use only relative paths or validate against allowlist
def safe_redirect_url(url)
  uri = URI.parse(url)
  # Reject absolute URLs (any URL with a host component)
  return root_path if uri.host.present?
  url
rescue URI::InvalidURIError
  root_path
end

redirect_to safe_redirect_url(params[:redirect_url])
```

### PHP

```php
<?php
// SAFE: Validate redirect target is a relative path only
function safe_redirect(string $url): string {
    $parsed = parse_url($url);
    // Reject anything with a scheme or host
    if (isset($parsed['scheme']) || isset($parsed['host'])) {
        return '/';
    }
    return $url;
}

$redirect = safe_redirect($_GET['url'] ?? '/');
header("Location: " . $redirect);
exit;
```

### Java

```java
// SAFE: Allowlist of permitted redirect destinations
private static final Set<String> ALLOWED_REDIRECTS = Set.of(
    "/dashboard", "/profile", "/settings"
);

String redirectUrl = request.getParameter("url");
if (!ALLOWED_REDIRECTS.contains(redirectUrl)) {
    redirectUrl = "/";
}
response.sendRedirect(redirectUrl);
```

### Express (Node.js)

```javascript
// SAFE: Only allow relative paths
function safeRedirect(res, url) {
    // Reject URLs with a protocol (absolute URLs)
    if (/^https?:\/\//i.test(url) || url.startsWith('//')) {
        return res.redirect('/');
    }
    res.redirect(url);
}

app.get('/login', (req, res) => {
    safeRedirect(res, req.query.redirect || '/');
});
```

### Go

```go
// SAFE: Validate URL is relative before redirecting
func safeRedirect(w http.ResponseWriter, r *http.Request, target string) {
    parsed, err := url.Parse(target)
    if err != nil || parsed.IsAbs() {
        target = "/"
    }
    http.Redirect(w, r, target, http.StatusFound)
}
```
