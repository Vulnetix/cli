---
title: "VNX-1391 – Use of Weak Credentials"
description: "Detect hardcoded weak credential pairs (admin/admin, root/root, user/password) and obvious weak password assignments in source code or configuration."
---

## Overview

Default and weak credentials hardcoded in source code, configuration files, or test fixtures provide trivial authentication bypass for anyone who reads the code. Default credential scanners routinely find these during penetration tests. Even if the credentials are only used in test environments, they frequently leak into production through configuration management mistakes. **Severity:** Critical | **CWE:** [CWE-1391 – Use of Weak Credentials](https://cwe.mitre.org/data/definitions/1391.html)

## Why This Matters

The Mirai botnet compromised over 100,000 IoT devices using a list of 61 default credential pairs. The SolarWinds SUNBURST attack was initially accessed via a hardcoded weak password. OWASP API Security Top 10 A7:2023 is specifically about "Security Misconfiguration" including default credentials. Shodan and similar scanners actively search for services using default credentials.

## What Gets Flagged

```python
# FLAGGED: Weak credential assignment
password = "admin"
password = "1234"
PASSWORD = "password"

# FLAGGED: Credential pair in source
credentials = {"username": "admin", "password": "admin"}
```

```yaml
# FLAGGED: Weak credential in configuration
username: admin
password: admin123
```

```javascript
// FLAGGED: Hardcoded weak password
const password = 'admin';
const db_password = '1234';
```

## Remediation

1. **Never hardcode credentials.** Use environment variables, secrets managers, or secure vaults.

2. **For test environments**, use generated random credentials that are rotated per test run:
```python
import secrets
TEST_PASSWORD = secrets.token_urlsafe(32)
```

3. **For default admin account creation**, require the password to be set on first run:
```python
import os
ADMIN_PASSWORD = os.environ['ADMIN_PASSWORD']
if not ADMIN_PASSWORD:
    raise RuntimeError("ADMIN_PASSWORD environment variable is required")
```

4. **Use a password strength policy** — minimum 12 characters, mixed case, numbers, and symbols.

5. **Use a `.env` file** excluded from git via `.gitignore`, and a `.env.example` with placeholder values.
