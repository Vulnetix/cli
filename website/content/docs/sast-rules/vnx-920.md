## Overview

Applications that poll in tight loops or perform unbounded computation in response to network events consume excessive CPU and power. On mobile and IoT devices this drains batteries and can render devices unusable. In server applications it enables denial-of-service by exhausting CPU resources.

**Severity:** Low | **CWE:** [CWE-920 – Improper Restriction of Power Consumption](https://cwe.mitre.org/data/definitions/920.html)

## Why This Matters

An attacker who can trigger tight polling loops (e.g. by sending events continuously) can degrade or disable devices. On battery-powered devices, this constitutes a denial-of-service attack through resource depletion.

## What Gets Flagged

```python
# FLAGGED: Busy-poll without sleep
while True:
    check_for_updates()  # no time.sleep()
```

```java
// FLAGGED: No delay in loop
while (true) {
    processQueue();  // no Thread.sleep()
}
```

## Remediation

Add appropriate delays or use event-driven patterns:

```python
# SAFE: Sleep between polls
while True:
    check_for_updates()
    time.sleep(5)

# BETTER: Event-driven (no polling needed)
```
