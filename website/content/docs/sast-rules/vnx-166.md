## Overview

This rule flags parsing operations that split or tokenize user input using a specific delimiter without first verifying the delimiter is actually present. If the expected delimiter is absent, `split()` may return the whole string as a single token or an empty result, causing incorrect logic or skipping validation steps. This maps to [CWE-166: Improper Handling of Missing Special Element](https://cwe.mitre.org/data/definitions/166.html).

**Severity:** Low | **CWE:** [CWE-166 – Improper Handling of Missing Special Element](https://cwe.mitre.org/data/definitions/166.html)

## What Gets Flagged

```python
# FLAGGED: split without checking delimiter exists
parts = input.split(":")
username = parts[0]
domain = parts[1]  # IndexError if no ":" present
```

```c
// FLAGGED: strtok without checking for delimiter
char *token = strtok(input, ":");
```

## Remediation

```python
# SAFE: verify delimiter before splitting
if ":" not in input:
    raise ValueError("Expected format user:domain")
username, domain = input.split(":", 1)
```

```c
// SAFE: check token before using
char *token = strtok(input, ":");
if (!token) {
    return -1;  // missing delimiter
}
```
