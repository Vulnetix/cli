---
title: "VNX-1390 – Weak Authentication"
description: "Detect JWT signing with weak secrets, short authentication tokens, and hardcoded weak authentication keys that can be brute-forced or guessed."
---

## Overview

Authentication mechanisms are only as strong as their secrets. JWT tokens signed with `'secret'` or short keys can be brute-forced offline using tools like hashcat. HMAC keys shorter than the hash output size provide reduced security. Hardcoded authentication secrets in source code can be extracted by anyone with access to the repository. **Severity:** High | **CWE:** [CWE-1390 – Weak Authentication](https://cwe.mitre.org/data/definitions/1390.html)

## Why This Matters

JWT secret brute-forcing is a real attack: tools like jwt_tool and hashcat can test millions of candidate secrets per second against captured tokens. The `alg: none` JWT attack and weak HS256 secrets have been used in numerous CTF challenges and real-world penetrations. The 2021 Twitch breach partially involved authentication token exposure.

## What Gets Flagged

```python
# FLAGGED: JWT signed with weak secret
import jwt
token = jwt.encode(payload, 'secret', algorithm='HS256')
token = jwt.encode(payload, 'password', algorithm='HS256')

# FLAGGED: Weak secret key assignment
JWT_SECRET = 'secret'
jwt_secret = 'key'
```

```javascript
// FLAGGED: JWT sign with weak secret
const token = jwt.sign(payload, 'secret');
const token = jwt.sign(data, 'changeme');
```

## Remediation

```python
# SAFE: Strong random secret from environment
import os
import secrets

JWT_SECRET = os.environ.get('JWT_SECRET')
if not JWT_SECRET or len(JWT_SECRET) < 32:
    raise RuntimeError("JWT_SECRET must be set to at least 32 characters")

# Generate a strong secret for initial setup:
# python -c "import secrets; print(secrets.token_hex(32))"
```

```javascript
// SAFE: Secret from environment with length validation
const JWT_SECRET = process.env.JWT_SECRET;
if (!JWT_SECRET || JWT_SECRET.length < 32) {
    throw new Error('JWT_SECRET must be at least 32 characters');
}
const token = jwt.sign(payload, JWT_SECRET, {
    algorithm: 'HS256',
    expiresIn: '1h',
});
```

Consider using RS256 (asymmetric) instead of HS256 so the signing key never needs to be shared with verification services.
