## Overview

Dead code — blocks that can never execute — often indicates logic errors, accidentally disabled security checks, or forgotten debug code that was intended to be removed. CWE-561 flags `if (false)`, `if (False):`, and similar always-false conditions.

**Severity:** Low | **CWE:** [CWE-561 – Dead Code](https://cwe.mitre.org/data/definitions/561.html)

## Why This Matters

A security check wrapped in `if (false)` provides no protection. Dead code in security-critical paths is a common indicator that a developer temporarily disabled a check and forgot to re-enable it before deployment.

## What Gets Flagged

```python
# FLAGGED: Security check in dead code block
if False:
    verify_signature(request)
```

```java
// FLAGGED: Dead code block
if (false) {
    checkPermissions(user);
}
```

## Remediation

Remove dead code entirely or restore the intended condition:

```python
# SAFE: Actually perform the security check
verify_signature(request)
```

Review the git history to understand why the condition was set to always-false before removing it.
