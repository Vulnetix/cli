## Overview

Detects goto statements used outside switch/case contexts. Goto creates unpredictable control flow that can bypass security checks, skip initialisation, jump over cleanup code (resource leaks), and make security auditing extremely difficult.

**Severity:** Low | **CWE:** [CWE-1075](https://cwe.mitre.org/data/definitions/1075.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1075](https://cwe.mitre.org/data/definitions/1075.html) for authoritative guidance.
