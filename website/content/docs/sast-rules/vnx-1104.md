## Overview

VNX-1104 maps to [CWE-1104: Use of Unmaintained Third Party Components](https://cwe.mitre.org/data/definitions/1104.html). Detects source patterns associated with CWE-1104 (Use of Unmaintained Third Party Components). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-1104](https://cwe.mitre.org/data/definitions/1104.html) | **Languages:** multiple

## Why This Matters

This weakness class (Use of Unmaintained Third Party Components) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1104 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```
// FLAGGED: contains '"*"' pattern
"*"
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1104 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1104` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
