---
title: "VNX-243 – chroot Jail Without Changing Working Directory"
description: "Detect chroot() calls not followed by chdir('/'), leaving an escape path via the pre-jail working directory."
---

## Overview

`chroot()` changes the root directory but does not change the current working directory. A process that retains a pre-chroot working directory can escape the jail via `chdir("../../..")` relative to the retained directory. The fix requires calling `chdir("/")` immediately after `chroot()` and dropping root privileges. This maps to [CWE-243: Creation of chroot Jail Without Changing Working Directory](https://cwe.mitre.org/data/definitions/243.html).

**Severity:** High | **CWE:** [CWE-243](https://cwe.mitre.org/data/definitions/243.html)

## Remediation

```c
// SAFE: correct chroot sequence
if (chroot("/var/www") != 0) { perror("chroot"); exit(1); }
if (chdir("/") != 0) { perror("chdir"); exit(1); }
// Drop privileges AFTER chroot+chdir
if (setgid(www_gid) != 0) { perror("setgid"); exit(1); }
if (setuid(www_uid) != 0) { perror("setuid"); exit(1); }
```
