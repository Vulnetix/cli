## Overview

VNX-125 maps to [CWE-125: Out-of-bounds Read](https://cwe.mitre.org/data/definitions/125.html). Detects source patterns associated with CWE-125 (Out-of-bounds Read). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-125](https://cwe.mitre.org/data/definitions/125.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Out-of-bounds Read) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-125 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'strcpy(' pattern
strcpy(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-125 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-125` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
