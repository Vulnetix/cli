title: "VNX-772 – Missing Release of Resource after Effective Lifetime"
description: "Detect file handles, database connections, and network sockets opened without being explicitly closed, causing resource exhaustion."
---

## Overview

This rule flags resources (file handles, database connections, streams, sockets) opened in Java, Python, and Go without being closed in a guaranteed cleanup path. If an exception is thrown before the explicit `close()` call, the resource leaks until the GC or process exit. This maps to [CWE-772: Missing Release of Resource after Effective Lifetime](https://cwe.mitre.org/data/definitions/772.html).

**Severity:** Medium | **CWE:** [CWE-772](https://cwe.mitre.org/data/definitions/772.html)

## Why This Matters

Resource exhaustion from file descriptor leaks is a common denial of service vector. Most operating systems limit file descriptors per process (typically 1024–65536). A service that leaks one file descriptor per request will exhaust its fd limit in minutes under normal load, becoming unable to accept new connections.

## What Gets Flagged

```python
# FLAGGED: open() without with statement
f = open('/etc/config')
data = f.read()
# if read() throws, f.close() is never called
f.close()
```

```java
// FLAGGED: FileInputStream without try-with-resources
FileInputStream fis = new FileInputStream(path);
int data = fis.read();  // if this throws, fis is never closed
fis.close();
```

```go
// FLAGGED: os.Open without defer Close
f, err := os.Open(path)
if err != nil { return err }
// no defer f.Close() — leaks fd if function returns early
data, _ := io.ReadAll(f)
```

## Remediation

```python
# SAFE: context manager guarantees close()
with open('/etc/config') as f:
    data = f.read()
```

```java
// SAFE: try-with-resources guarantees close()
try (FileInputStream fis = new FileInputStream(path)) {
    int data = fis.read();
}
```

```go
// SAFE: defer Close() immediately after error check
f, err := os.Open(path)
if err != nil { return err }
defer f.Close()
data, err := io.ReadAll(f)
```
