description: "Detect Python classes using multiple concrete inheritance, which creates ambiguous method resolution order."
---

## Overview

Detects Python classes inheriting from multiple concrete (non-abstract) classes. Multiple concrete inheritance creates diamond-problem ambiguity, unpredictable method resolution order, and makes it difficult to reason about security-critical behaviour such as authentication and permission checks.

**Severity:** Low | **CWE:** [CWE-1055 – Multiple Inheritance from Concrete Classes](https://cwe.mitre.org/data/definitions/1055.html)

## Why This Matters

When a class inherits a security method (e.g., `check_permission`) from multiple concrete parents, Python's MRO determines which parent's implementation runs. This can be non-obvious and change when the class hierarchy changes, potentially silently bypassing security checks.

## What Gets Flagged

```python
# FLAGGED: inheriting from two concrete classes
class AdminUser(User, AdminMixin):
    pass
```

## Remediation

Use abstract base classes (ABC) or mixins (pure behaviour classes without state):

```python
from abc import ABC, abstractmethod

class PermissionMixin:  # pure mixin, no state
    def check_permission(self, perm): ...

class User(ABC):
    @abstractmethod
    def authenticate(self): ...

class AdminUser(User, PermissionMixin):  # SAFE: mixin + abstract
    def authenticate(self): ...
```
