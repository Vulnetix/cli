---
title: "VNX-233 – Improper Handling of Parameters (PHP Mass Variable Import)"
description: "Detect PHP extract() or import_request_variables() importing all request parameters as local variables."
---

## Overview

PHP's `extract($_GET)` imports every query parameter as a local variable, allowing an attacker to overwrite any variable in scope — including authentication flags (`$is_admin = true`) or database handles. This is a well-known PHP security anti-pattern. This maps to [CWE-233: Improper Handling of Parameters](https://cwe.mitre.org/data/definitions/233.html).

**Severity:** High | **CWE:** [CWE-233](https://cwe.mitre.org/data/definitions/233.html)

## What Gets Flagged

```php
<?php
// FLAGGED: imports everything as variables
extract($_GET);
extract($_POST);
import_request_variables('gp');
parse_str($_SERVER['QUERY_STRING']);  // without second array arg
```

## Remediation

```php
<?php
// SAFE: access parameters explicitly
$username = filter_input(INPUT_POST, 'username', FILTER_SANITIZE_STRING);
$page = (int) $_GET['page'];

// SAFE: parse_str with output array
parse_str($_SERVER['QUERY_STRING'], $params);
$page = (int) ($params['page'] ?? 0);
```
