---
title: "VNX-563 – Assignment to Variable without Use"
description: "Detect security check results assigned to variables that are never used to enforce an access decision."
---

## Overview

Assigning the result of a security check (authentication, authorization, validation) to a variable but never using it in a conditional means the check's outcome has no effect. The check runs but its result is ignored, effectively making it a no-op.

**Severity:** Low | **CWE:** [CWE-563 – Assignment to Variable without Use](https://cwe.mitre.org/data/definitions/563.html)

## Why This Matters

A developer may add `is_authenticated = checkAuth(user)` intending to gate subsequent logic, but if the `if is_authenticated:` block is never added (or is removed later), the security check provides no protection. The code appears safe on inspection but is not.

## What Gets Flagged

```python
# FLAGGED: Result assigned but never checked
is_authenticated = verify_token(token)
# ... no if is_authenticated: follows
perform_sensitive_operation()
```

```go
// FLAGGED: Security function result discarded
_, err = authorize(ctx, user)
// err ignored - authorization result has no effect
```

## Remediation

Always use the result of security checks:

```python
# SAFE: Check result is enforced
if not verify_token(token):
    raise PermissionError("Invalid token")
perform_sensitive_operation()
```
