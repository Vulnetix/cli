## Overview

XML External Entity (XXE) injection occurs when an XML parser processes external entity declarations (`<!ENTITY xxe SYSTEM "file:///etc/passwd">`) embedded in attacker-controlled XML input. This allows attackers to read local files, initiate outbound HTTP requests (SSRF), enumerate open ports, and in some parsers achieve remote code execution. This maps to [CWE-611: Improper Restriction of XML External Entity Reference](https://cwe.mitre.org/data/definitions/611.html).

**Severity:** High | **CWE:** [CWE-611 – Improper Restriction of XML External Entity Reference](https://cwe.mitre.org/data/definitions/611.html)

## Why This Matters

XXE was listed in the OWASP Top Ten for many years and continues to appear in production applications. Most XML parsers enable external entity processing by default as part of the XML specification, meaning any application that parses user-supplied XML without explicitly disabling this feature is vulnerable. XXE is particularly dangerous in internal tooling, API gateways, and document processing pipelines. Blind XXE via DNS out-of-band channels can exfiltrate data even when the response is not reflected to the attacker. In 2018, the UK National Cyber Security Centre identified XXE as a root cause in several major data breaches.

## What Gets Flagged

```python
# FLAGGED: xml.etree.ElementTree is vulnerable to billion-laughs DoS
# and in some Python builds, external entity attacks
import xml.etree.ElementTree as ET
tree = ET.parse(user_uploaded_file)

# FLAGGED: lxml without explicit security settings
from lxml import etree
tree = etree.parse(user_file)  # processes external entities by default
```

```java
// FLAGGED: DocumentBuilderFactory without disabling external entities
DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
// Missing: dbf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true)
DocumentBuilder db = dbf.newDocumentBuilder();
Document doc = db.parse(inputStream);

// FLAGGED: SAXParserFactory without disabling external processing
SAXParserFactory spf = SAXParserFactory.newInstance();
SAXParser parser = spf.newSAXParser();

// FLAGGED: XMLInputFactory without disabling external entities
XMLInputFactory xif = XMLInputFactory.newInstance();
// Missing: xif.setProperty(XMLInputFactory.IS_SUPPORTING_EXTERNAL_ENTITIES, false)
XMLStreamReader reader = xif.createXMLStreamReader(inputStream);
```

```php
<?php
// FLAGGED: simplexml_load_string processes external entities by default
$xml = simplexml_load_string($userInput);

// FLAGGED: DOMDocument without disabling external entities
$dom = new DOMDocument();
$dom->loadXML($userInput);  // LIBXML_NOENT makes this worse
```

```javascript
// FLAGGED: libxmljs with external entities enabled
const libxml = require('libxmljs');
const doc = libxml.parseXml(userInput, { noent: true });  // explicitly dangerous
```

```csharp
// FLAGGED: XmlReader without prohibiting DTD processing
XmlReader reader = XmlReader.Create(inputStream);  // default allows DTD

// FLAGGED: XmlDocument loads DTD by default in older .NET
XmlDocument doc = new XmlDocument();
doc.Load(inputStream);
```

## Remediation

The fix is to disable DTD processing entirely. This is more reliable than disabling only external entities, as it prevents all DTD-based attacks including billion-laughs DoS.

### Python

```python
# SAFE: Use defusedxml — drop-in replacement that disables all DTD attacks
import defusedxml.ElementTree as ET
tree = ET.parse(user_uploaded_file)

# SAFE: lxml with explicit security settings
from lxml import etree
parser = etree.XMLParser(
    resolve_entities=False,
    no_network=True,
    load_dtd=False,
)
tree = etree.parse(user_file, parser=parser)
```

### Java

```java
// SAFE: Disable DTD processing entirely for DocumentBuilderFactory
DocumentBuilderFactory dbf = DocumentBuilderFactory.newInstance();
dbf.setFeature("http://apache.org/xml/features/disallow-doctype-decl", true);
dbf.setFeature("http://xml.org/sax/features/external-general-entities", false);
dbf.setFeature("http://xml.org/sax/features/external-parameter-entities", false);
dbf.setExpandEntityReferences(false);
DocumentBuilder db = dbf.newDocumentBuilder();

// SAFE: XMLInputFactory with external entities disabled
XMLInputFactory xif = XMLInputFactory.newInstance();
xif.setProperty(XMLInputFactory.IS_SUPPORTING_EXTERNAL_ENTITIES, false);
xif.setProperty(XMLInputFactory.SUPPORT_DTD, false);
```

### PHP

```php
<?php
// SAFE: Disable external entities before parsing
libxml_disable_entity_loader(true);
$xml = simplexml_load_string($userInput, 'SimpleXMLElement', LIBXML_NONET);

// SAFE: DOMDocument with entity loading disabled
$dom = new DOMDocument();
libxml_disable_entity_loader(true);
$dom->loadXML($userInput, LIBXML_NONET | LIBXML_NOERROR);
```

### .NET/C#

```csharp
// SAFE: XmlReader with DTD processing prohibited
XmlReaderSettings settings = new XmlReaderSettings {
    DtdProcessing = DtdProcessing.Prohibit,
    XmlResolver = null
};
XmlReader reader = XmlReader.Create(inputStream, settings);

// SAFE: XmlDocument with explicit null resolver
XmlDocument doc = new XmlDocument { XmlResolver = null };
doc.Load(inputStream);
```

### JavaScript

```javascript
// SAFE: xml2js does not process external entities by default
const xml2js = require('xml2js');
const parser = new xml2js.Parser({ explicitArray: false });
const result = await parser.parseStringPromise(userInput);

// SAFE: libxmljs without noent
const doc = libxml.parseXml(userInput);  // noent defaults to false
```
