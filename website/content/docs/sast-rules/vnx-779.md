---
title: "VNX-779 – Logging of Excessive Data"
description: "Detect log statements that include request bodies, passwords, tokens, or other sensitive data that should not appear in logs."
---

## Overview

This rule flags log statements that reference request bodies or security-sensitive field names (password, token, api_key, secret). Logging these values exposes credentials in log files, SIEM systems, and monitoring dashboards where access controls are often weaker than the primary application database. This maps to [CWE-779: Logging of Excessive Data](https://cwe.mitre.org/data/definitions/779.html).

**Severity:** Medium | **CWE:** [CWE-779](https://cwe.mitre.org/data/definitions/779.html)

## Why This Matters

Application logs are often stored in less secure locations than the main database, forwarded to multiple SIEM tools, retained for extended periods, and accessible to a wider range of personnel. Password hashes, API keys, and session tokens appearing in logs become the primary attack surface after a log management system breach.

## What Gets Flagged

```python
# FLAGGED: logging the entire request body
logger.debug(request.body)   # body may contain password
logging.info(request.json)   # includes all submitted fields

# FLAGGED: logging sensitive field
logger.info("User login: password=%s", password)
```

```javascript
// FLAGGED: logging request body (may contain credentials)
console.log('Request:', req.body)
logger.debug('Payload:', request.body)
```

```go
// FLAGGED: logging password field
log.Printf("Login attempt: password=%s", password)
fmt.Println("auth token:", token)
```

## Remediation

```python
# SAFE: log only safe, explicitly chosen fields
logger.info("Login attempt", extra={
    'username': request.json.get('username'),  # safe to log
    'ip': request.remote_addr,
    'user_agent': request.headers.get('User-Agent'),
    # never log: password, token, credit_card, etc.
})
```

```javascript
// SAFE: create a sanitized log object
const safePayload = {
    username: req.body.username,
    action: req.body.action,
    // explicitly exclude: password, token, apiKey, etc.
};
logger.info('Request received', safePayload);
```

Use a log sanitization library or middleware that strips known sensitive field names before any log output.
