## Overview

Detects instantiation of classes inside static initializer blocks (Java), class-level code blocks, or module-level constructor calls. Premature object instantiation in static context can bypass security initialization, cause unexpected state, and create difficult-to-audit singleton patterns.

**Severity:** Low | **CWE:** [CWE-1063](https://cwe.mitre.org/data/definitions/1063.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1063](https://cwe.mitre.org/data/definitions/1063.html) for authoritative guidance.
