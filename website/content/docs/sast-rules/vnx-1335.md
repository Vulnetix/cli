---
title: "VNX-1335 – Incorrect Bitwise Shift of Integer"
description: "Detect bitwise shift operations with shift amounts >= the bit width of the type (e.g. << 32 on a 32-bit int) or with variable shift amounts that may exceed valid range, causing undefined behaviour in C/C++ or unexpected results in Java."
---

## Overview

Shifting an integer by an amount greater than or equal to its bit width is undefined behaviour in C/C++ (the compiler may produce any result or optimise away the operation entirely). In Java the shift amount is masked (`x << 33` is equivalent to `x << 1` for int), which causes silent logic errors. Negative shift amounts are also undefined in C/C++. **Severity:** Medium | **CWE:** [CWE-1335 – Incorrect Bitwise Shift of Integer](https://cwe.mitre.org/data/definitions/1335.html)

## Why This Matters

Compilers exploit undefined behaviour aggressively. A shift by `>= 32` on a 32-bit integer that the compiler identifies as UB may be completely eliminated, producing always-zero results or causing security checks to be optimised away. In cryptographic code, incorrect shifts produce incorrect key material. In hash functions, they cause collision-inducing bias.

## What Gets Flagged

```c
// FLAGGED: Shift amount >= 32 for 32-bit int
uint32_t x = 1;
uint32_t result = x << 32;  // undefined behaviour in C

// FLAGGED: Variable shift without bounds check
uint32_t shift_by = user_supplied_shift;
uint32_t result = x << shift_by;  // UB if shift_by >= 32
```

```java
// FLAGGED: Shift by 32 on int (silently becomes << 0)
int flags = 1 << 32;  // same as 1 << 0 = 1, not 0 as expected

// FLAGGED: Variable shift with user input
int n = Integer.parseInt(request.getParameter("n"));
int result = value << n;  // unexpected if n >= 32 or negative
```

## Remediation

```c
// SAFE: Validate shift amount before use
uint32_t shift_by = user_supplied_shift;
if (shift_by >= 32) {
    result = 0;  // or error
} else {
    result = x << shift_by;
}

// SAFE: Use 64-bit type when 32-bit shift is needed
uint64_t result = (uint64_t)1 << 32;  // well-defined for uint64_t
```

```java
// SAFE: Mask shift amount for 32-bit int
int n = Integer.parseInt(request.getParameter("n"));
n = n & 0x1F;  // mask to 0-31
int result = value << n;

// Or validate explicitly
if (n < 0 || n >= 32) throw new IllegalArgumentException("shift out of range");
int result = value << n;
```
