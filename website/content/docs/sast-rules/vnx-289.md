## Overview

VNX-289 maps to [CWE-289: Authentication Bypass by Alternate Name](https://cwe.mitre.org/data/definitions/289.html). Detects source patterns associated with CWE-289 (Authentication Bypass by Alternate Name). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-289](https://cwe.mitre.org/data/definitions/289.html) | **Languages:** java, node, python

## Why This Matters

This weakness class (Authentication Bypass by Alternate Name) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-289 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'toLowerCase()' pattern
toLowerCase()
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-289 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-289` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
