## Overview

This rule flags the use of hash functions that are cryptographically weak: MD5 (128-bit, collision attacks demonstrated), SHA-1 (160-bit, SHAttered collision attack), CRC32 (non-cryptographic, designed for error detection), and Adler-32. These are unsuitable for integrity verification of security-sensitive data and completely inappropriate for password storage. This maps to [CWE-328: Use of Weak Hash](https://cwe.mitre.org/data/definitions/328.html).

**Severity:** High | **CWE:** [CWE-328 – Use of Weak Hash](https://cwe.mitre.org/data/definitions/328.html)

## Why This Matters

A hash function used for security must have three properties: pre-image resistance (cannot reverse the hash), second pre-image resistance (cannot find a different input with the same hash), and collision resistance (cannot find any two inputs with the same hash). MD5 and SHA-1 both fail collision resistance — an attacker can produce two different files with the same hash, enabling forged signatures, duplicate certificates, or bypassed integrity checks. The SHAttered attack (2017) demonstrated a practical SHA-1 collision costing roughly $110,000 of compute. The cost continues to fall. For password hashing, even SHA-256 is inappropriate because it is too fast — a modern GPU can compute billions of SHA-256 hashes per second, enabling trivial brute-force. CRC32 was never designed for security and can be reversed analytically.

## Remediation

1. **For file/data integrity**, use **SHA-256** or **SHA-3 (SHA3-256)**.

```python
# SAFE: SHA-256 for file integrity
import hashlib
checksum = hashlib.sha256(file_data).hexdigest()
```

```go
// SAFE: SHA-256 in Go
import "crypto/sha256"
hash := sha256.Sum256(data)
```

2. **For password storage**, use a slow adaptive algorithm: **bcrypt**, **argon2id**, or **scrypt**. These include a cost factor that can be increased as hardware improves.

```python
# SAFE: bcrypt for password hashing in Python
import bcrypt
hashed = bcrypt.hashpw(password.encode(), bcrypt.gensalt(rounds=12))
```

```go
// SAFE: bcrypt in Go
import "golang.org/x/crypto/bcrypt"
hash, err := bcrypt.GenerateFromPassword([]byte(password), bcrypt.DefaultCost)
```

3. **For non-security checksums** (e.g., hash tables, data partitioning), MD5/CRC32 may be acceptable only if no security property is required. Clearly comment the intent.

4. **Verify and update existing hashes.** If users' passwords are stored as MD5 or SHA-1 hashes, force a password reset or rehash on next login using a strong algorithm.
