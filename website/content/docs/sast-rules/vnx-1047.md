description: "Detect deferred imports inside function bodies that indicate circular dependency workarounds and initialization order problems."
---

## Overview

Circular module dependencies cause initialization order issues. Security-critical objects (loggers, validators, auth handlers) may not be fully initialized when first referenced. Developers commonly work around circular imports by deferring the import inside a function, which can mask the root problem.

**Severity:** Low | **CWE:** [CWE-1047 – Modules with Circular Dependencies](https://cwe.mitre.org/data/definitions/1047.html)

## Why This Matters

If a security module (e.g. a rate limiter or authentication handler) is imported lazily and the import fails silently, the security check may be skipped entirely on first use.

## What Gets Flagged

```python
# FLAGGED: Deferred import inside function (circular dep workaround)
def authenticate(user):
    from auth import check_token  # deferred to avoid circular import
    return check_token(user)
```

## Remediation

Resolve the circular dependency at the architecture level by extracting shared utilities into a third module that neither circular partner imports.
