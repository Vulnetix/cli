title: "VNX-131 – Incorrect Calculation of Buffer Size"
description: "Detect C/C++ allocation patterns that compute the wrong buffer size, including malloc(strlen(s)) without +1 and sizeof(ptr) instead of sizeof(*ptr)."
---

## Overview

Getting the allocation size wrong by even a single byte is enough to produce a heap overflow. CWE-131 covers three common off-by-one and size-calculation mistakes in C and C++:

1. **`malloc(strlen(str))`** — `strlen` returns the number of characters without the null terminator. The string therefore needs `strlen(str) + 1` bytes. Writing the null terminator into the underallocated buffer corrupts the next heap chunk.
2. **`sizeof(ptr)` instead of `sizeof(*ptr)`** — `sizeof` on a pointer yields the pointer width (4 or 8 bytes), not the size of the object it points to. This is a subtle bug when the intent was to allocate a copy of a struct.
3. **`malloc(count * sizeof(type))`** — If `count` is attacker-controlled and the multiplication overflows `size_t`, the result wraps to a small value, the allocation succeeds, but the subsequent writes overflow the tiny buffer.

**Severity:** High | **CWE:** [CWE-131 – Incorrect Calculation of Buffer Size](https://cwe.mitre.org/data/definitions/131.html)

## Why This Matters

Off-by-one heap overflows are difficult to detect in testing because they may only manifest as memory corruption in specific allocator states. In production they can be triggered reliably with crafted input, leading to heap metadata corruption and, in many allocator implementations, arbitrary code execution. The multiplication overflow variant is particularly dangerous in 32-bit code where a count of `0x10001` multiplied by a 4-byte struct size wraps to just `4` bytes allocated.

## What Gets Flagged

```c
/* FLAGGED: strlen does not count the null terminator */
char *copy = malloc(strlen(src));    // should be strlen(src) + 1
strcpy(copy, src);                   // writes the '\0' one byte past the end

/* FLAGGED: sizeof on a pointer, not the struct */
struct Config *cfg;
cfg = malloc(sizeof(cfg));           // 8 bytes (pointer size), not sizeof(*cfg)

/* FLAGGED: multiplication without overflow guard */
char *buf = malloc(count * item_size);  // may overflow size_t
```

## Remediation

```c
#include <stdlib.h>
#include <string.h>

/* SAFE: +1 for null terminator */
char *copy = malloc(strlen(src) + 1);
if (!copy) { /* handle OOM */ }
strcpy(copy, src);

/* Better: use strdup which handles this correctly */
char *copy2 = strdup(src);
if (!copy2) { /* handle OOM */ }

/* SAFE: dereference the pointer inside sizeof */
struct Config *cfg = malloc(sizeof(*cfg));
if (!cfg) { /* handle OOM */ }

/* SAFE: use calloc — it performs the overflow-safe multiply internally */
char *buf = calloc(count, item_size);
if (!buf) { /* handle OOM */ }

/* Or check for overflow explicitly before malloc */
if (count > SIZE_MAX / item_size) { /* error: would overflow */ }
char *buf2 = malloc(count * item_size);
```
