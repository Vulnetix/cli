---
title: "VNX-195 – Signed to Unsigned Conversion Error"
description: "Detect explicit or implicit conversion of negative signed values to unsigned types."
---

## Overview

Converting a negative signed integer to `size_t` or `unsigned` produces a very large positive value (e.g., -1 → SIZE_MAX). This is commonly used to bypass bounds checks: a negative `int` passes a `< maxLen` check when compared as signed, then when passed to `memcpy()` as `size_t` it requests a huge copy. This maps to [CWE-195: Signed to Unsigned Conversion Error](https://cwe.mitre.org/data/definitions/195.html).

**Severity:** Medium | **CWE:** [CWE-195](https://cwe.mitre.org/data/definitions/195.html)

## Remediation

```c
// SAFE: validate sign before using as unsigned
int len = get_user_length();
if (len < 0 || (size_t)len > sizeof(buf)) { return -1; }
memcpy(dest, src, (size_t)len);
```
