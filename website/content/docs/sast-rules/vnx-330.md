title: "VNX-330 – Use of Insufficiently Random Values"
description: "Detect non-cryptographic random number generators used for security-sensitive values such as tokens, session IDs, nonces, and passwords."
---

## Overview

This rule flags uses of non-cryptographic random number generators (RNGs) in contexts where unpredictability is a security requirement. Functions such as `Math.random()` (JavaScript), `random.randint()` (Python), `new Random()` (Java), `rand()` (PHP), and `math/rand` (Go) produce sequences that are statistically uniform but not cryptographically unpredictable — an attacker who observes a few outputs can reconstruct the internal state and predict all future values. This maps to [CWE-330: Use of Insufficiently Random Values](https://cwe.mitre.org/data/definitions/330.html).

**Severity:** High | **CWE:** [CWE-330 – Use of Insufficiently Random Values](https://cwe.mitre.org/data/definitions/330.html)

## Why This Matters

Weak random number generation has been the root cause of numerous high-profile vulnerabilities. The Debian OpenSSL vulnerability (CVE-2008-0166) reduced the key space for all generated keys to around 32,000 possibilities due to a broken RNG. Java's `java.util.Random` is a linear congruential generator (LCG) — given two consecutive outputs, an attacker can derive the seed and predict all subsequent values. PHP's `rand()` uses the system `rand()` which has a tiny state (often 32-bit). When weak RNGs are used to generate session tokens, password reset tokens, CSRF tokens, or nonces, an attacker can predict valid values without any server interaction and hijack accounts, bypass CSRF protection, or forge nonces.

## What Gets Flagged

- **Python**: `random.random(`, `random.randint(`, `random.choice(`, `random.uniform(`, `random.randrange(`
- **Java**: `new Random(`, `new java.util.Random(`, `Math.random(`
- **JavaScript/Node.js**: `Math.random(`
- **PHP**: `rand(`, `mt_rand(`, `array_rand(`, `shuffle(`
- **Ruby**: `Kernel.rand(`
- **Go**: `"math/rand"` import, `rand.Intn(`, `rand.Float64(`, `rand.Int(`

```javascript
// FLAGGED: Math.random() used for token generation
const token = Math.random().toString(36).substring(2);
const sessionId = Math.random().toString(16);
```

```python
# FLAGGED: random module for password generation
import random
password = ''.join(random.choices('abcdefghijklmnopqrstuvwxyz', k=16))
```

```go
// FLAGGED: math/rand for nonce generation
import "math/rand"
nonce := rand.Int63()
```

```java
// FLAGGED: java.util.Random for token
Random r = new Random();
String token = Long.toHexString(r.nextLong());
```

## Remediation

Use cryptographically secure random number generators for all security-sensitive values.

```go
// SAFE: crypto/rand in Go
import "crypto/rand"
import "encoding/hex"
b := make([]byte, 32)
rand.Read(b)
token := hex.EncodeToString(b)
```

```python
# SAFE: secrets module in Python (3.6+)
import secrets
token = secrets.token_hex(32)
session_id = secrets.token_urlsafe(32)
```

```javascript
// SAFE: crypto.randomBytes in Node.js
const crypto = require('crypto');
const token = crypto.randomBytes(32).toString('hex');
```

```java
// SAFE: SecureRandom in Java
import java.security.SecureRandom;
SecureRandom sr = new SecureRandom();
byte[] token = new byte[32];
sr.nextBytes(token);
```

```php
// SAFE: random_bytes in PHP 7+
$token = bin2hex(random_bytes(32));
```
