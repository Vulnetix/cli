title: "VNX-135 – Incorrect Calculation of Multi-Byte String Length"
description: "Detect strlen() used on wide-character (wchar_t) strings, which returns byte count instead of character count."
---

## Overview

This rule flags use of `strlen()` on wide-character (`wchar_t`) strings or in contexts involving multi-byte strings. `strlen()` counts bytes, not characters; for a `wchar_t` string where each character is 2 or 4 bytes, `strlen()` returns a count 2-4x smaller than the actual buffer size needed. Using this incorrect length in buffer allocations or copy operations causes truncation or buffer overflows. This maps to [CWE-135: Incorrect Calculation of Multi-Byte String Length](https://cwe.mitre.org/data/definitions/135.html).

**Severity:** Medium | **CWE:** [CWE-135 – Incorrect Calculation of Multi-Byte String Length](https://cwe.mitre.org/data/definitions/135.html)

## What Gets Flagged

```c
// FLAGGED: strlen() on wchar_t string
wchar_t wcs[] = L"Hello";
size_t len = strlen(wcs);  // wrong! returns bytes not chars

// FLAGGED: wchar_t with strlen
wchar_t wbuf[256];
int n = strlen(wbuf);
```

## Remediation

```c
// SAFE: use wcslen() for wide strings
wchar_t wcs[] = L"Hello";
size_t len = wcslen(wcs);  // correct character count

// When you need byte length for allocation:
size_t byte_len = (wcslen(wcs) + 1) * sizeof(wchar_t);
wchar_t *copy = malloc(byte_len);
```
