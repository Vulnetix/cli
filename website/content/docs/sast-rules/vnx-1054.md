## Overview

Commented-out code blocks are former implementations disabled rather than removed. These comments frequently contain old credentials, deprecated security bypass patterns (e.g. a temporarily removed permission check), or algorithms with known weaknesses. A developer can accidentally re-enable them by uncommenting, and they make security audits difficult.

**Severity:** Low | **CWE:** [CWE-1054 – Invokable Control Element with Excessive Volume of Commented-out Code](https://cwe.mitre.org/data/definitions/1054.html)

## Why This Matters

Commented-out code has contributed to real security incidents when developers re-enabled old authentication bypass code thinking it was a useful feature, and when code reviews failed to catch newly uncommented credential-handling code.

## What Gets Flagged

```python
# FLAGGED: Block of commented-out security code
# if password == old_password:
#     return admin_token
#     secret = "old_api_key_123"
# for user in all_users:
#     print(user.password)
```

## Remediation

Remove commented-out code entirely. Use version control to recover old implementations if needed. Never leave commented-out credential values in source code.
