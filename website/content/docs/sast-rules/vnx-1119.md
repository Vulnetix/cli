## Overview

VNX-1119 maps to [CWE-1119: Excessive Use of Unconditional Branching](https://cwe.mitre.org/data/definitions/1119.html). Detects source patterns associated with CWE-1119 (Excessive Use of Unconditional Branching). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1119](https://cwe.mitre.org/data/definitions/1119.html) | **Languages:** c

## Why This Matters

This weakness class (Excessive Use of Unconditional Branching) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1119 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'goto ' pattern
goto 
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1119 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1119` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
