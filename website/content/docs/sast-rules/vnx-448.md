## Overview

VNX-448 maps to [CWE-448: Obsolete Feature in UI](https://cwe.mitre.org/data/definitions/448.html). Detects source patterns associated with CWE-448 (Obsolete Feature in UI). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-448](https://cwe.mitre.org/data/definitions/448.html) | **Languages:** java

## Why This Matters

This weakness class (Obsolete Feature in UI) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-448 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains '@Deprecated' pattern
@Deprecated
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-448 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-448` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
