description: "Detect user-controlled values used as loop bounds without maximum validation, enabling denial-of-service."
---

## Overview

Using user-supplied input directly as a loop iteration count without validation allows attackers to cause excessive CPU consumption by supplying very large values. CWE-606 covers `range(int(request.GET['count']))` and similar patterns where no upper bound is enforced.

**Severity:** High | **CWE:** [CWE-606 – Unchecked Input for Loop Condition](https://cwe.mitre.org/data/definitions/606.html)

## Why This Matters

An attacker supplying `count=1000000000` for a loop that does any work per iteration can exhaust server CPU for an extended time, constituting a denial-of-service attack against the application.

## What Gets Flagged

```python
# FLAGGED: Unbounded loop from user input
for i in range(int(request.GET['count'])):
    process_item(i)
```

```java
// FLAGGED: Unbounded Java loop from request
for (int i = 0; i < Integer.parseInt(req.getParameter("n")); i++) {
    processItem(i);
}
```

## Remediation

Validate and cap user-supplied loop bounds:

```python
# SAFE: Cap at a safe maximum
MAX_ITEMS = 1000
count = min(int(request.GET.get('count', 10)), MAX_ITEMS)
for i in range(count):
    process_item(i)
```
