## Overview

VNX-606 maps to [CWE-606: Unchecked Input for Loop Condition](https://cwe.mitre.org/data/definitions/606.html). Detects source patterns associated with CWE-606 (Unchecked Input for Loop Condition). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-606](https://cwe.mitre.org/data/definitions/606.html) | **Languages:** c, cpp, go, java

## Why This Matters

This weakness class (Unchecked Input for Loop Condition) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-606 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'for(' pattern
for(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-606 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-606` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
