---
title: "VNX-791 – Incomplete Filtering of Special Elements"
description: "Detect single-pass string replacement of dangerous patterns like '<script>' that can be bypassed by nested input such as '<scr<script>ipt>'."
---

## Overview

This rule flags single-pass filtering that replaces dangerous substrings once. After `str.replace('<script>', '')` removes a `<script>` tag, the string `<scr<script>ipt>` becomes `<script>` — the attack survives. Attackers exploit this in any denylist sanitizer that doesn't apply replacements recursively until the output stabilizes. This maps to [CWE-791: Incomplete Filtering of Special Elements](https://cwe.mitre.org/data/definitions/791.html).

**Severity:** Medium | **CWE:** [CWE-791](https://cwe.mitre.org/data/definitions/791.html)

## Why This Matters

Single-pass replacements are almost always bypassable. The nested-tag trick (`<scr<script>ipt>`) is decades old and still effective against naive sanitizers deployed in production. The fix is not to apply the replacement more times, but to use an allowlist-based sanitizer that understands HTML structure.

## What Gets Flagged

```python
# FLAGGED: single-pass replace — bypassed by <scr<script>ipt>
clean = user_input.replace('<script>', '')
clean = user_input.replace('javascript:', '')
```

```javascript
// FLAGGED: replace without /g flag AND without case-insensitive
let clean = input.replace('<script>', '');  // only removes first occurrence
let clean = input.replace('javascript:', '');
```

```php
// FLAGGED: str_replace is single-pass
$clean = str_replace('<script>', '', $input);
```

## Remediation

```python
# SAFE: use a vetted HTML sanitizer with allowlist
import bleach
ALLOWED_TAGS = ['b', 'i', 'strong', 'em', 'a']
ALLOWED_ATTRS = {'a': ['href', 'title']}
clean = bleach.clean(user_input, tags=ALLOWED_TAGS, attributes=ALLOWED_ATTRS, strip=True)
```

```javascript
// SAFE: DOMPurify handles recursive nesting
const DOMPurify = require('dompurify');
const clean = DOMPurify.sanitize(userInput, { ALLOWED_TAGS: ['b', 'i', 'a'] });
```

```php
// SAFE: HTMLPurifier with strict configuration
$config = HTMLPurifier_Config::createDefault();
$config->set('HTML.Allowed', 'b,i,a[href]');
$purifier = new HTMLPurifier($config);
$clean = $purifier->purify($input);
```
