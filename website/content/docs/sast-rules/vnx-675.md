---
title: "VNX-675 – Multiple Operations on Resource in Single-Operation Context (Double-Free)"
description: "Detect double-free vulnerabilities where the same memory pointer is passed to free() twice."
---

## Overview

Calling `free()` on the same pointer twice corrupts the heap allocator's internal data structures. Modern heap implementations use these structures to track allocated chunks, and a double-free can overwrite arbitrary memory, enabling attackers to redirect code execution. Double-frees are a well-known exploitation primitive for memory corruption vulnerabilities.

**Severity:** Low | **CWE:** [CWE-675 – Multiple Operations on Resource in Single-Operation Context](https://cwe.mitre.org/data/definitions/675.html)

## Why This Matters

Heap corruption from double-free is exploitable to achieve arbitrary code execution. Exploits typically redirect control flow to attacker-controlled shellcode or ROP chains. Real-world CVEs from double-free vulnerabilities include remote code execution in network daemons, browsers, and system libraries. Modern allocators (tcmalloc, jemalloc) include some double-free detection, but this is not reliable and should not be relied upon.

## What Gets Flagged

```c
// FLAGGED: free() called twice on the same pointer
char *buf = malloc(256);
process(buf);
free(buf);  // first free
// ... later ...
free(buf);  // DOUBLE FREE — heap corruption

// FLAGGED: free in error path when buf is already freed in success path
if (result < 0) {
    free(buf);  // freed here on error
}
free(buf);  // and unconditionally here — double free on error
```

## Remediation

Set pointers to NULL after freeing and check before freeing:

```c
// SAFE: NULL pointer after free prevents double-free
char *buf = malloc(256);
process(buf);
free(buf);
buf = NULL;  // subsequent free(NULL) is a safe no-op per C standard

// SAFE: Single exit point or explicit NULL check
void cleanup(char *ptr) {
    if (ptr != NULL) {
        free(ptr);
    }
}
```

Use smart pointers in C++ (`std::unique_ptr`, `std::shared_ptr`) which automatically manage memory and prevent double-free:

```cpp
// SAFE: unique_ptr automatically frees when it goes out of scope
{
    auto buf = std::make_unique<char[]>(256);
    process(buf.get());
}  // automatically freed once, no double-free possible
```
