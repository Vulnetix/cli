---
title: "VNX-226 – Sensitive Information in Resource Not Removed Before Reuse"
description: "Detect heap buffers containing passwords or keys freed without prior zeroing."
---

## Overview

When a heap buffer containing a password, key, or other credential is freed without zeroing, the allocator may return the same memory to a subsequent allocation, exposing the credential to another part of the application. An attacker who can trigger controlled heap allocation can read residual secrets. This maps to [CWE-226: Sensitive Information in Resource Not Removed Before Reuse](https://cwe.mitre.org/data/definitions/226.html).

**Severity:** Medium | **CWE:** [CWE-226](https://cwe.mitre.org/data/definitions/226.html)

## Remediation

```c
// SAFE: zero before free
memset_s(password_buf, sizeof(password_buf), 0, sizeof(password_buf));
free(password_buf);

// Or: explicit_bzero (not optimized away by compiler)
explicit_bzero(key_buf, key_len);
free(key_buf);
```
