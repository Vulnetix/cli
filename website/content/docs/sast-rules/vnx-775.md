title: "VNX-775 – Missing Release of File Descriptor or Handle"
description: "Detect fopen/open/os.Open calls without matching close calls, causing file descriptor leaks that exhaust process limits."
---

## Overview

This rule flags file open operations (`fopen`, `open`, `os.Open`) in C, Python, and Go where no matching close call is found in the same file or immediately after the open. This maps to [CWE-775: Missing Release of File Descriptor or Handle after Effective Lifetime](https://cwe.mitre.org/data/definitions/775.html).

**Severity:** Medium | **CWE:** [CWE-775](https://cwe.mitre.org/data/definitions/775.html)

## Why This Matters

Each leaked file descriptor brings the process one step closer to hitting the OS fd limit. When the limit is reached, all subsequent open/socket/pipe calls fail, preventing the service from accepting connections, opening log files, or performing any I/O. Deliberately triggering this condition is a denial of service attack.

## What Gets Flagged

```c
// FLAGGED: fopen without fclose in file
FILE *f = fopen("data.bin", "rb");
fread(buf, 1, 256, f);
// fclose(f) missing
```

```python
# FLAGGED: open() without with or .close()
f = open(path)
data = f.read()
return data  # f never closed
```

```go
// FLAGGED: os.Open without defer Close
f, err := os.Open(path)
if err != nil { return err }
// defer f.Close() missing
data, _ := io.ReadAll(f)
```

## Remediation

```c
// SAFE: always fclose() in a cleanup path
FILE *f = fopen("data.bin", "rb");
if (f == NULL) { return -1; }
size_t n = fread(buf, 1, 256, f);
fclose(f);
```

```python
# SAFE: context manager
with open(path) as f:
    data = f.read()
```

```go
// SAFE: defer immediately after error check
f, err := os.Open(path)
if err != nil { return err }
defer f.Close()
```
