## Overview

Detects error handling code that lacks documentation about what the error means and how it should be interpreted by callers. Undocumented error handling in security code can cause callers to misinterpret error conditions as success.

**Severity:** Low | **CWE:** [CWE-1118](https://cwe.mitre.org/data/definitions/1118.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1118](https://cwe.mitre.org/data/definitions/1118.html) for authoritative guidance.
