## Overview

VNX-391 maps to [CWE-391: Unchecked Error Condition](https://cwe.mitre.org/data/definitions/391.html). Detects source patterns associated with CWE-391 (Unchecked Error Condition). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-391](https://cwe.mitre.org/data/definitions/391.html) | **Languages:** java, python

## Why This Matters

This weakness class (Unchecked Error Condition) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-391 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'catch(Exception e) {}' pattern
catch(Exception e) {}
```

```python
// FLAGGED: contains 'except:' pattern
except:
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-391 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-391` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
