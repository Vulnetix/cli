---
title: "VNX-1069 – Empty Exception Block"
description: "Detect empty catch/except blocks that silently swallow exceptions and hide security failures."
---

## Overview

Detects empty exception handlers — `catch (Exception e) {}` in Java/JavaScript and `except: pass` in Python. Empty exception handlers hide failures, mask security violations, prevent proper logging of attacks, and allow programs to continue in undefined state.

**Severity:** Medium | **CWE:** [CWE-1069 – Empty Exception Block](https://cwe.mitre.org/data/definitions/1069.html)

## Why This Matters

An empty catch block on an authentication function means a database failure or network error is silently treated as success or failure without logging. Attackers can exploit infrastructure errors to trigger inconsistent authentication states.

## What Gets Flagged

```python
# FLAGGED: bare except with pass
try:
    result = authenticate(user, password)
except:
    pass  # authentication failure hidden!

# FLAGGED: specific exception swallowed
try:
    validate_token(token)
except Exception as e:
    pass
```

```java
// FLAGGED: empty catch block
try {
    verifySignature(data);
} catch (Exception e) {
    // nothing - signature failure silently ignored!
}
```

## Remediation

```python
# SAFE: log and re-raise or handle properly
try:
    result = authenticate(user, password)
except AuthenticationError as e:
    logger.warning("Authentication failed: %s", str(e))
    return False
except Exception as e:
    logger.error("Unexpected error during authentication: %s", str(e))
    raise  # unexpected errors should propagate
```
