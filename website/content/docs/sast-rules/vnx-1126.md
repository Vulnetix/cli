## Overview

Detects sensitive variables (passwords, tokens, keys) declared at function scope when they are only needed in a narrower inner scope. Wider scope means sensitive data lives longer in memory, increasing the window for exposure through memory dumps, debugging, or logging.

**Severity:** Low | **CWE:** [CWE-1126](https://cwe.mitre.org/data/definitions/1126.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1126](https://cwe.mitre.org/data/definitions/1126.html) for authoritative guidance.
