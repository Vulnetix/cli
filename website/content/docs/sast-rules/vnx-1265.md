## Overview

VNX-1265 maps to [CWE-1265: Unintended Reentrant Invocation of Non-reentrant Code Via Nested Calls](https://cwe.mitre.org/data/definitions/1265.html). Detects source patterns associated with CWE-1265 (Unintended Reentrant Invocation of Non-reentrant Code Via Nested Calls). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-1265](https://cwe.mitre.org/data/definitions/1265.html) | **Languages:** c

## Why This Matters

This weakness class (Unintended Reentrant Invocation of Non-reentrant Code Via Nested Calls) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1265 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'signal(SIGCHLD' pattern
signal(SIGCHLD
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1265 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1265` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
