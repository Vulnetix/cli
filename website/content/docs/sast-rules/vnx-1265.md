title: "VNX-1265 – Unintended Reentrant Invocation of Non-reentrant Code"
description: "Detect signal handlers and interrupt service routines that call non-async-signal-safe functions, which can cause crashes, data corruption, or exploitable race conditions if the signal fires during the function's execution."
---

## Overview

POSIX signal handlers run asynchronously, interrupting whatever code was executing. If a signal fires while `malloc()` is in the middle of manipulating heap structures and the signal handler also calls `malloc()`, the heap becomes corrupted. Non-reentrant functions — those that use global state, locks, or stdio — must never be called from signal handlers. **Severity:** Medium | **CWE:** [CWE-1265 – Unintended Reentrant Invocation of Non-reentrant Code Via Nested Calls](https://cwe.mitre.org/data/definitions/1265.html)

## Why This Matters

Signal handler bugs are notoriously difficult to reproduce but have been exploited in practice. CVE-2014-6271 (Shellshock) involved signal handling in bash. Many Linux privilege escalation exploits leverage SIGSEGV or other signals to corrupt memory in ways that redirect execution. Python's asyncio and threading code can also exhibit reentrancy issues when signal handlers modify shared state.

## What Gets Flagged

```c
// FLAGGED: signal() registration — review handler body
signal(SIGINT, my_handler);

// If my_handler calls printf(), malloc(), or any non-async-safe function,
// it is vulnerable to reentrancy corruption
void my_handler(int sig) {
    printf("Received signal %d\n", sig);  // NOT async-signal-safe
    free(global_buffer);  // NOT async-signal-safe
}
```

```python
# FLAGGED: signal handler that does complex operations
import signal

def handle_shutdown(signum, frame):
    time.sleep(1)  # blocking in signal handler
    requests.post(url, data=state)  # I/O in signal handler
```

## Remediation

```c
// SAFE: Signal handler sets a volatile flag only
volatile sig_atomic_t g_shutdown = 0;

void handle_signal(int sig) {
    g_shutdown = 1;  // async-signal-safe write
}

// Main loop checks the flag
while (!g_shutdown) {
    do_work();
}
cleanup_and_exit();
```

```python
# SAFE: Python signal handler sets a flag, main loop handles it
import signal

_shutdown = False

def handle_shutdown(signum, frame):
    global _shutdown
    _shutdown = True  # simple assignment is safe

signal.signal(signal.SIGTERM, handle_shutdown)

while not _shutdown:
    process_next_item()
```
