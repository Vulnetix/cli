---
title: "VNX-1041 – Use of Redundant Code"
description: "Detect duplicate security checks that may become inconsistent when one copy is updated without the other."
---

## Overview

Duplicate security checks produced by copy-paste indicate that the check is not consolidated into a single reusable function. When developers later update the security check (e.g. to add rate limiting or a new permission), they may update only one copy, creating inconsistent enforcement.

**Severity:** Low | **CWE:** [CWE-1041 – Use of Redundant Code](https://cwe.mitre.org/data/definitions/1041.html)

## Why This Matters

Copy-pasted security checks drift apart over time. One copy gets patched, the other does not, creating an exploitable path that appears to be protected.

## What Gets Flagged

```python
# FLAGGED: Same authorization check appears twice within 10 lines
if not request.user.is_authenticated:
    return HttpResponseForbidden()
# ... some code ...
if not request.user.is_authenticated:  # duplicate
    return HttpResponseForbidden()
```

## Remediation

Extract the check into a decorator, middleware, or helper function:

```python
# SAFE: Single authoritative check
@login_required
def sensitive_view(request):
    ...
```
