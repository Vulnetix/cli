---
title: "VNX-1220 – Insufficient Granularity of Access Control"
description: "Detect coarse-grained access control using only is_admin/role checks without fine-grained per-resource permission validation, which allows any admin to perform all administrative actions regardless of intended scope."
---

## Overview

Access control systems that distinguish only between "admin" and "non-admin" (or "superuser" vs "user") provide insufficient granularity for security. When any admin can perform any administrative action, compromise of a single admin account or role grants full control. Fine-grained access control assigns specific permissions per action and resource. **Severity:** Medium | **CWE:** [CWE-1220 – Insufficient Granularity of Access Control](https://cwe.mitre.org/data/definitions/1220.html)

## Why This Matters

The principle of least privilege requires users to have only the permissions needed for their specific role. A customer service admin should not automatically have database export permissions. A content editor should not have user management access. When roles are too broad, a single phishing attack or insider threat can compromise the entire system. OWASP API Security A1:2023 "Broken Object Level Authorization" is closely related.

## What Gets Flagged

```python
# FLAGGED: Only is_admin check, no per-resource permission
def delete_user(request, user_id):
    if not request.user.is_admin:
        raise PermissionDenied
    User.objects.get(id=user_id).delete()

# FLAGGED: Broad staff check without object-level permission
@staff_member_required
def export_all_data(request):
    return export_database()
```

```javascript
// FLAGGED: Binary admin check without action-level permission
function deleteRecord(req, res) {
    if (!req.user.isAdmin) {
        return res.status(403).json({ error: 'Forbidden' });
    }
    Record.delete(req.params.id);
}
```

## Remediation

```python
# SAFE: Fine-grained permission check per action
from django.contrib.auth.decorators import permission_required

@permission_required('myapp.delete_user')
def delete_user(request, user_id):
    User.objects.get(id=user_id).delete()

# SAFE: Object-level permission with django-guardian
from guardian.decorators import permission_required_or_403

@permission_required_or_403('myapp.change_document', (Document, 'pk', 'pk'))
def edit_document(request, pk):
    doc = Document.objects.get(pk=pk)
    # ...
```

```javascript
// SAFE: Action-specific permission check using CASL or similar
const { subject, defineAbility } = require('@casl/ability');

const ability = defineAbility((can, cannot) => {
    can('read', 'Article');
    can('manage', 'Article', { authorId: user.id });
    cannot('delete', 'Article');
});

if (ability.can('delete', subject('Record', record))) {
    record.delete();
} else {
    return res.status(403).json({ error: 'Insufficient permissions' });
}
```
