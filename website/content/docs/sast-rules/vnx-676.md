---
title: "VNX-676 – Use of Potentially Dangerous Function"
description: "Detect calls to functions with well-known unsafe characteristics such as gets, strcpy, eval, exec, system, and passthru."
---

## Overview

Certain functions are inherently dangerous regardless of how they are called. `gets()` has no bounds checking, `strcpy()` does not check destination length, `eval()` and `exec()` execute arbitrary code, and `system()`/`passthru()` invoke shell commands. Their presence in source code constitutes a vulnerability or at minimum a strong indicator of poor security hygiene.

**Severity:** High | **CWE:** [CWE-676 – Use of Potentially Dangerous Function](https://cwe.mitre.org/data/definitions/676.html)

## Why This Matters

Buffer overflow attacks via `gets()`/`strcpy()` have powered some of the most significant historical exploits. Dynamic code evaluation via `eval()` is the root cause of many remote code execution vulnerabilities in web applications.

## What Gets Flagged

```c
// FLAGGED: No bounds checking
char buf[64];
gets(buf);
strcpy(dest, src);
```

```php
<?php
// FLAGGED: Code/command execution
eval($user_input);
system($cmd);
passthru($input);
```

```python
# FLAGGED: Arbitrary code execution
eval(user_data)
exec(user_code)
```

## Remediation

Replace dangerous functions with safe alternatives:

```c
// SAFE: Bounds-checked alternatives
fgets(buf, sizeof(buf), stdin);
strncpy(dest, src, sizeof(dest) - 1);
snprintf(buf, sizeof(buf), fmt, args);
```

For Python/PHP, avoid eval/exec entirely; use data structures or AST parsers.
