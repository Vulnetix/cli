---
title: "VNX-940 – Improper Verification of Source of a Communication Channel"
description: "Detect message handlers and webhook endpoints that do not verify the origin or sender of incoming data."
---

## Overview

When an application accepts messages (postMessage, WebSocket, webhooks) without verifying the sender's identity, an attacker can send spoofed messages from a malicious origin. For webhooks, any system can send requests to the endpoint and trigger privileged operations.

**Severity:** Medium | **CWE:** [CWE-940 – Improper Verification of Source of a Communication Channel](https://cwe.mitre.org/data/definitions/940.html)

## Why This Matters

A postMessage handler without an origin check accepts data from any page including attacker-controlled pages. A webhook without signature verification can be triggered by attackers to inject events.

## What Gets Flagged

```javascript
// FLAGGED: No origin check
window.addEventListener('message', function(event) {
    processData(event.data);  // event.origin not checked
});
```

```python
# FLAGGED: Webhook without signature check
@app.route('/webhook', methods=['POST'])
def webhook():
    process(request.json)  # no signature verification
```

## Remediation

Verify origin/signature before processing:

```javascript
// SAFE
window.addEventListener('message', function(event) {
    if (event.origin !== 'https://trusted.example.com') return;
    processData(event.data);
});
```

```python
# SAFE: Verify GitHub webhook signature
sig = request.headers.get('X-Hub-Signature-256')
if not verify_signature(sig, request.data, SECRET): abort(403)
```
