---
title: "VNX-551 – Authorization Before URL Canonicalization"
description: "Detect authorization checks performed on raw/uncanonical URLs before path normalization, enabling bypass via encoded paths."
---

## Overview

When access control checks use the raw request URI before canonicalization, attackers can bypass them using URL-encoded, double-encoded, or non-standard path representations. A check for `/admin` fails to block `/%61dmin` or `//admin`.

**Severity:** Medium | **CWE:** [CWE-551 – Incorrect Behavior Order: Authorization Before Parsing and Canonicalization](https://cwe.mitre.org/data/definitions/551.html)

## Why This Matters

URL canonicalization bypass is a well-known attack class. Apache Tomcat, IIS, and various web frameworks have had vulnerabilities where URL-encoded paths bypassed filter-based access control. Always normalize URLs before performing authorization decisions.

## What Gets Flagged

```java
// FLAGGED: Auth check on raw URI before decode
String uri = request.getRequestURI();
if (uri.startsWith("/admin")) {
    checkAdminAuth(request);
}
```

```javascript
// FLAGGED: Raw path check in Express middleware
if (req.path.startsWith('/admin')) {
    requireAuth(req, res, next);
}
```

## Remediation

Canonicalize paths before checking:

```java
// SAFE: Decode and normalize before checking
String uri = URLDecoder.decode(request.getRequestURI(), "UTF-8");
uri = Paths.get(uri).normalize().toString();
if (uri.startsWith("/admin")) {
    checkAdminAuth(request);
}
```

Use framework-level middleware that handles canonicalization before routing.
