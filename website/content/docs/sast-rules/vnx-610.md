---
title: "VNX-610 – Externally Controlled Reference to a Resource"
description: "Detect user-controlled input used to construct file paths, URLs, or resource identifiers without validation."
---

## Overview

When user input directly determines which resource to access (file path, URL, database record), an attacker can manipulate the reference to access unintended resources — traversing directories, triggering SSRF, or enumerating internal data.

**Severity:** Medium | **CWE:** [CWE-610 – Externally Controlled Reference to a Resource in Another Sphere](https://cwe.mitre.org/data/definitions/610.html)

## Why This Matters

Path traversal, SSRF (Server-Side Request Forgery), and IDOR (Insecure Direct Object Reference) all stem from this pattern. An attacker can read arbitrary files, access internal services, or retrieve other users' data.

## What Gets Flagged

```python
# FLAGGED: File path from user input
open(request.GET['filename'])

# FLAGGED: SSRF - URL from user input
requests.get(request.GET['url'])
```

## Remediation

Validate against an allowlist and use canonical path checking:

```python
# SAFE: Validate against allowlist
ALLOWED_FILES = {'report.pdf', 'invoice.pdf'}
filename = request.GET.get('filename', '')
if filename not in ALLOWED_FILES:
    return HttpResponseForbidden()
open(os.path.join(BASE_DIR, filename))

# SAFE: URL allowlist for outbound requests
ALLOWED_HOSTS = {'api.trusted.com', 'data.trusted.com'}
parsed = urllib.parse.urlparse(url)
if parsed.hostname not in ALLOWED_HOSTS:
    raise ValueError("Disallowed host")
```
