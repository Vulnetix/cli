## Overview

Detects patterns where a method directly accesses internal members of another class (friend access, reflection, accessing private fields). This violates encapsulation and can bypass security invariants maintained by the enclosing class's public interface.

**Severity:** Low | **CWE:** [CWE-1090](https://cwe.mitre.org/data/definitions/1090.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1090](https://cwe.mitre.org/data/definitions/1090.html) for authoritative guidance.
