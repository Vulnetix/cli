## Overview

Detects mutable global variables used to store security-sensitive state such as authentication status, current user identity, or session data. Global security state is vulnerable to race conditions in multi-threaded environments and unintended mutation.

**Severity:** Medium | **CWE:** [CWE-1108 – Excessive Reliance on Global Variables](https://cwe.mitre.org/data/definitions/1108.html)

## Why This Matters

In a Node.js server, a module-level `var currentUser` is shared across all concurrent requests. Request A sets `currentUser = userA`, then Request B sets `currentUser = userB` before Request A completes. Request A then operates on Request B's user, causing privilege confusion or data leakage.

## What Gets Flagged

```python
# FLAGGED: global authentication state
global authenticated
global current_user
authenticated = True
current_user = user
```

```javascript
// FLAGGED: module-level security state
var currentUser = null;
var authenticated = false;
```

## Remediation

```python
# SAFE: use Flask/Django request context
from flask import g
g.current_user = user
g.authenticated = True
```

```javascript
// SAFE: use request-scoped middleware
app.use((req, res, next) => {
    req.currentUser = null;
    req.authenticated = false;
    next();
});
```
