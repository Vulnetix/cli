title: "VNX-602 – Client-Side Enforcement of Server-Side Security"
description: "Detect security controls implemented only in JavaScript that can be bypassed by disabling scripts or crafting direct HTTP requests."
---

## Overview

Security checks implemented exclusively in client-side JavaScript can be bypassed by any user. Disabling JavaScript, using browser developer tools, or sending direct HTTP requests bypasses all client-side validation. Authentication, authorization, and input validation must always be enforced server-side.

**Severity:** Medium | **CWE:** [CWE-602 – Client-Side Enforcement of Server-Side Security](https://cwe.mitre.org/data/definitions/602.html)

## Why This Matters

Client-side validation is useful for UX but provides zero security. An attacker using `curl` or Burp Suite can submit any data directly to the server, bypassing all JavaScript checks entirely.

## What Gets Flagged

```javascript
// FLAGGED: Authorization from localStorage
if (localStorage.getItem('role') === 'admin') {
    showAdminPanel();
}
```

```html
<!-- FLAGGED: Security role in hidden form field -->
<input type="hidden" name="is_admin" value="0" />
```

## Remediation

Enforce all security decisions server-side:

```javascript
// SAFE: UI hint only, server enforces
if (userRole === 'admin') {
    showAdminButton();  // UI convenience only
}
// The actual admin API endpoints check the session server-side
```
