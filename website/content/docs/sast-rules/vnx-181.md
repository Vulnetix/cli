## Overview

Context-dependent canonicalization (Unicode normalization, charset conversion) must happen before security validation. Homoglyph characters or alternate Unicode representations can pass a validation check but resolve to a dangerous character after normalization. This maps to [CWE-181](https://cwe.mitre.org/data/definitions/181.html).

**Severity:** Low | **CWE:** [CWE-181](https://cwe.mitre.org/data/definitions/181.html)

## Remediation

Apply `unicodedata.normalize('NFKC', input)` or equivalent before validation to ensure the normalized form is checked, not the raw input.
