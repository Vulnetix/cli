## Overview

Detects database table definitions with an excessive number of indices (more than 8). Too many indices slow down write operations, consume storage, and can indicate poorly designed schemas that are hard to reason about from a data-access security perspective.

**Severity:** Low | **CWE:** [CWE-1089](https://cwe.mitre.org/data/definitions/1089.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1089](https://cwe.mitre.org/data/definitions/1089.html) for authoritative guidance.
