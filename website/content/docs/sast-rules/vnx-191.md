---
title: "VNX-191 – Integer Underflow (Wrap or Wraparound)"
description: "Detect unsigned integer subtraction that wraps to a large value when the result would be negative."
---

## Overview

Unsigned integer subtraction wraps around when the minuend is smaller than the subtrahend. `size_t len = end - start` wraps to ~SIZE_MAX if `start > end`, causing subsequent `malloc(len)` or `memcpy(dest, src, len)` to operate on a gigantic size. This maps to [CWE-191: Integer Underflow](https://cwe.mitre.org/data/definitions/191.html).

**Severity:** High | **CWE:** [CWE-191](https://cwe.mitre.org/data/definitions/191.html)

## Remediation

```c
// SAFE: guard before subtraction
if (end < start) { return -1; }
size_t len = end - start;
```
