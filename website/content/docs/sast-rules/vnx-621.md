---
title: "VNX-621 – Variable Extraction Error (PHP extract Injection)"
description: "Detect PHP extract() calls on user-supplied superglobals that allow attackers to overwrite arbitrary variables."
---

## Overview

PHP's `extract()` function imports all keys of an associative array as variable names into the current symbol table. When called with `$_GET`, `$_POST`, or `$_REQUEST`, any parameter name supplied by the attacker becomes a PHP variable, allowing them to overwrite internal authentication flags, file paths, and SQL query fragments.

**Severity:** High | **CWE:** [CWE-621 – Variable Extraction Error](https://cwe.mitre.org/data/definitions/621.html)

## Why This Matters

An attacker can send `?is_admin=1` or `?query=DROP+TABLE` to overwrite any in-scope variable. This commonly leads to authentication bypass, SQL injection, or file inclusion.

## What Gets Flagged

```php
<?php
// FLAGGED: imports all GET params as variables
extract($_GET);
extract($_POST);
extract($_REQUEST);
```

## Remediation

Never pass superglobals to `extract()`. Assign variables explicitly:

```php
<?php
// SAFE: explicit assignment with type validation
$page = (int)$_GET['page'];
$name = htmlspecialchars($_GET['name'], ENT_QUOTES);
```
