---

## Overview

This rule flags web server configuration and static file serving code that may inadvertently expose sensitive files to the internet. Serving the application's root directory, enabling directory listing, or misconfiguring static file paths can expose `.env` files with credentials, `.git` directories with full source history, database configuration files, and internal configuration data. This maps to [CWE-552: Files or Directories Accessible to External Parties](https://cwe.mitre.org/data/definitions/552.html).

**Severity:** High | **CWE:** [CWE-552 – Files or Directories Accessible to External Parties](https://cwe.mitre.org/data/definitions/552.html)

## Why This Matters

Exposed `.env` files are one of the most common causes of cloud credential leaks, with automated scanners continuously crawling the internet for them. An accessible `.git` directory leaks the entire commit history, including previously deleted secrets, TODO comments describing vulnerabilities, and the full source code. A 2021 analysis by GitGuardian found over 6 million `.env` files exposed on public web servers. Directory listing enables trivial enumeration of all files in a directory, which may include backup files, temporary uploads, and internal documents. MITRE ATT&CK T1083 covers file and directory discovery used to map the target environment.

## What Gets Flagged

```javascript
// FLAGGED: express.static serving the project root — exposes .env, node_modules, etc.
const express = require('express');
const app = express();
app.use(express.static(__dirname));  // dangerous: serves everything

// FLAGGED: serve-static with current directory
app.use(serveStatic(__dirname));
```

```python
# FLAGGED: Django MEDIA_ROOT pointing to source code directory
# settings.py
MEDIA_ROOT = BASE_DIR  # exposes all source files as media

MEDIA_ROOT = os.path.join(BASE_DIR, '')  # still exposes source

# FLAGGED: STATICFILES_DIRS including the project root
STATICFILES_DIRS = [BASE_DIR]  # serves .env, settings.py, etc. as static files
```

```apache
# FLAGGED: Apache directory listing enabled
Options +Indexes

# FLAGGED: Allowing access from all — missing deny rules for sensitive paths
allow from all
```

```nginx
# FLAGGED: Nginx autoindex enabled
autoindex on;
```

## Remediation

### Node.js / Express

```javascript
// SAFE: Serve only a dedicated public directory, never the project root
const path = require('path');
const express = require('express');
const app = express();

// Create a dedicated 'public' directory containing only files safe to serve
app.use(express.static(path.join(__dirname, 'public')));

// SAFE: Block access to sensitive file patterns with middleware
app.use((req, res, next) => {
    const blocked = ['.env', '.git', '.htaccess', 'config.json', 'package.json'];
    const basename = path.basename(req.path);
    if (blocked.some(b => basename === b || req.path.includes('/.git/'))) {
        return res.status(403).send('Forbidden');
    }
    next();
});
```

### Python / Django

```python
# settings.py
# SAFE: MEDIA_ROOT and STATICFILES_DIRS point to dedicated subdirectories
MEDIA_ROOT = os.path.join(BASE_DIR, 'media')     # only uploaded media files
STATIC_ROOT = os.path.join(BASE_DIR, 'staticfiles')  # collected static assets
STATICFILES_DIRS = [
    os.path.join(BASE_DIR, 'static'),  # only the 'static' subdirectory
]
# NEVER include BASE_DIR itself in STATICFILES_DIRS
```

### Apache

```apache
# SAFE: Disable directory listing globally, then enable only where needed
Options -Indexes

# SAFE: Deny access to sensitive files and directories
<FilesMatch "^\.">
    Require all denied
</FilesMatch>

<DirectoryMatch "^.*/(\.git|\.env|node_modules|vendor)/">
    Require all denied
</DirectoryMatch>

<FilesMatch "\.(env|config|yml|yaml|json|lock|sum)$">
    Require all denied
</FilesMatch>
```

### Nginx

```nginx
# SAFE: Disable autoindex
autoindex off;

# SAFE: Block sensitive file access
location ~ /\. {
    deny all;
    return 404;
}

location ~* \.(env|config|yml|yaml|json|lock|sql)$ {
    deny all;
    return 404;
}

# SAFE: Only serve from the webroot directory, not the application root
root /var/www/myapp/public;
```

### General File System Layout

Separate web-accessible files from application code:

```
myapp/
├── app/           # application source — NEVER web-accessible
│   ├── .env       # secrets
│   ├── config/
│   └── src/
└── public/        # web root — only safe-to-serve assets
    ├── index.html
    ├── css/
    ├── js/
    └── images/
```
