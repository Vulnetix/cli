## Overview

VNX-805 maps to [CWE-805: Buffer Access with Incorrect Length Value](https://cwe.mitre.org/data/definitions/805.html). Detects source patterns associated with CWE-805 (Buffer Access with Incorrect Length Value). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-805](https://cwe.mitre.org/data/definitions/805.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Buffer Access with Incorrect Length Value) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-805 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

// FLAGGED: contains 'memcpy(' pattern

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-805 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-805` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
