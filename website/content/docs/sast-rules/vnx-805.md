---
title: "VNX-805 – Buffer Access with Incorrect Length Value"
description: "Detect C string/memory functions called with length arguments derived from the wrong buffer, enabling partial copies or buffer overflows."
---

## Overview

This rule flags C/C++ memory and string operations where the length argument comes from the wrong source — for example, `strncpy(dest, src, strlen(src))` omits the null terminator, and `strncat(dest, src, sizeof(dest))` ignores what is already in the destination. Using the wrong length leads to buffer overflows or unterminated strings. This maps to [CWE-805: Buffer Access with Incorrect Length Value](https://cwe.mitre.org/data/definitions/805.html).

**Severity:** High | **CWE:** [CWE-805](https://cwe.mitre.org/data/definitions/805.html)

## Why This Matters

Off-by-one errors in length calculations are among the most common sources of stack and heap buffer overflows. `strncpy(dst, src, strlen(src))` copies exactly `strlen(src)` bytes, leaving the destination without a null terminator if the buffer is exactly that size. `strncat(dst, src, sizeof(dst))` ignores the bytes already written to `dst`, potentially writing past the end.

## What Gets Flagged

```c
// FLAGGED: strncpy with strlen(src) — no null terminator
char dest[64];
strncpy(dest, src, strlen(src));  // should be sizeof(dest) - 1

// FLAGGED: strncat with sizeof(dest) — ignores current length
char buf[128] = "prefix_";
strncat(buf, suffix, sizeof(buf));  // should be sizeof(buf) - strlen(buf) - 1

// FLAGGED: memcpy with strlen — may be wrong for binary data
memcpy(dest, src, strlen(src));

// FLAGGED: gets — no length limit at all
char buf[256];
gets(buf);
```

## Remediation

```c
// SAFE: strncpy with destination size minus 1, then explicit null termination
char dest[64];
strncpy(dest, src, sizeof(dest) - 1);
dest[sizeof(dest) - 1] = '\0';

// SAFE: strncat accounting for existing content
char buf[128] = "prefix_";
strncat(buf, suffix, sizeof(buf) - strlen(buf) - 1);

// SAFE: memcpy with explicit length for binary data
size_t copy_len = MIN(src_len, sizeof(dest));
memcpy(dest, src, copy_len);

// SAFE: fgets instead of gets
char buf[256];
fgets(buf, sizeof(buf), stdin);
buf[strcspn(buf, "\n")] = '\0';
```

For new code, prefer `strlcpy` and `strlcat` (available on BSD/macOS, or via libbsd on Linux) which always null-terminate and return the required length:

```c
// SAFE: strlcpy always null-terminates
strlcpy(dest, src, sizeof(dest));
strlcat(buf, suffix, sizeof(buf));
```
