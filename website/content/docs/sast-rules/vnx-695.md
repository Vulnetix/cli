## Overview

VNX-695 maps to [CWE-695: Use of Low-Level Functionality](https://cwe.mitre.org/data/definitions/695.html). Detects source patterns associated with CWE-695 (Use of Low-Level Functionality). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-695](https://cwe.mitre.org/data/definitions/695.html) | **Languages:** c

## Why This Matters

This weakness class (Use of Low-Level Functionality) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-695 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains '__asm__' pattern
__asm__
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-695 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-695` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
