---
title: "VNX-197 – Numeric Truncation Error"
description: "Detect narrowing casts (int to byte/short) on user-supplied values that silently discard bits."
---

## Overview

Casting a user-supplied value to a narrower type silently discards the upper bits. A value of 256 cast to `byte` in Java becomes 0; 257 becomes 1. This can bypass range checks if the checked value is the wide type but the used value is the truncated type. This maps to [CWE-197: Numeric Truncation Error](https://cwe.mitre.org/data/definitions/197.html).

**Severity:** Medium | **CWE:** [CWE-197](https://cwe.mitre.org/data/definitions/197.html)

## Remediation

```java
// SAFE: range check before narrowing cast
int val = Integer.parseInt(request.getParameter("port"));
if (val < 0 || val > 65535) throw new IllegalArgumentException("Invalid port");
short port = (short) val;
```
