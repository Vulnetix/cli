## Overview

Data structures that nest collections within collections (e.g. `Map<String, List<Map<String, Object>>>`) create unbounded memory and complexity risks. When populated from user-controlled input, these structures can grow without limit, exhausting memory or making it impossible to audit all fields for sensitive data exposure.

**Severity:** Low | **CWE:** [CWE-1043 – Data Element Aggregating an Excessively Large Number of Non-Primitive Elements](https://cwe.mitre.org/data/definitions/1043.html)

## Why This Matters

Deeply nested data structures are hard to audit for PII exposure, make serialization/deserialization unpredictable, and can exhaust resources when deeply nested.

## What Gets Flagged

```java
// FLAGGED: Doubly-nested collection
Map<String, List<Map<String, Object>>> nestedData;
List<List<String>> matrixData;
```

## Remediation

Flatten the data model or introduce typed value objects with explicit field bounds.
