---
title: "VNX-140 – Improper Neutralization of Delimiters (CSV Injection)"
description: "Detect CSV output that may contain unescaped formula trigger characters (=, +, -, @) from user-supplied data."
---

## Overview

This rule flags CSV generation functions (`csv.writer`, `fputcsv`, `csvWriter.writeNext`, CSV stringify) that may write user-controlled data without stripping or escaping leading formula characters (`=`, `+`, `-`, `@`). When a user opens the CSV in a spreadsheet application like Excel or LibreOffice Calc, cells starting with these characters are interpreted as formulas, enabling data exfiltration or remote code execution via DDE payloads. This maps to [CWE-140: Improper Neutralization of Delimiters](https://cwe.mitre.org/data/definitions/140.html).

**Severity:** Medium | **CWE:** [CWE-140 – Improper Neutralization of Delimiters](https://cwe.mitre.org/data/definitions/140.html)

## What Gets Flagged

```python
# FLAGGED: csv.writer with user data
import csv
writer = csv.writer(response)
writer.writerow([user.name, user.email, user.comment])  # comment may start with =
```

```php
<?php
// FLAGGED: fputcsv with user data
fputcsv($output, [$_POST['name'], $_POST['comment']]);
```

## Remediation

```python
# SAFE: sanitize formula characters
def sanitize_csv(value):
    if isinstance(value, str) and value and value[0] in ('=', '+', '-', '@', '\t', '\r'):
        return "'" + value  # prefix with single quote
    return value

writer.writerow([sanitize_csv(v) for v in row])
```

```php
<?php
// SAFE: prefix formula chars
function sanitize_csv($value) {
    if (strlen($value) > 0 && in_array($value[0], ['=', '+', '-', '@'])) {
        return "'" . $value;
    }
    return $value;
}
fputcsv($output, array_map('sanitize_csv', $row));
```
