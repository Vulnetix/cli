## Overview

Insecure Direct Object Reference (IDOR) occurs when an application uses user-controlled input (like `?id=123` or `/users/456/profile`) to directly retrieve database records without verifying that the requesting user owns or has permission to access that specific record.

**Severity:** High | **CWE:** [CWE-639 – Authorization Bypass Through User-Controlled Key](https://cwe.mitre.org/data/definitions/639.html)

## Why This Matters

A user who discovers `GET /api/invoices?id=1001` returns their invoice can simply increment the ID to access other users' invoices, medical records, financial data, or private messages. IDOR is consistently in the OWASP Top 10 and is responsible for numerous major data breaches where attackers enumerate resources by iterating identifiers.

## What Gets Flagged

```python
# FLAGGED: Direct lookup by user-supplied ID, no ownership check
def get_document(request):
    doc_id = request.GET['id']
    document = Document.objects.get(id=doc_id)  # any user can access any doc
    return JsonResponse(document.to_dict())

# FLAGGED: Filter by user-supplied ID without user ownership
Order.objects.filter(id=request.GET['order_id']).first()
```

```javascript
// FLAGGED: findById with user-supplied param, no ownership check
app.get('/api/records/:id', async (req, res) => {
    const record = await Record.findById(req.params.id);
    res.json(record);
});
```

## Remediation

Always scope queries to the authenticated user's owned resources:

```python
# SAFE: scope query to current user
def get_document(request):
    doc_id = request.GET['id']
    try:
        document = Document.objects.get(id=doc_id, owner=request.user)
    except Document.DoesNotExist:
        return HttpResponseForbidden()
    return JsonResponse(document.to_dict())

# SAFE: explicit ownership check
order = Order.objects.filter(id=order_id, user=request.user).first()
if not order:
    return HttpResponseForbidden("Access denied")
```

```javascript
// SAFE: always filter by authenticated user
app.get('/api/records/:id', authenticate, async (req, res) => {
    const record = await Record.findOne({ _id: req.params.id, userId: req.user.id });
    if (!record) return res.status(403).json({ error: 'Access denied' });
    res.json(record);
});
```
