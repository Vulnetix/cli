## Overview

VNX-1056 maps to [CWE-1056: Invokable Control Element with Variadic Parameters](https://cwe.mitre.org/data/definitions/1056.html). Detects source patterns associated with CWE-1056 (Invokable Control Element with Variadic Parameters). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1056](https://cwe.mitre.org/data/definitions/1056.html) | **Languages:** c

## Why This Matters

This weakness class (Invokable Control Element with Variadic Parameters) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1056 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'void f(...' pattern
void f(...
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1056 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1056` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
