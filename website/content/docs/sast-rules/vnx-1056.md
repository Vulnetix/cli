## Overview

Detects security-relevant functions that use variadic parameter lists (`*args`, `**kwargs` in Python; `...args` in JavaScript). Variable-argument functions bypass static type checking, making it easy to omit required security-critical parameters without compile-time detection.

**Severity:** Low | **CWE:** [CWE-1056 – Invokable Control Element with Variable Number of Parameters](https://cwe.mitre.org/data/definitions/1056.html)

## Why This Matters

If `authenticate(*args)` accepts arbitrary arguments, callers can pass arguments in the wrong order or omit required ones without errors. This makes security-critical code fragile and hard to audit.

## What Gets Flagged

```python
# FLAGGED: security function with variadic args
def authenticate(*args, **kwargs):
    username = args[0]  # fragile - positional dependency
    password = kwargs.get('password')  # may be missing
```

## Remediation

```python
# SAFE: explicit typed parameters
def authenticate(username: str, password: str) -> bool:
    ...
```
