---
title: "VNX-523 – Unprotected Transport of Credentials"
description: "Detect credentials transmitted over plain HTTP instead of HTTPS, exposing them to interception."
---

## Overview

Transmitting credentials (passwords, tokens, API keys) over unencrypted HTTP allows any network observer to capture them. CWE-523 covers Basic Authentication headers sent over HTTP, HTTP URLs used in authenticated API calls, and credentials embedded in plain HTTP requests.

**Severity:** High | **CWE:** [CWE-523 – Unprotected Transport of Credentials](https://cwe.mitre.org/data/definitions/523.html)

## Why This Matters

A network attacker, compromised proxy, or malicious Wi-Fi access point can capture HTTP traffic trivially. Credentials observed in transit can be replayed immediately to take over accounts. TLS is mandatory for any request carrying sensitive data.

## What Gets Flagged

```python
# FLAGGED: Basic auth header over HTTP
import requests
resp = requests.get("http://api.example.com/data",
                    headers={"Authorization": "Basic dXNlcjpwYXNz"})
```

```javascript
// FLAGGED: HTTP with Authorization header
fetch("http://api.example.com/endpoint", {
  headers: { "Authorization": "Bearer " + token }
});
```

## Remediation

Always use HTTPS for authenticated requests:

```python
# SAFE: HTTPS endpoint
resp = requests.get("https://api.example.com/data",
                    headers={"Authorization": "Bearer " + token})
```

Enforce HTTPS in configuration and reject HTTP connections at the infrastructure level using HSTS.
