## Overview

VNX-523 maps to [CWE-523: Unprotected Transport of Credentials](https://cwe.mitre.org/data/definitions/523.html). Detects source patterns associated with CWE-523 (Unprotected Transport of Credentials). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-523](https://cwe.mitre.org/data/definitions/523.html) | **Languages:** go, java, node, python

## Why This Matters

This weakness class (Unprotected Transport of Credentials) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-523 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'http://' pattern
http://
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-523 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-523` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
