## Overview

Detects commented-out code blocks using block comments that obscure whether security checks are active. Block-commented security controls are visually similar to active code and easy to miss during review.

**Severity:** Low | **CWE:** [CWE-1113](https://cwe.mitre.org/data/definitions/1113.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1113](https://cwe.mitre.org/data/definitions/1113.html) for authoritative guidance.
