description: "Detect code that catches errors or exceptions but silently discards them — empty catch blocks, except: pass, swallowed Promise rejections, and bare Go error returns."
---

## Overview

CWE-755 covers the case where an exception or error is caught but then ignored — the opposite of CWE-754 (not checking at all). An empty `catch` block, `except: pass`, a `.catch(() => {})` callback that discards a Promise rejection, or a Go `if err != nil { return }` without any logging or wrapping all hide failures from the caller and from observability tooling. The program continues in a degraded or incorrect state without any indication that something went wrong.

**Severity:** Medium | **CWE:** [CWE-755 – Improper Handling of Exceptional Conditions](https://cwe.mitre.org/data/definitions/755.html)

## Why This Matters

Silent failure modes are a significant source of security vulnerabilities: audit log writes that silently fail mean attacks go unrecorded; authentication checks that swallow exceptions may default to "allowed"; database writes that are silently discarded cause data loss without alarming the user. In JavaScript, promise rejections that are caught and discarded mask network errors, authentication failures, and storage failures. Go's error handling discipline means callers rely on error propagation — a bare `return` without wrapping loses the context needed to diagnose problems.

## What Gets Flagged

```python
# FLAGGED: exception caught and explicitly ignored
try:
    write_audit_log(event)
except: pass

try:
    authenticate(user)
except Exception: pass
```

```javascript
// FLAGGED: Promise rejection swallowed with empty callback
fetch('/api/data')
    .then(process)
    .catch(() => {})           // rejection silently ignored

somePromise.catch(e => {})    // error parameter unused
```

```go
// FLAGGED: error returned without context or logging
if err != nil { return }
```

## Remediation

```python
# SAFE: log and re-raise, or handle specifically
try:
    write_audit_log(event)
except OSError as e:
    logger.error("Audit log write failed: %s", e)
    raise  # propagate so caller knows logging failed
```

```javascript
// SAFE: log and optionally rethrow
fetch('/api/data')
    .then(process)
    .catch((err) => {
        console.error('API request failed:', err);
        showUserError('Could not load data. Please try again.');
    });
```

```go
// SAFE: wrap error with context before returning
if err != nil {
    return fmt.Errorf("writing audit record: %w", err)
}

// SAFE: log when context makes return inappropriate
if err != nil {
    slog.Error("background sync failed", "error", err)
    return
}
```
