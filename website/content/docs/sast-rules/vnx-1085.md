## Overview

Detects large blocks of commented-out code. Commented-out code often contains outdated security logic, disabled checks, or old vulnerable implementations that confuse reviewers about the actual security posture of the system.

**Severity:** Low | **CWE:** [CWE-1085](https://cwe.mitre.org/data/definitions/1085.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1085](https://cwe.mitre.org/data/definitions/1085.html) for authoritative guidance.
