---
title: "VNX-1241 – Use of Predictable Algorithm in Random Number Generator"
description: "Detect use of non-cryptographic PRNGs (Python random, JavaScript Math.random, Java java.util.Random, PHP rand) in contexts requiring unpredictable values such as tokens, session IDs, or keys."
---

## Overview

Standard pseudo-random number generators are designed for statistical properties, not unpredictability. Given knowledge of the algorithm and seed (or enough output), an attacker can reconstruct the internal state and predict future values. Session tokens, password reset tokens, CSRF tokens, and cryptographic keys generated with these PRNGs are vulnerable to prediction attacks. **Severity:** High | **CWE:** [CWE-1241 – Use of Predictable Algorithm in Random Number Generator](https://cwe.mitre.org/data/definitions/1241.html)

## Why This Matters

In 2012, Trustwave demonstrated that PHP `rand()` seeds are predictable, allowing session token prediction. In 2019, a bug in a cryptocurrency wallet used `Math.random()` to generate private keys — resulting in fund theft. The Mersenne Twister (`mt_rand`, Python `random`) internal state can be fully reconstructed after observing 624 consecutive outputs.

## What Gets Flagged

```python
# FLAGGED: Standard random module for security values
import random
token = random.randint(100000, 999999)  # predictable OTP
session_id = ''.join(random.choices('abcdefghijklmnopqrstuvwxyz', k=32))
```

```javascript
// FLAGGED: Math.random() for token generation
const token = Math.random().toString(36).substring(2);
const sessionId = Math.random() * 0xFFFFFF;
```

```java
// FLAGGED: java.util.Random for security
Random rand = new Random();
int otp = rand.nextInt(1000000);  // predictable
```

## Remediation

```python
# SAFE: Use secrets module for security-sensitive values
import secrets

token = secrets.token_urlsafe(32)    # URL-safe base64 token
otp = secrets.randbelow(1000000)     # Cryptographic random integer
session_id = secrets.token_hex(32)   # Hex string
```

```javascript
// SAFE: Web Crypto API or Node.js crypto module
const { randomBytes } = require('crypto');
const token = randomBytes(32).toString('hex');

// Browser:
const array = new Uint8Array(32);
crypto.getRandomValues(array);
```

```java
// SAFE: java.security.SecureRandom
SecureRandom secureRandom = new SecureRandom();
int otp = secureRandom.nextInt(1000000);
byte[] tokenBytes = new byte[32];
secureRandom.nextBytes(tokenBytes);
String token = Base64.getEncoder().encodeToString(tokenBytes);
```
