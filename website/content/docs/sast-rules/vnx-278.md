## Overview

VNX-278 maps to [CWE-278: Insecure Preserved Inherited Permissions](https://cwe.mitre.org/data/definitions/278.html). Detects source patterns associated with CWE-278 (Insecure Preserved Inherited Permissions). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-278](https://cwe.mitre.org/data/definitions/278.html) | **Languages:** c

## Why This Matters

This weakness class (Insecure Preserved Inherited Permissions) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-278 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'fork(' pattern
fork(
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-278 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-278` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
