---
title: "VNX-804 – Guessable CAPTCHA"
description: "Detect CAPTCHA implementations using small random number ranges that are trivially brute-forceable within seconds."
---

## Overview

This rule flags CAPTCHA logic that generates challenges from a tiny numeric range (e.g., `random.randint(1, 10)` producing only 10 possible values). An attacker can enumerate all possibilities in a single loop iteration or submit them in rapid succession, defeating the bot-protection purpose entirely. This maps to [CWE-804: Guessable CAPTCHA](https://cwe.mitre.org/data/definitions/804.html).

**Severity:** Medium | **CWE:** [CWE-804](https://cwe.mitre.org/data/definitions/804.html)

## Why This Matters

CAPTCHAs exist to distinguish humans from automated bots. A CAPTCHA with only 10 possible answers can be brute-forced in milliseconds with no human involvement. Attackers exploit guessable CAPTCHAs to automate account creation, credential stuffing, and form spam.

## What Gets Flagged

```python
# FLAGGED: tiny range makes CAPTCHA trivially guessable
import random
captcha_answer = random.randint(1, 10)
captcha = random.randint(1, 9)
```

```php
// FLAGGED: rand with small range in CAPTCHA context
$captcha = rand(1, 10);
$_SESSION['captcha'] = rand(1, 9);
```

```javascript
// FLAGGED: Math.random with tiny range
const captchaAnswer = Math.floor(Math.random() * 10);
const captcha = Math.floor(Math.random() * 9) + 1;
```

## Remediation

```python
# SAFE: use a well-established CAPTCHA service
# Option 1: reCAPTCHA v3 (server-side verification)
import requests

def verify_recaptcha(token, secret_key):
    response = requests.post('https://www.google.com/recaptcha/api/siteverify', {
        'secret': secret_key,
        'response': token
    })
    result = response.json()
    return result.get('success') and result.get('score', 0) >= 0.5

# Option 2: hCaptcha
def verify_hcaptcha(token, secret_key):
    response = requests.post('https://hcaptcha.com/siteverify', {
        'secret': secret_key,
        'response': token
    })
    return response.json().get('success', False)
```

```javascript
// SAFE: use a CAPTCHA service with server-side verification
const fetch = require('node-fetch');

async function verifyRecaptcha(token) {
    const response = await fetch('https://www.google.com/recaptcha/api/siteverify', {
        method: 'POST',
        body: new URLSearchParams({
            secret: process.env.RECAPTCHA_SECRET,
            response: token
        })
    });
    const data = await response.json();
    return data.success && data.score >= 0.5;
}
```

If a text-based CAPTCHA is required, use cryptographically random tokens of sufficient length (at least 6 alphanumeric characters from a 62-character alphabet, giving ~57 billion combinations) and expire them after a short time window with one-use enforcement.
