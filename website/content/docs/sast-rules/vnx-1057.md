title: "VNX-1057 – Data Access Operations Outside of Expected Data Manager Component"
description: "Detect direct database access outside of repository/DAO layer files."
---

## Overview

Detects raw database queries (`db.Query`, `cursor.execute`, etc.) in files that are not part of the expected data-access layer (repository, DAO, store). Scattered database access makes it impossible to enforce consistent input validation, access controls, and audit logging.

**Severity:** Low | **CWE:** [CWE-1057 – Data Access Operations Outside of Expected Data Manager Component](https://cwe.mitre.org/data/definitions/1057.html)

## Why This Matters

When database queries are spread across controllers, views, and utility functions, there is no single place to enforce access control checks. An attacker who can trigger any code path that touches the database may bypass authorisation that was only enforced in one path.

## What Gets Flagged

```go
// FLAGGED: direct DB query in HTTP handler (not repository)
func HandleGetUser(w http.ResponseWriter, r *http.Request) {
    rows, _ := db.Query("SELECT * FROM users WHERE id = $1", id)
}
```

## Remediation

Move all database access to a dedicated repository layer:

```go
// SAFE: handler calls repository
func HandleGetUser(w http.ResponseWriter, r *http.Request) {
    user, err := userRepo.GetByID(r.Context(), id)
}

// Repository enforces access control
func (r *UserRepo) GetByID(ctx context.Context, id string) (*User, error) {
    // authorisation check here, once, consistently
}
```
