## Overview

VNX-347 maps to [CWE-347: Improper Verification of Cryptographic Signature](https://cwe.mitre.org/data/definitions/347.html). Detects source patterns associated with CWE-347 (Improper Verification of Cryptographic Signature). Each finding should be manually reviewed for exploitability in context.

**Severity:** High | **CWE:** [CWE-347](https://cwe.mitre.org/data/definitions/347.html) | **Languages:** go, java, node, python

## Why This Matters

This weakness class (Improper Verification of Cryptographic Signature) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-347 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'verify=False' pattern
verify=False
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-347 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-347` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
