## Overview

VNX-115 maps to [CWE-115: Misinterpretation of Input](https://cwe.mitre.org/data/definitions/115.html). Detects source patterns associated with CWE-115 (Misinterpretation of Input). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-115](https://cwe.mitre.org/data/definitions/115.html) | **Languages:** node, php, python

## Why This Matters

This weakness class (Misinterpretation of Input) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-115 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'int(' pattern
int(
```

```javascript
// FLAGGED: contains 'parseInt(' pattern
parseInt(
```

```php
// FLAGGED: contains '$_GET' pattern
$_GET
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-115 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-115` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
