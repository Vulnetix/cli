## Why This Matters

This weakness class (CWE-432) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-432 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-432 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-432` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
