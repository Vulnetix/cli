## Overview

Detects deeply nested code (5+ levels of indentation) that makes security code paths hard to follow and test. Deep nesting often hides security checks inside complex conditional logic, making it easy to introduce bypasses.

**Severity:** Low | **CWE:** [CWE-1124](https://cwe.mitre.org/data/definitions/1124.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1124](https://cwe.mitre.org/data/definitions/1124.html) for authoritative guidance.
