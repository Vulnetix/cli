---
title: "VNX-270 – Privilege Context Switching Error"
description: "Detect privilege transitions where the return value is not checked, leaving the process with unintended privileges."
---

## Overview

Functions like `seteuid()`, `setreuid()`, and `os.seteuid()` switch the process effective user ID. If the call fails (e.g., lacking `CAP_SETUID`) and the error is not checked, the process continues executing under the wrong privilege level. This maps to [CWE-270: Privilege Context Switching Error](https://cwe.mitre.org/data/definitions/270.html).

**Severity:** Medium | **CWE:** [CWE-270](https://cwe.mitre.org/data/definitions/270.html)

## Remediation

```c
// SAFE: check and abort on failure
if (seteuid(target_uid) != 0) {
    perror("seteuid");
    exit(EXIT_FAILURE);
}
```

```python
# SAFE: verify with try/except and confirm
try:
    os.seteuid(target_uid)
except PermissionError:
    raise RuntimeError("Could not drop privileges")
assert os.geteuid() == target_uid
```
