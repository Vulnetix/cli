---
title: "VNX-618 – Exposed Unsafe ActiveX Method"
description: "Detect ActiveX/COM object instantiation that exposes dangerous capabilities to web pages."
---

## Overview

ActiveX controls marked as safe for scripting can be invoked by any web page viewed in Internet Explorer. Controls that expose file system access, shell execution, or other privileged operations create remote code execution vectors.

**Severity:** Low | **CWE:** [CWE-618 – Exposed Unsafe ActiveX Method](https://cwe.mitre.org/data/definitions/618.html)

## Why This Matters

Historically, malicious websites used drive-by ActiveX attacks to execute arbitrary code on visitors' machines by instantiating ActiveX controls with dangerous capabilities. While Internet Explorer usage has declined, legacy applications still using ActiveX remain vulnerable.

## What Gets Flagged

```vbscript
' FLAGGED: Dangerous ActiveX object
Set shell = CreateObject("WScript.Shell")
shell.Run "cmd.exe"

Set fs = CreateObject("Scripting.FileSystemObject")
```

```html
<!-- FLAGGED: ActiveX control embedding -->
<object classid="clsid:DANGEROUS-GUID">
```

## Remediation

- Do not use ActiveX for web applications; migrate to modern browser APIs
- If ActiveX is required, audit all exposed methods and implement `IObjectSafety` to restrict usage
- Apply the kill bit for controls that should not be instantiated from web pages
