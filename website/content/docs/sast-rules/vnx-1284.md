title: "VNX-1284 – Improper Validation of Specified Quantity in Input"
description: "Detect numeric values from user input parsed without an upper-bound validation, which can cause integer overflow, excessive resource allocation, or denial of service."
---

## Overview

When user-supplied strings are parsed to integers and used as counts, sizes, limits, or loop bounds without validating a maximum value, attackers can supply values like `2147483647` or `999999999` to trigger integer overflow, allocate gigabytes of memory, create billions of loop iterations, or exhaust database connection pools. **Severity:** Medium | **CWE:** [CWE-1284 – Improper Validation of Specified Quantity in Input](https://cwe.mitre.org/data/definitions/1284.html)

## Why This Matters

Integer overflow in size calculations is the root cause of classic heap overflow vulnerabilities. Unvalidated `LIMIT` values in SQL queries can cause full table scans. Unvalidated loop counts in API endpoints can bring down a server. The fix is always the same: add a min/max check after parsing.

## What Gets Flagged

```java
// FLAGGED: No upper bound check after parse
int count = Integer.parseInt(request.getParameter("count"));
List<Item> items = getItems(count);  // count could be Integer.MAX_VALUE
```

```python
# FLAGGED: No validation after int() conversion
items_to_process = int(request.POST['count'])
for i in range(items_to_process):  # could be 2147483647 iterations
    process(i)
```

```javascript
// FLAGGED: No max check after parseInt
const limit = parseInt(req.query.limit);
const results = await db.query('SELECT * FROM items LIMIT ?', [limit]);
```

## Remediation

```java
// SAFE: Validate range after parsing
int count = Integer.parseInt(request.getParameter("count"));
if (count < 1 || count > 1000) {
    throw new IllegalArgumentException("count must be between 1 and 1000");
}
```

```python
# SAFE: Clamp or validate with explicit bounds
MAX_COUNT = 1000
items_to_process = int(request.POST.get('count', 10))
if not 1 <= items_to_process <= MAX_COUNT:
    return HttpResponseBadRequest(f"count must be 1–{MAX_COUNT}")
```

```javascript
// SAFE: Validate and clamp
const MAX_LIMIT = 100;
const limit = Math.min(Math.max(parseInt(req.query.limit, 10) || 10, 1), MAX_LIMIT);
```
