## Overview

Detects machine-dependent operations (endianness conversions, bitwise operations on typed values, raw memory operations) scattered outside of a dedicated portability layer. These should be isolated to prevent security bugs when code is ported to different architectures.

**Severity:** Low | **CWE:** [CWE-1105](https://cwe.mitre.org/data/definitions/1105.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1105](https://cwe.mitre.org/data/definitions/1105.html) for authoritative guidance.
