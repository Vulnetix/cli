## Overview

VNX-1105 maps to [CWE-1105: Insufficient Encapsulation of Machine-Dependent Functionality](https://cwe.mitre.org/data/definitions/1105.html). Detects source patterns associated with CWE-1105 (Insufficient Encapsulation of Machine-Dependent Functionality). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1105](https://cwe.mitre.org/data/definitions/1105.html) | **Languages:** java

## Why This Matters

This weakness class (Insufficient Encapsulation of Machine-Dependent Functionality) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1105 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'public int' pattern
public int
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1105 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1105` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
