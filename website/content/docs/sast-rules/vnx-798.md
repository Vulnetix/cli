title: "VNX-798 – Use of Hard-coded Credentials"
description: "Detect hard-coded passwords, API keys, tokens, and private keys embedded directly in source code."
---

## Overview

This rule flags hard-coded credentials including passwords assigned to literals, API key assignments, and PEM private key headers embedded in source code. Hard-coded secrets are exposed whenever the source code is shared, committed to version control, decompiled, or discovered via a binary search. This maps to [CWE-798: Use of Hard-coded Credentials](https://cwe.mitre.org/data/definitions/798.html).

**Severity:** Critical | **CWE:** [CWE-798](https://cwe.mitre.org/data/definitions/798.html)

## Why This Matters

Hard-coded credentials never expire and are virtually impossible to rotate without a code change and redeployment. They are trivially discoverable in public repositories via tools like `truffleHog`, `git-secrets`, and GitHub's own secret scanning. A single leaked credential can provide attackers with persistent, authorized access to production systems.

## What Gets Flagged

```python
# FLAGGED: hard-coded password assignment
password = "SuperSecret123!"
db_password = "my_database_pass"
admin_password = 'changeme'
```

```javascript
// FLAGGED: API key and token assignments
const apiKey = "AIzaSyD-9tSrke72...";
const api_key = "sk-live-abc123...";
const SECRET_TOKEN = "ghp_xxxxxxxxxxxxxxxxxxx";
```

```go
// FLAGGED: hard-coded credentials in Go
password := "hardcoded_pass"
apiKey := "AKIAIOSFODNN7EXAMPLE"
```

```python
# FLAGGED: PEM private key in source
private_key = """-----BEGIN RSA PRIVATE KEY-----
MIIEpAIBAAKCAQEA0Z3VS5JJcds3xHn/ygWep4...
-----END RSA PRIVATE KEY-----"""
```

## Remediation

```python
# SAFE: load credentials from environment variables
import os
password = os.environ['DB_PASSWORD']
api_key = os.environ['API_KEY']
```

```javascript
// SAFE: use environment variables or a secrets manager
const apiKey = process.env.API_KEY;
const dbPassword = process.env.DB_PASSWORD;

// SAFE: AWS Secrets Manager
const AWS = require('aws-sdk');
const client = new AWS.SecretsManager();
const secret = await client.getSecretValue({ SecretId: 'my/secret' }).promise();
```

```go
// SAFE: read from environment
password := os.Getenv("DB_PASSWORD")
apiKey := os.Getenv("API_KEY")
```

Use a secrets management solution (HashiCorp Vault, AWS Secrets Manager, GCP Secret Manager, Azure Key Vault) and never commit credentials to source control. Add a `.gitignore` entry for `.env` files and use pre-commit hooks or CI secret scanning to prevent accidental commits.
