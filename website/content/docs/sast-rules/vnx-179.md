---
title: "VNX-179 – Incorrect Behavior Order: Early Validation"
description: "Detect validation checks applied before path canonicalization, enabling bypass via non-canonical paths."
---

## Overview

This rule flags code that checks for path traversal sequences (`../`) before calling `os.path.realpath()`, `filepath.Clean()`, or `getCanonicalPath()`. An attacker supplies a double-encoded or backslash-variant path that passes the string check but resolves to a parent directory after normalization. This maps to [CWE-179: Incorrect Behavior Order: Early Validation](https://cwe.mitre.org/data/definitions/179.html).

**Severity:** Medium | **CWE:** [CWE-179](https://cwe.mitre.org/data/definitions/179.html)

## Remediation

```python
# SAFE: normalize FIRST, then validate
import os
real_path = os.path.realpath(os.path.join(base_dir, user_path))
if not real_path.startswith(os.path.realpath(base_dir)):
    abort(403)
```

```go
// SAFE: Go — clean then check
clean := filepath.Clean(filepath.Join(baseDir, userPath))
if !strings.HasPrefix(clean, filepath.Clean(baseDir)) {
    http.Error(w, "Forbidden", 403)
}
```
