## Overview

VNX-267 maps to [CWE-267: Privilege Defined With Unsafe Actions](https://cwe.mitre.org/data/definitions/267.html). Detects source patterns associated with CWE-267 (Privilege Defined With Unsafe Actions). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-267](https://cwe.mitre.org/data/definitions/267.html) | **Languages:** c, cpp

## Why This Matters

This weakness class (Privilege Defined With Unsafe Actions) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-267 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```c
// FLAGGED: contains 'CAP_SYS_ADMIN' pattern
CAP_SYS_ADMIN
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-267 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-267` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
