## Overview

VNX-1079 maps to [CWE-1079: Parent Class without Virtual Destructor Method](https://cwe.mitre.org/data/definitions/1079.html). Detects source patterns associated with CWE-1079 (Parent Class without Virtual Destructor Method). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1079](https://cwe.mitre.org/data/definitions/1079.html) | **Languages:** cpp

## Why This Matters

This weakness class (Parent Class without Virtual Destructor Method) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1079 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```cpp
// FLAGGED: contains 'class Base' pattern
class Base
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1079 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1079` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
