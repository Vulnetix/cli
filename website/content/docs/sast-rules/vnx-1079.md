title: "VNX-1079 – Parent Class without Virtual Destructor Method"
description: "Detect C++ base classes with virtual methods but without a virtual destructor, causing use-after-free."
---

## Overview

Detects C++ base classes that have virtual methods but lack a virtual destructor. Deleting a derived object through a base class pointer without a virtual destructor invokes only the base destructor, causing resource leaks and undefined behaviour (potential use-after-free or double-free).

**Severity:** Medium | **CWE:** [CWE-1079 – Parent Class without Virtual Destructor Method](https://cwe.mitre.org/data/definitions/1079.html)

## Why This Matters

Use-after-free vulnerabilities are exploitable for arbitrary code execution. When a derived class manages sensitive resources (cryptographic keys, file handles) and its destructor is never called, those resources leak and can potentially be accessed after they should have been destroyed.

## What Gets Flagged

```cpp
// FLAGGED: virtual methods but no virtual destructor
class AuthProvider {
public:
    virtual bool authenticate(const std::string& token) = 0;
    // missing: virtual ~AuthProvider() = default;
};
```

## Remediation

```cpp
class AuthProvider {
public:
    virtual ~AuthProvider() = default;  // REQUIRED
    virtual bool authenticate(const std::string& token) = 0;
};
```
