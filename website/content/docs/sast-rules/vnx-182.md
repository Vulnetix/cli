title: "VNX-182 – Collapse of Data into Unsafe Value"
description: "Detect single-pass script tag removal that produces a new script tag from nested payloads."
---

## Overview

Removing `<script>` from `<scr<script>ipt>` yields `<script>`, creating the very string that was meant to be removed. Single-pass string replacement sanitization is never safe because the filtering step itself generates new attack payloads. This maps to [CWE-182: Collapse of Data into Unsafe Value](https://cwe.mitre.org/data/definitions/182.html).

**Severity:** Medium | **CWE:** [CWE-182](https://cwe.mitre.org/data/definitions/182.html)

## Remediation

Replace string-based sanitization with a proper HTML sanitization library (DOMPurify, bleach, OWASP Java HTML Sanitizer) that parses the DOM and applies an allowlist of safe elements.
