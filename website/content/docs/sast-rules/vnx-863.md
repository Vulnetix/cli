---
title: "VNX-863 – Incorrect Authorization"
description: "Detect authorization checks that use the wrong permission, role, or condition, enabling privilege escalation."
---

## Overview

Incorrect authorization occurs when the application performs an access check but uses the wrong permission or role. Common variants include checking user identity attributes (email, username) instead of roles, comparing against a weaker permission than required, or using loose equality for role comparisons.

**Severity:** High | **CWE:** [CWE-863 – Incorrect Authorization](https://cwe.mitre.org/data/definitions/863.html)

## Why This Matters

An attacker who satisfies the wrong condition gains access they should not have. For example, checking `if user.email == 'admin@example.com'` is bypassed by any user with that email, not just administrators with the admin role.

## What Gets Flagged

```python
# FLAGGED: Identity check instead of role check
if request.user.email == "admin@company.com":
    return admin_panel()
```

```php
<?php
// FLAGGED: Loose role comparison
if ($_SESSION['role'] == 'admin') { ... }  // should use ===
```

## Remediation

Check roles/permissions, not identity attributes:

```python
# SAFE: Permission-based check
if request.user.has_perm('app.access_admin_panel'):
    return admin_panel()
```

```php
<?php
// SAFE: Strict role comparison
if ($_SESSION['role'] === 'admin') { ... }
```
