title: "VNX-682 – Incorrect Calculation"
description: "Detect integer overflow, off-by-one errors, and truncating size calculations used in security-sensitive contexts like buffer allocation."
---

## Overview

Incorrect calculations in security-sensitive contexts — buffer size calculations, array bounds, loop limits — lead to buffer overflows, heap corruption, and memory safety violations. Classic patterns include `malloc(strlen(s))` missing the null terminator byte, integer overflow in multiplication for buffer sizing, and unsigned/signed conversion issues that make negative numbers appear as large positive values.

**Severity:** Medium | **CWE:** [CWE-682 – Incorrect Calculation](https://cwe.mitre.org/data/definitions/682.html)

## Why This Matters

`malloc(strlen(s))` allocates one byte too few — writing the null terminator overflows the buffer. `malloc(count * sizeof(struct))` overflows to a small allocation when `count` is large, then the subsequent data copy overflows the heap. These calculation errors are a leading cause of exploitable heap corruption vulnerabilities in C/C++ applications.

## What Gets Flagged

```c
// FLAGGED: strlen without +1 for null terminator
char *copy = malloc(strlen(src));
strcpy(copy, src);  // null terminator overflows

// FLAGGED: Integer multiplication that can overflow
size_t buf_size = count * sizeof(struct record);  // overflow if count is large
char *buf = malloc(buf_size);

// FLAGGED: Signed-to-unsigned conversion creating huge values
int user_len = get_user_length();  // could be negative
char *buf = malloc((size_t)user_len);  // -1 becomes SIZE_MAX
```

## Remediation

Account for null terminators, validate inputs before calculations, and use overflow-safe functions:

```c
// SAFE: +1 for null terminator
char *copy = malloc(strlen(src) + 1);
if (copy == NULL) return NULL;
strcpy(copy, src);

// SAFE: Check for overflow before multiplication
if (count > SIZE_MAX / sizeof(struct record)) {
    return NULL;  // would overflow
}
size_t buf_size = count * sizeof(struct record);
char *buf = malloc(buf_size);

// SAFE: Validate range before conversion
if (user_len < 0 || user_len > MAX_ALLOWED) {
    return NULL;
}
char *buf = malloc((size_t)user_len);
```

In C++, use `std::vector` and `std::string` which handle size management automatically.
