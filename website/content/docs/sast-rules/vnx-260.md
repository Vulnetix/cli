---
title: "VNX-260 – Password in Configuration File"
description: "Detect plaintext passwords and credentials in configuration files committed to version control."
---

## Overview

Passwords stored in `.env`, `application.yml`, `config.php`, or similar configuration files are exposed to anyone with repository access, and frequently end up in logs, backups, and version history. This maps to [CWE-260: Password in Configuration File](https://cwe.mitre.org/data/definitions/260.html).

**Severity:** High | **CWE:** [CWE-260](https://cwe.mitre.org/data/definitions/260.html)

## What Gets Flagged

```yaml
# FLAGGED: application.yml with real password
spring:
  datasource:
    password: Sup3rS3cr3t!
```

```bash
# FLAGGED: .env with password
DB_PASSWORD=mysecretpassword
```

## Remediation

1. Use environment variables, not config files, for secrets.
2. Use a secrets manager (HashiCorp Vault, AWS Secrets Manager, GCP Secret Manager).
3. Add `.env` and config files with secrets to `.gitignore`.
4. Rotate any credentials already committed to version control.
