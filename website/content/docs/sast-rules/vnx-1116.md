## Overview

Detects comments that contradict the code they describe, particularly around security controls. Inaccurate security comments mislead reviewers into thinking controls are in place when they are not (or vice versa).

**Severity:** Low | **CWE:** [CWE-1116](https://cwe.mitre.org/data/definitions/1116.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1116](https://cwe.mitre.org/data/definitions/1116.html) for authoritative guidance.
