## Overview

VNX-437 maps to [CWE-437: Incomplete Model of Endpoint Features](https://cwe.mitre.org/data/definitions/437.html). Detects source patterns associated with CWE-437 (Incomplete Model of Endpoint Features). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-437](https://cwe.mitre.org/data/definitions/437.html) | **Languages:** node, python

## Why This Matters

This weakness class (Incomplete Model of Endpoint Features) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-437 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```python
// FLAGGED: contains 'except NotImplementedError' pattern
except NotImplementedError
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-437 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-437` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
