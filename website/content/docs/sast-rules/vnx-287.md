## Overview

This rule flags patterns where authentication is disabled, bypassed, or improperly implemented. It covers JWT signature verification disabled via `algorithm: 'none'` or `verify_signature: False`, SSL certificate verification bypassed with `verify=False`, and other patterns that remove authentication guarantees. This maps to [CWE-287: Improper Authentication](https://cwe.mitre.org/data/definitions/287.html).

**Severity:** Critical | **CWE:** [CWE-287 – Improper Authentication](https://cwe.mitre.org/data/definitions/287.html)

## Why This Matters

Authentication is the gatekeeper that determines who is allowed to access a system. When authentication is bypassed or improperly implemented, any user — authenticated or not — can access protected resources, perform privileged operations, or impersonate other users. The JWT `algorithm: 'none'` vulnerability is particularly dangerous: it instructs the library to skip cryptographic verification entirely, accepting any token as valid regardless of the signature. This was a real-world vulnerability in several JWT libraries (CVE-2015-9235 for node-jsonwebtoken) that allowed attackers to forge tokens for arbitrary users. Similarly, Python's `verify=False` in requests disables TLS certificate verification, making the connection vulnerable to man-in-the-middle attacks that can intercept and modify authentication credentials.

## What Gets Flagged

- JWT algorithm none: `"alg": "none"`, `algorithm: 'none'`, `algorithms=["none"]`
- PyJWT verification disabled: `options={"verify_signature": False}`
- Python requests SSL bypass: `requests.get(verify=False`, `requests.post(verify=False`
- PHP loose comparison: `if ($password ==` (loose `==` vs strict `===`)
- JWT library misuse producing unauthenticated decodes

```python
# FLAGGED: JWT verification disabled in Python
import jwt
payload = jwt.decode(token, options={"verify_signature": False})
payload = jwt.decode(token, algorithms=["none"])
```

```javascript
// FLAGGED: algorithm none in JWT header
const token = jwt.sign(payload, '', { algorithm: 'none' });
const claims = jwt.decode(token); // no verification
```

```python
# FLAGGED: SSL verification disabled
import requests
response = requests.get("https://api.example.com/users", verify=False)
```

```php
// FLAGGED: loose comparison for authentication
if ($password == $storedHash) {
    // PHP loose comparison allows type juggling attacks
}
```

## Remediation

1. **Always verify JWT signatures** with a strong algorithm. Never accept `none` as a valid algorithm.

```python
# SAFE: verify signature, explicit algorithm allowlist
payload = jwt.decode(token, secret_key, algorithms=["HS256"])
```

```javascript
// SAFE: use jwt.verify, not jwt.decode; specify algorithms
const payload = jwt.verify(token, publicKey, { algorithms: ['RS256'] });
```

2. **Never disable SSL verification in production.** If you need to trust a custom CA, configure it explicitly.

```python
# SAFE: specify custom CA bundle instead of disabling verification
response = requests.get(url, verify='/path/to/ca-bundle.crt')
```

3. **Use strict comparison for credentials in PHP.**

```php
// SAFE: strict comparison
if ($password === $storedHash) { ... }
// Better: use password_verify() for password comparison
if (password_verify($password, $storedHash)) { ... }
```

4. **Test authentication paths.** Automated tests should explicitly verify that unauthenticated requests are rejected with 401/403.
