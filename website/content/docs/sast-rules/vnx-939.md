---
title: "VNX-939 – Improper Authorization in Handler for Custom URL Scheme"
description: "Detect mobile deep link and custom URL scheme handlers that process parameters without verifying the caller's identity."
---

## Overview

Mobile applications register custom URL schemes (e.g. `myapp://`) and Android intent filters that allow any other app or web page to invoke them. Without verifying the caller's identity and intent, an attacker can craft a URL that triggers privileged operations such as auto-login, fund transfers, or OAuth token exchanges.

**Severity:** Medium | **CWE:** [CWE-939 – Improper Authorization in Handler for Custom URL Scheme](https://cwe.mitre.org/data/definitions/939.html)

## Why This Matters

Deep link hijacking can redirect OAuth flows, bypass authentication, or trigger actions without user consent. This is particularly dangerous for payment and authentication flows.

## What Gets Flagged

```java
// FLAGGED: Intent handler without permission check
public void onNewIntent(Intent intent) {
    String action = intent.getAction();  // no checkPermission call
    processAction(action);
}
```

```javascript
// FLAGGED: Deep link handler without auth
function handleDeepLink(url) {
    processUrl(url);  // no authentication check
}
```

## Remediation

Verify the caller identity and require authentication for sensitive deep link operations:

```java
// SAFE
public void onNewIntent(Intent intent) {
    if (isAuthenticated()) {
        String pkg = getCallingPackage();
        if (TRUSTED_PACKAGES.contains(pkg)) {
            processAction(intent.getAction());
        }
    }
}
```
