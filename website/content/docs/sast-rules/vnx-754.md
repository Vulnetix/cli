description: "Detect code that ignores error return values in Go, C, Java, and Python — a root cause of silent failures and undefined behaviour."
---

## Overview

Every function that can fail should have its return value checked. Ignoring error returns means the program continues executing in a state that was never intended, often with uninitialized or invalid data. In Go, the blank identifier `_` explicitly discards an error; in C, `malloc` and `open` return `NULL` and `-1` respectively when they fail; in Java and Python, broad exception handlers can mask all failure modes. CWE-754 covers the failure to check at all, as distinct from CWE-755 which covers checking but discarding the result.

**Severity:** Medium | **CWE:** [CWE-754 – Improper Check for Unusual or Exceptional Conditions](https://cwe.mitre.org/data/definitions/754.html)

## Why This Matters

In many security vulnerabilities the actual bug is not in the dangerous operation itself but in the error-handling path that was never reached. A `malloc` failure that isn't checked produces a NULL pointer which is subsequently dereferenced — a guaranteed crash that becomes a DoS vector. In Go, discarding an error from a write to a network connection means the caller never knows the data wasn't sent — which can bypass audit logging or security controls. Unchecked `open()` return values in C mean subsequent reads operate on file descriptor `-1`, causing unpredictable behaviour.

## What Gets Flagged

```go
// FLAGGED: error explicitly discarded with blank identifier
_, _ = fmt.Fprintf(w, "data")
_, err := db.Query(sql)  // err never checked below
```

```c
/* FLAGGED: malloc return not checked before use */
char *buf = malloc(size);
memcpy(buf, src, size);  // buf could be NULL

/* FLAGGED: file descriptor not validated */
int fd = open(path, O_RDONLY);
read(fd, buf, sizeof(buf));  // fd could be -1
```

```java
// FLAGGED: catch block that could mask any exception
try {
    riskyOperation();
} catch (Exception e) {
    // nothing here
}
```

```python
# FLAGGED: bare except silently swallows every exception type
try:
    connect()
except:
    pass
```

## Remediation

```go
// SAFE: check every error explicitly
n, err := fmt.Fprintf(w, "data")
if err != nil {
    return fmt.Errorf("writing response: %w", err)
}
_ = n  // only the byte count is intentionally ignored
```

```c
/* SAFE: always validate malloc and syscall returns */
char *buf = malloc(size);
if (buf == NULL) {
    perror("malloc");
    return -1;
}

int fd = open(path, O_RDONLY);
if (fd == -1) {
    perror("open");
    return -1;
}
```

```java
// SAFE: handle meaningfully or re-throw with context
try {
    riskyOperation();
} catch (IOException e) {
    log.error("Operation failed", e);
    throw new ServiceException("Could not complete operation", e);
}
```

```python
# SAFE: catch specific exceptions and handle them
try:
    connect()
except ConnectionRefusedError as e:
    logger.error("Connection refused: %s", e)
    raise
```
