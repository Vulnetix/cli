---
title: "VNX-665 – Improper Initialization"
description: "Detect uninitialized pointer, boolean, and integer variables in C/C++ that may contain security-relevant garbage values."
---

## Overview

Variables declared without initialization in C and C++ contain whatever memory was previously at that address ("garbage values"). If the uninitialized variable is used in a security decision (authentication check, buffer size calculation, permission flag) before being assigned, the decision is made on an unpredictable value that can vary between debug and release builds, between runs, and between platforms.

**Severity:** Medium | **CWE:** [CWE-665 – Improper Initialization](https://cwe.mitre.org/data/definitions/665.html)

## Why This Matters

An uninitialized `bool authenticated;` that happens to contain `0x01` at runtime will pass `if (authenticated)` checks without any login. In optimized builds, compilers may reuse stack space from previous function calls, meaning an attacker who controls the call stack can influence the "random" initial value of security-critical variables.

## What Gets Flagged

```c
// FLAGGED: Uninitialized pointer — may be non-NULL
char *ptr;
if (ptr) {
    process_data(ptr);  // uses garbage pointer
}

// FLAGGED: Uninitialized bool used for auth
bool authenticated;
if (authenticated) {  // unpredictable
    grant_access();
}

// FLAGGED: Uninitialized int used as buffer size
int size;
char *buf = malloc(size);  // malloc with garbage size
```

```cpp
// FLAGGED: Uninitialized member used in security check
class Session {
    bool is_admin;  // not initialized in constructor
public:
    bool checkAdmin() { return is_admin; }
};
```

## Remediation

Always initialize variables at the point of declaration:

```c
// SAFE: Explicit initialization
char *ptr = NULL;
bool authenticated = false;
int size = 0;

// Check before use
if (ptr != NULL) {
    process_data(ptr);
}
```

```cpp
// SAFE: Initialize in constructor initializer list
class Session {
    bool is_admin = false;  // C++11 in-class initializer
public:
    bool checkAdmin() { return is_admin; }
};
```

Enable compiler warnings (`-Wall -Wuninitialized`) and use static analysis tools to catch uninitialized variables at build time.
