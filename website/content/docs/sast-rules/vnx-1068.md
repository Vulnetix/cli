## Overview

Detects TODO/FIXME comments that indicate known deviations between the implementation and intended design, particularly in security-relevant code. These markers often indicate temporary bypasses of security controls that were never resolved.

**Severity:** Low | **CWE:** [CWE-1068](https://cwe.mitre.org/data/definitions/1068.html)

## Why This Matters

Review the flagged patterns and assess whether they represent security risks in your codebase. Even quality/maintainability weaknesses can have security implications when they affect how security-critical code is written and audited.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule metadata.
3. Consult [CWE-1068](https://cwe.mitre.org/data/definitions/1068.html) for authoritative guidance.
