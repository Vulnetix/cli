description: "Detect unbounded queries and findAll() patterns that fetch entire collections without pagination."
---

## Overview

Detects patterns that fetch all records without pagination or limits. `findAll()`, `.all()`, and similar operations on large tables load entire collections into memory, cause excessive database load, and can be triggered by attackers to cause denial of service.

**Severity:** Low | **CWE:** [CWE-1060 – Excessive Number of Inefficient Server-Side Data Access Requests](https://cwe.mitre.org/data/definitions/1060.html)

## Why This Matters

A table with 10 million rows fetched with `.all()` will exhaust server memory and cause the application to crash or time out. If this can be triggered by an unauthenticated endpoint, it is a DoS vulnerability.

## What Gets Flagged

```python
# FLAGGED: fetches all records
users = User.objects.all()

# FLAGGED
records = session.query(AuditLog).fetchall()
```

## Remediation

```python
# SAFE: paginated query
users = User.objects.order_by('id')[offset:offset+page_size]

# SAFE: with limit
records = session.query(AuditLog).limit(100).offset(page * 100).all()
```
