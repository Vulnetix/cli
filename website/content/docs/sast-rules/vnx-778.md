title: "VNX-778 – Insufficient Logging"
description: "Detect authentication failure responses returned without nearby security event logging, making attack detection impossible."
---

## Overview

This rule flags authentication and authorization failure responses (HTTP 401/403) returned without a nearby log statement. Missing security event logging prevents detection of brute force attacks, credential stuffing, and unauthorized access attempts. This maps to [CWE-778: Insufficient Logging](https://cwe.mitre.org/data/definitions/778.html).

**Severity:** Medium | **CWE:** [CWE-778](https://cwe.mitre.org/data/definitions/778.html)

## Why This Matters

OWASP lists "Security Logging and Monitoring Failures" as a Top 10 risk. Without logging authentication failures, attackers can silently perform millions of login attempts with no trace in the logs. Compliance frameworks (PCI DSS, SOC 2, ISO 27001) require logging of all authentication events.

## What Gets Flagged

```javascript
// FLAGGED: 401 returned without logging
app.post('/login', (req, res) => {
    const user = db.findUser(req.body.username);
    if (!user || !user.verifyPassword(req.body.password)) {
        return res.status(401).json({ error: 'Invalid credentials' });
        // no logging of the failure!
    }
});
```

```python
# FLAGGED: abort(401) without logging
@app.route('/api/resource')
def get_resource():
    if not current_user.is_authenticated:
        abort(401)  # no log of who tried to access this
```

## Remediation

```javascript
// SAFE: log authentication failures with context
app.post('/login', (req, res) => {
    const user = db.findUser(req.body.username);
    if (!user || !user.verifyPassword(req.body.password)) {
        logger.warn('Authentication failure', {
            username: req.body.username,
            ip: req.ip,
            userAgent: req.get('User-Agent'),
            timestamp: new Date().toISOString(),
        });
        return res.status(401).json({ error: 'Invalid credentials' });
    }
});
```

```python
# SAFE: log before aborting
import logging
logger = logging.getLogger(__name__)

@app.route('/api/resource')
def get_resource():
    if not current_user.is_authenticated:
        logger.warning(
            "Unauthorized access attempt",
            extra={'ip': request.remote_addr, 'path': request.path}
        )
        abort(401)
```
