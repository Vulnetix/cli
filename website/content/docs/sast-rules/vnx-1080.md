## Overview

VNX-1080 maps to [CWE-1080: Source Code File with Excessive Number of Lines of Code](https://cwe.mitre.org/data/definitions/1080.html). Detects source patterns associated with CWE-1080 (Source Code File with Excessive Number of Lines of Code). Each finding should be manually reviewed for exploitability in context.

**Severity:** Low | **CWE:** [CWE-1080](https://cwe.mitre.org/data/definitions/1080.html) | **Languages:** python

## Why This Matters

This weakness class (Source Code File with Excessive Number of Lines of Code) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-1080 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-1080 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-1080` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
