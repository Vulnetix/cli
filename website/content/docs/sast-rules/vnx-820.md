## Overview

VNX-820 maps to [CWE-820: Missing Synchronization](https://cwe.mitre.org/data/definitions/820.html). Detects source patterns associated with CWE-820 (Missing Synchronization). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-820](https://cwe.mitre.org/data/definitions/820.html) | **Languages:** java

## Why This Matters

This weakness class (Missing Synchronization) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-820 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```java
// FLAGGED: contains 'static int' pattern
static int
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-820 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-820` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
