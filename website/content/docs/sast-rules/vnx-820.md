---
title: "VNX-820 – Missing Synchronization"
description: "Detect shared mutable state in multi-threaded Java or Go code accessed without locks, volatile, or atomic operations."
---

## Overview

This rule flags static mutable fields in Java files that spawn threads, and goroutine usage in Go without any synchronization primitives. Shared mutable state read or written from multiple threads without synchronization causes data races, producing non-deterministic behavior, stale reads, or torn writes. This maps to [CWE-820: Missing Synchronization](https://cwe.mitre.org/data/definitions/820.html).

**Severity:** High | **CWE:** [CWE-820](https://cwe.mitre.org/data/definitions/820.html)

## Why This Matters

The Java Memory Model and Go memory model both allow compilers and CPUs to reorder memory operations. Without synchronization, one thread may never see updates written by another, or may see a partially-written value. This causes bugs ranging from stale cache data to security bypasses (e.g., a flag that should be set after authentication remains unset in one thread).

## What Gets Flagged

```java
// FLAGGED: static mutable int shared across threads without volatile/synchronized
public class RequestCounter {
    static int count = 0;  // not volatile, not AtomicInteger
    
    public void handleRequest() {
        count++;  // data race: non-atomic read-modify-write
    }
}
```

```java
// FLAGGED: static List shared across threads without synchronization
public class SessionStore {
    static List<Session> sessions = new ArrayList<>();  // not CopyOnWriteArrayList, not synchronized
    
    public void addSession(Session s) {
        sessions.add(s);  // concurrent modification race
    }
}
```

```go
// FLAGGED: goroutine accessing shared variable without sync
var counter int

func increment() {
    go func() {
        counter++  // data race: concurrent write without mutex
    }()
}
```

## Remediation

```java
// SAFE: use AtomicInteger for counter
import java.util.concurrent.atomic.AtomicInteger;

public class RequestCounter {
    static AtomicInteger count = new AtomicInteger(0);
    
    public void handleRequest() {
        count.incrementAndGet();  // atomic
    }
}

// SAFE: volatile for simple flag
static volatile boolean shutdownRequested = false;

// SAFE: CopyOnWriteArrayList for infrequently-written shared list
import java.util.concurrent.CopyOnWriteArrayList;
static List<Session> sessions = new CopyOnWriteArrayList<>();
```

```go
// SAFE: use sync.Mutex
import "sync"

var (
    mu      sync.Mutex
    counter int
)

func increment() {
    go func() {
        mu.Lock()
        counter++
        mu.Unlock()
    }()
}

// SAFE: use sync/atomic for counters
import "sync/atomic"

var counter int64

func increment() {
    go func() {
        atomic.AddInt64(&counter, 1)
    }()
}
```
