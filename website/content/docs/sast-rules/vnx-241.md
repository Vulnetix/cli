## Overview

CRLF sequences (`\r\n`) in user input can inject additional HTTP headers, forge log lines, or confuse parsers that treat newlines as record separators. Mixed escaping — where some instances of a character are escaped but others from concatenation are not — enables injection through unescaped portions. This maps to [CWE-241: Improper Handling of Inconsistent Special Elements](https://cwe.mitre.org/data/definitions/241.html).

**Severity:** Low | **CWE:** [CWE-241](https://cwe.mitre.org/data/definitions/241.html)

## Remediation

Strip or encode CRLF from user-supplied values before including them in HTTP headers or log records.
