## Overview

Detects direct file system or external storage access (reading config files, credential files, or sensitive paths) from components that should not be handling raw I/O. Like CWE-1057 but focused on file and external storage access rather than database queries.

**Severity:** Low | **CWE:** [CWE-1083](https://cwe.mitre.org/data/definitions/1083.html)

## Why This Matters

Review the flagged patterns in context. Even quality and maintainability weaknesses can have security implications when they affect how security-critical code is structured, audited, and tested.

## Remediation

1. Review all flagged instances in context.
2. Refactor code to follow the recommended patterns described in the rule.
3. Consult [CWE-1083](https://cwe.mitre.org/data/definitions/1083.html) for authoritative guidance.
