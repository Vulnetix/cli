## Overview

VNX-346 maps to [CWE-346: Origin Validation Error](https://cwe.mitre.org/data/definitions/346.html). Detects source patterns associated with CWE-346 (Origin Validation Error). Each finding should be manually reviewed for exploitability in context.

**Severity:** Medium | **CWE:** [CWE-346](https://cwe.mitre.org/data/definitions/346.html) | **Languages:** node, python

## Why This Matters

This weakness class (Origin Validation Error) creates a concrete exploit surface: the rule searches for the concrete source-level patterns most commonly associated with CWE-346 and surfaces them for review. Each finding should be evaluated in context — the rule catches the pattern, not the context.

## What Gets Flagged

```javascript
// FLAGGED: contains 'cors({' pattern
cors({
```

```python
// FLAGGED: contains 'CORS_ORIGIN_ALLOW_ALL = True' pattern
CORS_ORIGIN_ALLOW_ALL = True
```

## Remediation

1. Review each flagged line and determine whether the pattern represents a real instance of CWE-346 or a false positive.
2. Replace the flagged construct with a documented safe alternative appropriate to your language and framework.
3. For confirmed false positives, add a `# vulnetix-ignore: VNX-346` comment on the line.
4. Ensure equivalent test coverage exists to prevent regression.
