---
title: "VNX-698 – Execution After Redirect (EAR)"
description: "Detect PHP and Node.js code that continues executing after sending a redirect response, allowing bypass of redirect-based access control."
---

## Overview

Execution After Redirect (EAR) occurs when code continues running after sending an HTTP redirect response. In PHP, `header("Location: /login")` sends the redirect but does not stop execution — subsequent code including database queries, file operations, and data rendering continues to run. An attacker using a proxy or curl can capture the response body (ignoring the redirect) and read the protected data.

**Severity:** High | **CWE:** [CWE-698 – Execution After Redirect](https://cwe.mitre.org/data/definitions/698.html)

## Why This Matters

Many PHP developers assume that `header("Location: /login")` works like a return statement. It does not — it sets a response header and continues executing. A proxy tool like Burp Suite or a simple `curl -v` captures the 302 response body, which contains the "protected" page content rendered for the unauthenticated request. This completely bypasses redirect-based authentication gates.

## What Gets Flagged

```php
<?php
// FLAGGED: header redirect without exit — code continues executing
function require_auth() {
    if (!isset($_SESSION['user'])) {
        header("Location: /login");
        // Missing exit; — the rest of the page renders anyway
    }
}
require_auth();
echo "Secret data: " . $secret;  // This executes even for unauthenticated users
```

```php
<?php
// FLAGGED: die() or exit() not called after redirect
header("Location: /login.php");
$data = getSecretData();  // still executes
```

```javascript
// FLAGGED: res.redirect without return
app.get('/admin', (req, res) => {
    if (!req.user.isAdmin) {
        res.redirect('/login');
        // missing return — code continues
    }
    res.render('admin', { secrets: getAdminData() });  // still executes
});
```

## Remediation

Always terminate execution immediately after sending a redirect:

```php
<?php
// SAFE: exit after redirect
function require_auth() {
    if (!isset($_SESSION['user'])) {
        header("Location: /login");
        exit;  // REQUIRED: stops all further execution
    }
}
```

```javascript
// SAFE: return after redirect
app.get('/admin', (req, res) => {
    if (!req.user.isAdmin) {
        return res.redirect('/login');  // return stops handler execution
    }
    res.render('admin', { secrets: getAdminData() });
});
```
